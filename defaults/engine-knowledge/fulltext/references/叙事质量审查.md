# 叙事质量审查 `[M15c·通用]`

> 来源：NovelGenerator 审核闭环 → 世界树适配为非阻塞  
> 定位：M15 规则审核的子模块，补充「叙事质量」维度

---

## 与 M15 主模块的关系

```
M15 世界规则机制
├── M15 主模块：行为可行性审核（物理/魔法/社会规则）
├── M15a：物理/现实规则（校园/都市/日常）
├── M15b：魔法/超能力规则（奇幻/科幻/武侠）
└── M15c：叙事质量审查（本模块）  ← 新增
```

M15 主模块审核「能不能做」，M15c 审核「做得好不好」。

---

## 核心区别：非阻塞审查

| | NovelGenerator 审核闭环 | 世界树 M15c |
|---|---|---|
| 时机 | 写完后立即审核 | 叙事推进后自动运行 |
| 不通过后果 | **重写**（最多3轮） | **标记+建议**，不中断 |
| 用户感知 | 不可见（内部循环） | 可见标记 + 下次提示 |
| 目的 | 确保输出质量 | 提升叙事一致性 |

**为什么世界树不能「不通过就重写」**：互动叙事是实时进行的，用户每轮都在参与。阻塞式重写会破坏互动体验。

## 四维审查

### ① 角色一致性

| 检查项 | 标准 | 标记 |
|--------|------|------|
| 行为一致性 | 角色行为是否符合人物档案（性格/缺陷/声纹）？ | ⚠️ 角色偏离 |
| 对话声音 | 遮名测试：不看名字能分辨谁在说话？ | ⚠️ 声纹模糊 |
| 成长弧线 | 角色变化是否有铺垫？（不能突然变性格） | ⚠️ 突变无铺垫 |
| 关系动态 | 角色间互动是否符合之前建立的关系？ | ⚠️ 关系不一致 |

### ② 场景连续性

| 检查项 | 标准 | 标记 |
|--------|------|------|
| 时间连续 | 时间线推进合理？（不能上一秒白天下一秒深夜） | ⚠️ 时间跳跃 |
| 空间连续 | 角色位置合理？（不能在两个场景同时出现） | ⚠️ 空间矛盾 |
| 道具追踪 | 已获得的道具/物品是否还在？ | ⚠️ 道具丢失 |
| 状态继承 | 角色体力/情绪/伤势是否从上一场景正确继承？ | ⚠️ 状态重置 |

### ③ 叙事节奏

| 检查项 | 标准 | 标记 |
|--------|------|------|
| 冲突密度 | 当前场景是否有足够的张力/冲突？ | ⚠️ 节奏拖沓 |
| 信息密度 | 是否过度倾泻设定？（参照反信息倾泻系统） | ⚠️ 信息过载 |
| 场景目的 | 当前场景是否推进了剧情或角色？ | ⚠️ 场景空洞 |
| 节奏变化 | 连续多场景高张力后是否有喘息？连续松弛后是否有推进？ | ⚠️ 节奏失调 |

### ④ 风格一致性

| 检查项 | 标准 | 标记 |
|--------|------|------|
| 风格匹配 | 叙事风格是否与当前预设一致？ | ⚠️ 风格偏移 |
| 感官覆盖 | 场景描写是否覆盖≥2种感官？ | ⚠️ 感官贫乏 |
| 对话比例 | 对话是否过少或过多？ | ⚠️ 对话失调 |
| 描写深度 | 关键时刻是否有足够的慢镜头/细节？ | ⚠️ 描写不足 |

## 审查流程

```
叙事推进完成（毛球输出一段叙事）
    │
    ▼
┌─────────────────────────────┐
│ M15c 自动运行（非阻塞）       │
│                             │
│ ① 角色一致性 扫描            │
│ ② 场景连续性 扫描            │
│ ③ 叙事节奏 扫描              │
│ ④ 风格一致性 扫描            │
└─────────────────────────────┘
    │
    ▼
┌─────────────────────────────┐
│ 结果处理                     │
│                             │
│ ✅ 全部通过 → 无操作          │
│ ⚠️ 有问题 → 标记到追踪系统    │
│             不中断叙事流程    │
│             下次推进时展示提示 │
└─────────────────────────────┘
```

## 标记格式

审查不通过时，在 Tracking 追踪系统中追加标记：

```
[叙事质量·第N场景] ⚠️ 角色偏离
  角色「艾琳」在本场景中的行为与其人物档案不一致：
  - 档案设定：谨慎多疑，不会轻易信任陌生人
  - 本场景表现：对初次见面的商人毫无戒备地透露了关键信息
  建议：在后续场景中补充「为什么艾琳这次破例」（如：商人持有某种信物）

[叙事质量·第N场景] ⚠️ 节奏拖沓
  当前场景已连续3轮无冲突推进。建议：
  - 引入一个意外事件（参照M17）
  - 或推进到下一场景
  - 或让其中一个角色主动打破现状
```

---

## 下次推进时的提示

当 tracking 中存在未解决的叙事质量标记时，毛球在推进前追加**一行提示**：

```
💡 上次叙事提醒：艾琳的行为与她多疑的性格有出入，考虑在本次推进中补充动机。
```

用户可以选择：
- 无视提示，继续
- 在本次叙事中自然修正
- 用 `/追踪 dismiss [标记ID]` 关闭提示

## 严格度

与 M15 严格度联动：

| 严格度 | M15c 行为 |
|--------|----------|
| 铁律 | 全维度审查，所有标记展示 |
| 严格 | 角色一致性 + 场景连续性（跳过节奏和风格） |
| 正常 | 仅角色一致性（轻量扫描） |
| 宽松 | 仅标记明显矛盾（如角色突然在两地同时出现） |
| 关闭 | 不审查 |

## 管理指令

| 指令 | 功能 |
|------|------|
| `/审查 check` | 手动触发当前场景质量审查 |
| `/审查 status` | 查看所有未解决的叙事质量标记 |
| `/审查 dismiss [ID]` | 关闭某个标记 |
| `/审查 strictness [级别]` | 调整审查严格度（跟随M15或独立） |
