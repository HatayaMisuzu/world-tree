# 宏系统（模板变量）参考文档

> 对应 SKILL.md §九点五。本文档提供宏系统的完整规范和使用指南。

## 一、什么是宏

宏（Macro）是模板文本中的占位符，语法为 `{{变量名}}`。毛球在读取模板数据时，将宏替换为对应变量当前的实时值。

**解析时机：** 数据读取→注入上下文 的中间环节。
**不解析的场景：** 主人输入的内容、毛球输出的内容。

## 二、语法规则

### 2.1 基本语法

```
{{变量名}}           # 基础变量
{{域:键}}            # 带域名的引用变量
{{域:键:子键}}       # 深层引用
```

### 2.2 规则

| 规则 | 说明 |
|------|------|
| 大小写敏感 | `{{模组名}}` 和 `{{模组名}}` 不同 |
| 命名规范 | 中文变量名使用全中文；英文保持小写 |
| 未定义宏 | 保留原样输出 `{{未知变量}}`，在日志中记录 |
| 循环引用 | 宏替换后的文本不再二次解析 |
| 跨世界书隔离 | 宏只解析当前活跃世界书的数据 |
| 用户输入保护 | 用户手动输入内容中的 `{{}}` 不解析 |

## 三、完整宏清单

### 3.1 模组级

| 宏 | 来源 | 类型 | 替换为 |
|----|------|------|-------|
| `{{模组名}}` | module.json | 字符串 | 当前模组名称 |
| `{{类型}}` | module.json | 字符串 | 当前模组类型（史诗奇幻/校园等） |

### 3.2 场景级（M11）

| 宏 | 来源 | 替换为 |
|----|------|-------|
| `{{场景}}` | M11 runtime | 当前场景名 |
| `{{场景描述}}` | M11 runtime | 当前场景描述文本 |
| `{{场景:类型}}` | M11 | 场景类型（室内/室外/私密/公共） |
| `{{场景:氛围}}` | M11 | 场景氛围描述 |
| `{{在场角色}}` | M11 runtime | 逗号分隔的在场角色名列表 |
| `{{在场角色数}}` | M11 runtime | 在场角色数量 |

### 3.3 时间级（M16）

| 宏 | 来源 | 替换为 |
|----|------|-------|
| `{{时间}}` | M16 timeline | 当前游戏内时间 |
| `{{时间:年月}}` | M16 | 年份+月份 |
| `{{时间:季节}}` | M16 | 当前季节 |
| `{{时间:时段}}` | M16 | 当前时段（清晨/正午/傍晚/深夜） |

### 3.4 角色级（M8）

| 宏 | 来源 | 替换为 |
|----|------|-------|
| `{{角色:名}}` | M8 角色卡 | 角色名+身份简述 |
| `{{角色:名:全称}}` | M8 角色卡 | 角色全名 |
| `{{角色:名:性格}}` | M8 角色卡 | 角色性格关键词 |
| `{{角色:名:身份}}` | M8 角色卡 | 角色身份/职业 |
| `{{角色:名:外貌}}` | M8 角色卡 | 角色外貌简述 |
| `{{角色:名:状态}}` | M8 runtime | 角色当前状态（体力/情绪） |

### 3.5 预设级（M12）

| 宏 | 来源 | 替换为 |
|----|------|-------|
| `{{主角}}` | M12 preset | 当前主角名 |
| `{{主角:驱动}}` | M12 preset | 主角驱动力描述 |
| `{{主角:背景}}` | M12 preset | 主角背景简述 |
| `{{预设}}` | M12 | 当前预设名称 |
| `{{预设:风格}}` | M12 | 预设风格描述 |

### 3.6 世界状态级（M3）

| 宏 | 来源 | 替换为 |
|----|------|-------|
| `{{世界状态:变量名}}` | M3 | 指定状态变量的当前值 |
| `{{世界状态:变量名:历史}}` | M3 changelog | 最近一次变更原因 |
| `{{世界状态:变量名:上次变更}}` | M3 changelog | 最近变更的时间 |

### 3.7 组织级（M4）

| 宏 | 来源 | 替换为 |
|----|------|-------|
| `{{组织:名称}}` | M4 | 组织名称+种族简述 |
| `{{组织:名称:状态}}` | M4 | 组织动态状态 |
| `{{组织:名称:领袖}}` | M4 | 组织当前领袖 |
| `{{组织:名称:规模}}` | M4 | 组织规模描述 |

### 3.8 世界书条目级（M2）

| 宏 | 来源 | 替换为 |

## 四、使用示例

### 4.1 世界书条目中使用宏

假设条目内容写的是：

```
{{组织:翠风冒险团}}的团长{{角色:团长}}正在{{场景}}招募新成员。
当前时间是{{时间}}，距离上次{{世界状态:兽人袭击}}已经过去了三个月。
```

毛球读了这条后会替换成：

```
翠风冒险团（精灵主导·B级冒险团）的团长艾伦·晨风正在王都北门招募新成员。
当前时间是星历782年秋，距离上次兽人袭击（北方边境击退）已经过去了三个月。
```

### 4.2 预设模板中使用宏

```
{{主角}}是一个{{角色:艾琳:性格}}的{{角色:艾琳:身份}}。
她的驱动力是{{主角:驱动}}。
故事发生在{{时间}}的{{场景}}。
```

### 4.3 场景描述中使用宏

```
{{时间}}的{{场景}}，{{在场角色}}都在场。
{{组织:翠风冒险团}}的任务板上贴着一张新的委托。
```

## 五、与SillyTavern宏的对比

| 特性 | SillyTavern | 世界树 |
|------|-------------|--------|
| 语法 | `{{char}}` `{{user}}` | `{{角色:名}}` `{{模组名}}` |
| 用途 | 提示词模板替换 | 叙事模板数据替换 |
| 解析时机 | 发送提示词前 | 读取模板数据时 |
| 使用范围 | 提示词相关字段 | 世界书条目/预设/场景描述 |
| 自定义 | 支持扩展宏 | 域:键格式天然支持扩展 |

## 六、注意事项

1. 宏名不要太长。超过三级（域:键:子键:孙键）的引用不建议使用。
2. 多个宏相邻时建议用空格或标点隔开，避免解析歧义。
3. 宏的值如果包含换行，在模板中会被压缩为空格。
4. 宏系统不改变引擎的隔离规则——所有宏解析都在当前活跃世界书范围内进行。
