## 一、引擎全局指令

### `/引擎 new 模组名`

新建一个故事模组（自动创建独立数据目录并进入世界树模式）。

```
语法: /引擎 new [模组名]
流程: 毛球会依次询问：
  1. 世界类型（或自动判断）
  2. 初始角色
  3. 初始场景
  4. 风格预设
然后创建 D:/world-tree-data/模组名/ 目录结构并启动叙事。

示例:
  /引擎 new 奇幻大陆
  /引擎 new 我的高中
```

注意：一个模组对应一个世界书容器。同一世界观的不同故事线可以在同一世界书下。

### `/引擎 load 模组名`

加载已有模组，从独立数据目录恢复上次进度。

```
语法: /引擎 load [模组名]
流程: 读取 module.json → timeline-tree.json → branches/<active_branch>/runtime.json → branches/<active_branch>/canon_state.json
      → context/snapshot.json → 按需加载各模块数据文件
      → 恢复叙事

示例:
  /引擎 load 奇幻大陆
```

如果有未完成的故事，毛球会提示断点位置并询问是否继续。

### `/引擎 status`

查看引擎当前状态总览。

```
语法: /引擎 status
输出: 当前模式（日常/世界树）、活跃模组名、激活模块列表、
      当前场景、在场角色、叙事进度摘要、隔离状态
```

### `/引擎 exit`

退出世界树模式，回到日常对话。

```
语法: /引擎 exit
自动行为: 保存 context/snapshot.json → 更新脏数据文件
         → 压缩上下文归档 → 清空世界树上下文
```

### `/压缩`

手动触发上下文压缩。

```
语法: /压缩
适用: 当世界树会话轮次较多或感觉上下文拥挤时
行为: 合并冗余描写、去重角色状态、打包历史到 context/history/
```

## 二、存档指令

### `/存档`

保存当前进度。

```
语法: /存档              ← 自动命名（日期_叙事阶段）
      /存档 保存点名称    ← 自定义命名

存储位置: D:/world-tree-data/模组名/branches/<active_branch>/archive/
```

### `/读档`

读取存档。

```
语法: /读档 存档名
      /读档               ← 无参数时列出所有存档供选择

示例:
  /读档 第三章_王都陷落
```

### `/存档列表`

列出所有存档。

```
语法: /存档列表
输出: 存档名称/创建时间/叙事阶段摘要
```

### `/存档 delete`

删除指定存档。

```
语法: /存档 delete 存档名

示例:
  /存档 delete 废弃的初始版本
```

## 三、世界书集指令（全局）

### `/世界书集 list`

列出所有世界书。

```
语法: /世界书集 list
输出: 世界书ID / 名称 / 类型 / 条目数 / 角色数
```

### `/世界书集 create`

创建新世界书。

```
语法: /世界书集 create 名称 [类型]
类型可选: 史诗奇幻 / 科幻未来 / 古风武侠 / 现代都市 / 校园 / 自定义
不指定类型时毛球根据描述自动判断。

示例:
  /世界书集 create 樱丘高中 校园
  /世界书集 create 艾泽拉斯
```

创建后自动切换为活跃世界书。

### `/世界书集 switch`

切换到另一个世界书。

```
语法: /世界书集 switch 名称
注意: 切换后原世界书的角色全部冻结，不会出现在新世界书中。
```

### `/世界书集 now`

查看当前活跃世界书的状态。

```
语法: /世界书集 now
输出: 世界书名称/类型/已激活模块列表/条目数/角色数
```

### `/世界书集 delete`

删除世界书。

```
语法: /世界书集 delete 名称
警告: 不可恢复，确认后执行。
```

### `/世界书集 rename`

重命名世界书。

```
语法: /世界书集 rename 旧名称:新名称

示例:
  /世界书集 rename 奇幻大陆:奥瑞利安大陆
```

## 四、模块管理指令

### `/模块 list`

列出当前世界书已激活的模块。

```
语法: /模块 list
输出: 模块ID / 名称 / 状态（已激活/未激活）
```

### `/模块 on`

激活某模块。

```
语法: /模块 on 模块ID
依赖: 自动激活依赖的父模块（如激活M5自动激活M4）

示例:
  /模块 on M3       ← 激活动态世界状态
  /模块 on M16      ← 激活时间模块
  /模块 on M15a     ← 激活物理规则子模块
```

### `/模块 off`

关闭某模块。

```
语法: /模块 off 模块ID
效果: 相关指令不可用、相关规则不注入上下文、已存在数据保留（恢复后可用）

示例:
  /模块 off M6      ← 关闭关系网络
  /模块 off M17     ← 关闭随机事件
```

### `/模块 preset`

切换到某个世界类型的预设模块组合。

```
语法: /模块 preset 类型
类型: 史诗奇幻 / 科幻未来 / 古风武侠 / 现代都市 / 校园 / 日常对话

示例:
  /模块 preset 校园     ← 切换到校园模块组合
```

### `/模块 all`

激活全部模块。

```
语法: /模块 all
```

### `/模块 minimal`

最小模式（仅核心模块 M1 M2 M8 M11 M12 M13 M15a）。

```
语法: /模块 minimal
```

## 十六、存档分支指令（v12.3 时间树升级）

### `/存档 [名称] --new-branch 分支名`

保存当前进度，并从此节点分出新分支。

```
语法: /存档 存档名 --new-branch 分支名
行为: 创建存档节点 + 复制当前状态到新分支目录 + 切换活跃分支

示例:
  /存档 王都门前 --new-branch 帮助艾琳
  /存档 分叉点 --new-branch 反派线
```

### `/读档 [存档名] --new-branch 分支名`

从历史存档点分出新分支。

```
语法: /读档 存档名 --new-branch 分支名

示例:
  /读档 王都门前 --new-branch 艾琳线
```

### `/分支 list`

列出所有分支及其状态。

```
语法: /分支 list
输出: 分支名 / 当前节点 / 状态(active/paused/completed) / 创建时间
```

### `/分支 switch`

切换到另一个已有分支。

```
语法: /分支 switch 分支名
行为: 保存当前分支状态 → 加载目标分支状态 → 上下文重置

示例:
  /分支 switch main
  /分支 switch 反派线

### `/分支 create`

从指定节点创建新分支。

```
语法: /分支 create 分支名 --from 节点ID

示例:
  /分支 compare main branch-A

语法: /分支 delete 分支名
