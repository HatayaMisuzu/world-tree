# 场景走向预测 `[M18·通用]`

> 来源：NovelGenerator 预测链 → 世界树适配为场景级  
> 目的：防止开放式互动变成无目的闲聊，提供叙事惯性

---

## 核心理念

世界树是交互驱动的——用户输入决定走向。但这不意味着叙事应该完全被动地等待用户每一次指令。

**场景级预测**在每次场景转换时，基于当前局势生成 2-3 个可能的下一走向，作为：
1. **叙事惯性**：让 GM（毛球）有一个方向参考，叙事保持推力
2. **可选引导**：用户可以看到方向提示，也可自由选择其他方向

---

## 触发时机

以下情况触发预测生成：

| 触发条件 | 频率 |
|---------|------|
| 场景转换（/场景 move/new） | 每次 |
| 重大剧情节点（关键时刻完成） | 每次 |
| 章节/幕结束 | 每次 |
| 用户主动请求 | `/预测` |

**不触发**：
- 场景内日常对话
- 琐碎动作（喝水/开门/打招呼）
- 短指令（继续/嗯/然后呢）

---

## 预测格式

```
📡 叙事走向预测：

A. [方向一名称]
   [1-2句话描述] — 这个方向的风险和收益

B. [方向二名称]
   [1-2句话描述] — 这个方向的风险和收益

C. [方向三名称]（可选）
   [1-2句话描述] — 这个方向的风险和收益

💡 你也可以选择其他方向。
```

---

## 预测生成原则

1. **基于已有叙事逻辑**：不是随机，是基于当前局势、角色动机、势力动态的合理推断
2. **保持开放性**：不给「唯一正确」的方向，至少 2 个选项
3. **标注风险收益**：每个方向简要说明可能的后果
4. **不强制**：预测只是参考，用户可以忽略
5. **不预测过远**：只预测「下一个场景或事件」，不跳到整个故事的结局

---

## 预测示例

```
场景：冒险者公会大厅。团长刚发布了一个危险任务——调查北方废弃矿洞的失踪事件。

📡 叙事走向预测：

A. 接下任务，立即出发
   直奔北方矿洞，在天黑前赶到。 —— 推进主线，但准备不足可能遭遇危险

B. 先在公会搜集情报
   向酒馆里的老冒险者打听矿洞的传闻。—— 获得关键信息，但可能错过最佳时机

C. 拒绝任务，追查自己的身世
   刚才团长提到矿洞里有「古老的遗迹」——那和你的身世有关吗？ —— 深入个人线，但任务可能被他人接走

💡 你也可以选择其他方向。
```

## 与 M17 随机事件的区别

| | M17 随机事件 | M18 场景预测 |
|---|---|---|
| 触发方式 | 概率触发 | 确定性触发（场景转换时） |
| 内容 | 随机事件提案 | 基于逻辑的走向推断 |
| 用户感知 | "发生了什么" | "接下来可能发生什么" |
| 目的 | 增加意外性 | 保持叙事方向 |

---

## 存储

预测不持久化存储——只在场景转换时生成，注入当前上下文，场景结束后丢弃。

---

## 管理指令

| 指令 | 功能 |
|------|------|
| `/预测` | 生成当前局势的走向预测 |
| `/预测 off` | 关闭自动预测（场景转换时不再生成） |
| `/预测 on` | 开启自动预测 |
| `/预测 style 详细/简略` | 切换预测风格 |
