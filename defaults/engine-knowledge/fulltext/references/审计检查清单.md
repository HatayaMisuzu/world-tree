# 世界树 Skill 全流程审计检查清单

> 使用时机：修改 world-tree SKILL.md 或任一 reference 后，变更 ≥2 处时自动触发。
> 也触发于版本号跳跃（如 shared → writer 同步时发现版本不一致）。
> 事故复盘规则：2026-06-02 v12.19 漂移修复后，任何 world-tree 改动必须能检出真相源、旧入口、writer 副本、`D:/world-tree-data/_engine` 模板漂移。

- [ ] 历史归档文档必须在文件顶部和导航中同时标注“历史归档/不作为当前入口”

## 常见坑

| 坑 | 检测方式 | 修复 |
|----|---------|------|
| 步骤编号错乱 | 扫描 §3.2 数字序列 | 重编号 1→N |
| Unix/Windows 路径混用 | 搜索 `~/.hermes/` | 替换为实际 OS 路径 |
| 幽灵指令 | 交叉 grep SOUL.md vs SKILL.md | 删除或追加实现 |
| 版本漂移 | 对比 truth-source.yaml vs SKILL.md metadata | 更新 lagging 端 |
| 真相源文件名旧但内容新 | 检查 `governance.authority_version` 和 filename_note | 保留文件名时必须声明内容权威版本 |
| 旧入口复活 | 搜索 `自主输出小说模式`、`novel-engine`、`独立子技能` | 标历史归档或纳入 M-子模块，不能作为当前入口 |
| writer 副本漂移 | 对比 shared 与 profiles/writer 关键文件 | 同步后双 validator |
| `_engine` 模板漂移 | 读 `D:/world-tree-data/_engine/*.json` 的 version 和模式映射 | 同步版本、角色卡模式、agent profile |
| writer 默认人格错误 | 读 config.yaml `personality:` 行 | 改为 `writer` |
| writer 人格描述过时 | 读 `agent.personalities.writer` | 同步 SOUL.md 摘要 |

- [ ] 叙事一致性雷达不向 user 显示面板，阻断级冲突不写入 confirmed。

- [ ] 素材吸收评分器将外部素材先隔离评分，不直接摄入世界事实。
