# 世界书生长树

> world-tree v12.19 创新升级模块。定位：把剧情中自然出现的新设定转成可审核的世界书候选枝芽。

## 核心概念

世界书不是一次性写完的静态文件，而是在叙事中自然生长。新概念先进入 `proposed`，通过用户确认或主 agent 审核后再变成正式世界书条目。

## 候选状态

| 状态 | 含义 | 可写入位置 |
|------|------|------------|
| seed | 叙事中第一次出现的新概念 | `canon_state.proposed` |
| sprout | 已出现 2 次以上，具备条目价值 | `growth_tree.candidates` |
| branch | 用户确认或主 agent 审核通过 | `shared/worldbook.json` |
| pruned | 被否决或污染风险过高 | `growth_tree.pruned` |

## 建议格式

```json
{
  "id": "sprout-001",
  "name": "银雾钟楼",
  "status": "sprout",
  "source_turns": [18, 24],
  "candidate_keys": ["银雾钟楼", "雾钟"],
  "candidate_content": "王都北侧只在雾夜响起的旧钟楼。",
  "risk": "低",
  "needs_user_confirm": true
}
```

## 写入规则

- seed/sprout 不得直接进入 confirmed。
- 重要世界规则、组织、角色秘密必须等待用户确认。
- 小地点、小道具、重复出现的场景锚点可由主 agent 提议合并。
- 每次生长必须保留来源轮次，避免凭空补设定。
