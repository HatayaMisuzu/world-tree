| 条目长度 | 建议200-500字，过长拆多条 | 无明确长度规范 | 缺乏质量控制 | 已吸收：将长度规范写入世界书制作检查清单 |
| 构建策略 | 小步迭代：先5-10条核心→测试→再扩 | 无引导，用户直接写 | 新手上手成本高 | 已吸收：在/引擎 new的引导和快速上手中加入"先写5-10条核心条目"建议 |

## 已实施的5项吸收改进

### 1. 蓝灯常驻条目
条目增加 `模式: 触发 | 常驻 | 禁用` 字段。常驻条目每轮自动注入，不依赖关键词匹配。用于世界基本法则、核心文化基调、当前主线目标极简描述。限制3条以内、100字以内。

### 2. 三层注入位置
条目增加 `注入层级: base | context | instant` 字段。
- base（基底）：叙事开始时注入，占最高注意力
- context（情境）：与关键词/场景匹配时注入，现有行为
- instant（即时）：靠近用户输入前注入，占近因效应

### 3. 优先级排序
条目增加 `优先级: 1-100` 整数。同层多个条目命中时按优先级降序注入。1为最高，100为最低。

### 4. 扫描深度
条目增加 `扫描深度: 近距 | 中距 | 远程 | 全局`。高频角色/地点用小范围避免误触，罕见事件用全局。默认中距（5条）。

### 5. 条目长度规范 + 小步迭代引导
- 世界书制作检查清单中加入200-500字长度规范
- 制作流程中加入小步迭代建议（先5-10条核心，跑轮次再扩）

## 世界树现有优势（SillyTavern没有的）

| 功能 | SillyTavern | 世界树引擎 |
|------|------------|-----------|
| 多世界书隔离 | 只能挂一个世界书，切换需手动 | M1模块：多世界书容器+守门人+串台阻断 |
| 动态世界状态 | 无 | M3模块：状态变量/事件驱动/影响传导 |
| 组织实体与关系网络 | 无 | M4-M7/M10：组织层级/20+种关系/关键人物 |
| 时间树与分支 | 无 | M16模块：分支/存档分叉/时间线管理 |
| 角色认知层 | 无 | M9模块：信息不对称/认知群组/双维度认知 |
| 随机事件体系 | 无 | M17模块：三级事件+权重自适应 |
| 世界规则审核 | 无 | M15模块：三阶段审核/物理/社会/魔法规则 |
| 子agent协作 | 无 | 多agent并行调度 |
| 素材学习系统 | 无 | 分类归档/自动提取 |
| 独立数据目录隔离 | 无 | D:/world-tree-data 完全独立于Hermes体系 |
