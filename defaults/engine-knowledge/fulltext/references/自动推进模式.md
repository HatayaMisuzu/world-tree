# 自动推进模式

> world-tree v12.19 · 原"自主输出小说模式"已简化为此模式
> 旧完整AI小说写作流水线已归档；若恢复书籍级生产，必须作为 world-tree 的 M-子模块纳入，不另建独立 skill。

## 定位

用户不想互动、只想看剧情推进时，AI主动向前推进叙事。不替代互动模式，而是在互动间隙提供"自动播放"能力。

## 触发

| 指令 | 行为 |
|------|------|
| `/推进` | AI主动推进1-3段叙事（含场景/对话/冲突推进），停在剧情分支点 |
| `/推进 停止` | 停止自动推进，回到互动模式等待用户输入 |

## 推进规则

1. **不替用户做决定** — 推进到剧情分支点（角色面临选择/新信息出现/场景转换前）时自动停下
2. **保持角色一致性** — 所有角色行为从已建立的性格基线推导，不OOC
3. **世界状态同步** — 推进产生的世界状态变更正常写入 `branches/<active_branch>/runtime.json` + `branches/<active_branch>/canon_state.json`
4. **不引入新主线** — 自动推进不创造新的核心冲突，只在现有剧情线上延伸
5. **可中断** — 用户随时可以用自然语言打断推进，夺回控制权

## 与互动模式的关系

```
互动模式（主）                自动推进（辅）
    │                            │
    │  用户说"/推进"              │
    ├──────────────────────────→│
    │                            │ 推进1-3段叙事
    │                            │ 遇到分支点→自动停
    │  自动停下，交还控制权       │
    │←──────────────────────────┤
    │  用户继续互动               │
    │                            │
```

## 与旧完整小说流水线的区别

| | 自动推进 | 旧完整小说流水线 |
|------|---------|-------------|
| 目标 | 互动间隙的剧情推进 | 完整的书籍级小说生产 |
| 输出量 | 1-3段叙事 | 整章3000-5000字 |
| 规划 | 无，沿现有剧情线 | 有，章节大纲+情绪曲线+线索分布 |
| 审阅 | 无 | 三阶段审查+外部审阅闭环 |
| 适用 | "不想操作只想看" | "我要写一本完整小说" |
