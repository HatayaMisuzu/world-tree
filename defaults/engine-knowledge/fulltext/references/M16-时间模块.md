## 〇.九、时间模块（World Timeline）`[M16·中大型世界]`

时间模块分为两个独立子模块，分别追踪不同尺度的时间：

```
M16a 个人/组织时间           M16b 世界时间轴
─────────────────────       ─────────────────────
追踪谁的钟？                追踪世界的钟
角色经历/组织状态变化       全球事件排序/历法推进
对"现在"的感知              "现在"的客观定义
对"未来"的驱动力            事件因果链和传播延迟
「艾琳走了三天了」          「帝国历1274年秋·第45日」
「元帅磨刀霍霍准备政变」     「皇帝驾崩后第三天」
```

**两层的关系：参差有别，整齐有序。**
- M16b 是世界公用的钟——所有人都共享同一个"现在"
- M16a 是每个人/组织的私钟——不同人对同一时刻的感知和驱动力完全不同
- 同一时刻（M16b），元帅在谋划政变、艾琳在逃亡路上哭泣、皇太子在加冕——三个人的M16a截然不同

### 0.9.1 核心理念

时间不是一条均匀的线——它是有质感的。对有的人它是倒计时的炸弹，对有的人它是慢慢愈合的伤口。

**三大时间法则：**

**法则一：时间不可逆。** 过去的事件不能撤销。国王死了就是死了。铁门关丢了就是丢了。时间只能向前。

**法则二：时间对每个人不等速。** 对等待元帅回王都的元老院，每一天都像一年。对赶路的艾琳，三天一眨眼就过了。但客观时间（M16b）对所有人在同一时刻是相同的。

**法则三：未来不是已知的。** 角色不能预知未来（除非有规则内的预知能力且有代价）。叙事不能随意跳到未来又倒回来改变。计划≠预言≠已发生。

### 0.9.4a 组织时间状态 `[M16a]`

组织不是静止的——它在时间中演化。每个组织有自己的「时间体感」：

```yaml
组织时间状态模板:
  组织: "元老院"
  
  组织时钟:
    当前关注: "皇帝驾崩后第1天"            # 组织对时间的感知锚点
    紧迫度: 10/10（继承危机刻不容缓）
    节奏: "加速"                            # 慢（常规运作）/正常/加速（危机）/停顿（瘫痪）
    
  内部驱动力:
    保守派: "尽快立皇太子登基，稳定局面"     # 这个群体的时间方向→
    主战派: "拖住元帅别让他回来夺权"         # 这个群体想延迟某事
    改革派: "趁乱推动制度改革"              # 这个群体想利用时间窗口
    
  "现在"的张力:                             # 不同群体对同一时刻的不同解读
    对保守派: "现在是保护帝统的关键时刻"
    对主战派: "现在是军方可能失控的危险时刻"
    对改革派: "现在是千载难逢的改革窗口"
    
  预期行为（按时间）:
    今天: 紧急会议，讨论继位
    3天后: 加冕仪式
    1周后: 如果元帅还没回来，权力结构初步稳定
    1月后: 如果元帅回来了，可能爆发政变
```

### 0.9.4b 经历与状态驱动力 `[M16a]`

个人和组织的「过去经历」在时间中积累，形成对「现在」的状态和对「未来」的推力：

```
经历的三个时间效应：

①伤疤效应 —— 过去的事件在现在留下印记
  艾琳经历了「铁门关失守」
  → 现在（秋·第45日）：听到任何关于战争的消息都会僵住
  → 未来：对北方的任何军事行动有强烈情绪反应

②惯性效应 —— 正在做的事有继续做下去的倾向
  元帅卡修斯正在「赶回王都」
  → 现在：在路上的每一天，他的军队离王都更近
  → 除非有重大事件阻止，否则3天后他会到达
  → 这个惯性本身就是一条悬在元老院头顶的叙事线

③窗口效应 —— 某些机会只在特定时间存在
  改革派：皇帝驾崩后的权力真空期是改革的唯一窗口
  → 窗口期：约1-2周（元帅回来之前）
  → 如果错过这个窗口，可能再等十年
  → 这个时间压力驱动改革派铤而走险
```

**驱动力的叙事体现：**

毛球在场景中自动注入角色/组织的「时间驱动力」：

```
场景：元老院会议
客观时间（M16b）：秋·第45日·深夜
个人时间（M16a）：
  - 保守派议员：「每一刻都像一年」（紧迫）
  - 改革派议员：「时间不够了」（窗口焦虑）
  - 中立议员：「先等等看元帅那边怎么说」（观望→时间方向不同）

→ 毛球的叙事中自然体现这三种不同的时间体感
→ 同一次会议，三种人对「现在」的感知完全不同
```

### 0.9.5 时间一致性检查 `[M16b]`

每次场景切换或人物登场时，毛球自动执行：

```
时间一致性检查清单：
□ 当前时间是什么？上一个场景是什么时间？间隔多久？
□ 在这段时间内，世界上发生了什么事件？
□ 当前场景中出现的角色，在这个时间点应该在哪里？
□ 有没有角色在这个时间点已经死了？
□ 有没有角色在这个时间点应该已经离开了？
□ 角色知道哪些在这段时间内发生的事件？（认知传递延迟）
□ 角色的状态有没有随时间自然变化？（伤口愈合、年龄增长等）
```

**典型违规示例：**

```
❌ 秋·第45日·黄昏
   「皇帝奥瑞斯三世坐在王座上接见使者」
   → 时间一致性违规：皇帝在秋·第44日深夜已驾崩！
   → 毛球纠正：「王座上坐的是年幼的奥瑞斯四世，旁边站着摄政太后。」

❌ 秋·第45日·黄昏
   「艾琳出现在王都的冒险者公会」
   → 时间一致性违规：艾琳在秋·第44日清晨离开王都，步行一天不可能往返600里。
   → 毛球阻止或提议：「艾琳现在应该在王都以南约150里的地方。需要她回来需要叙事理由——传送？骑马日夜兼程？还是暂时不写她的戏？」
```

### 0.9.6 灵活淡化机制

时间模块不等于「每走一步都要计时」。主人说的「前往某地中途需要时间但可以暂时忽略」——这正是灵活淡化的核心。

**三种时间处理模式：**

| 模式 | 使用场景 | 行为 |
|------|---------|------|
| **精确追踪** | 关键剧情、多线同步 | 精确计时，严格一致性检查 |
| **灵活淡化** | 旅行、等待、日常过渡 | 不逐日描写，但记录时间流逝 |
| **时间跳跃** | 明确的时间跨段 | 标记跳跃幅度，执行跨越事件 |

**灵活淡化的具体规则：**

```
旅行场景：
  主人: "我前往北方的矮人王国"
  毛球: 「路途约需两周。在这段时间里——」
  → 不逐日描写风景和住宿（除非主人要求）
  → 但时间轴静默推进了两周
  → 两周内世界上发生的事件照常执行
  → 抵达后，毛球告知：「两周后你抵达矮人王国。这段时间里，
      皇帝驾崩的消息已经传遍大陆，矮人王国正在举行哀悼仪式。」

等待场景：
  主人: "我在旅店等三天"
  毛球: 「三天里——」
  → 不逐小时描写
  → 但三天后，角色的认知可能更新（听说了某些消息）
  → 时间轴上记录这三天

日常过渡：
  主人: "放学了"
  毛球（校园）: 
  → 不纠结精确的分钟数
  → 但保持场景连续性（今天放学→明天上学，不能出现"今天已经放学但角色还在上课"）
```

**时间跳过指令：**

```
主人: /时间 skip 3天
毛球: 「时间推进3天。现在是秋·第48日·清晨。
      这段时间内发生：
      - 皇帝驾崩的消息传到了300里外
      - 艾琳现在位于王都以南约450里处（如果继续向南走）
      - 元帅卡修斯已收到消息，正在回王都的路上
      你要关注哪一条？」
```

### 0.9.7 未来管控机制 `[M16b]`

**未来的概念在叙事中是危险的。** 如果角色可以随意预知未来、叙事可以跳来跳去、预言永远准确——故事就失去了悬念和意义。

时间模块对「未来」实施严格管控：

**A. 未来的三种形态（确定性递减）**

```
确定性      名称          含义                    示例
──────────────────────────────────────────────────────
100%       已锚定事件     因果链已锁死，必然发生    皇帝驾崩（已发生，不可撤销）
70-90%     强预期事件     因果链高度指向，极可能    元帅3天后抵达王都（除非被拦截）
30-70%     弱预期事件     有可能性但变数大           改革派可能发动政变
<30%       推测           纯属角色/组织的猜测        西境侯国可能叛变
0%         不可知         叙事不应触及              下一章谁死不能提前说
```

**B. 预言与预知的严格限制**

预言在叙事中是一种剧毒——它要么毁了悬念，要么让自己显得可笑。因此：

```
规则：预言永远不精确。

❌ 不允许: "先知预言了皇帝将在秋·第44日驾崩，结果分毫不差"
   → 这毁了所有悬念。读者不会在意过程。

✅ 允许: "先知在火焰中看到了王座上的血——但看不清是谁的。"
   → 模糊、多义、可以被不同人解读。制造悬念而不是摧毁它。
   
✅ 允许: 角色声称自己有预知能力，预测了三件事，结果两件对了一件错了。
   → 预知是角色的主观能力，不是叙事的确凿事实。
```

**角色预知能力的限制（如有）：**

如果世界书规则允许预知类超能力（M15b），必须满足：
- 只能看到模糊片段（"火焰中的影子"而非清晰画面）
- 未来可以改变（预知到的场景可能因为角色的行动而不发生）
- 预知有代价（每次消耗寿命/理智/资源）
- 预知的内容可以被不同解读（十个看到同一幻象的人得出十个结论）

**C. 时间跳跃的严格管控**

叙事不能随意跳到未来又倒回来：

```
允许的时间操作:
  /时间 skip 3天   → 向前推进，不可逆，不可撤销
  毛球叙述跨段时间   → "三天后，你抵达了矮人王国"
  角色回忆过去       → "她想起三天前离开王都的那个清晨"

禁止的时间操作:
  跳到未来查看结果 → 再倒回来 → "原来我三天后会中埋伏，那我先绕路"
  倒流时间改变过去 → "其实皇帝没死，我回到前天救了"
  叙事者提前泄露未来 → "（剧透：艾琳会在三天后见到她父亲最后一面）"
```

上帝模式 `{...}` 可以绕过时间跳跃限制，但毛球会明确标记：
> `{时间倒流到秋·第44日}` → 🜲 上帝模式：时间已倒流。所有事件和状态回滚。这是超叙事操作，请谨慎。

**D. "命运"的处理**

命运是最危险的叙事概念——它让角色失去能动性。

```
规则：命运不存在于叙事引擎中。

- 没有"命中注定"的事件
- 没有"不可逃避"的结局
- 预言可以被打破（如果角色付出足够的代价）
- 角色始终有选择的权利（即使选择很糟糕）

如果主人想在故事中引入命运主题：
  → 把它写成一个角色/组织的"信念"，而不是叙事的"事实"
  → "帝国教会相信皇帝家族受神命统治" ← 这是信念
  → "皇帝家族真的受神命统治" ← 毛球拒绝接受这种确定性
```

### 0.9.8 双层时间与各模块的联动 `[M16a+M16b]`

```
M16a 个人/组织时间
    │
    ├──→ M3 世界状态: 角色的经历积累到一定程度 → 触发状态变更
    ├──→ M4 组织实体: 组织的"窗口效应"和"惯性效应"驱动组织行为
    ├──→ M7 关键人物: 每个人物的时间体感不同（紧迫/观望/等待）
    └──→ M9 角色认知: 经历塑造认知（伤疤效应）；信息传播延迟

M16b 世界时间轴
    │
    ├──→ M3 世界状态: 事件按时间线顺序激活状态变更
    ├──→ M15 世界规则: 时间不可逆（三大法则）
    ├──→ M15b 魔法规则: 预知能力严格限制
    ├──→ M11 场景管理: 时间一致性检查
    └──→ 事件时间线: 因果链追踪，"已锚定"vs"预期"vs"推测"
```

**关键联动：信息传播延迟**

信息在世界上传播需要时间——这是时间模块与角色认知层（M9）的最重要交集：

```
事件: 皇帝在秋·第44日深夜驾崩
─────────────────────────────────────
王都内: 秋·第45日黎明 → 所有在王都的角色认知更新
100里内: 秋·第46日 → 附近的城镇开始听说
300里内: 秋·第48日 → 艾琳在旅途中听到传闻（她认知更新）
前线: 秋·第50日 → 元帅卡修斯收到正式军报
偏远村庄: 秋·第55日+ → 一些地方还不知道皇帝已经换了
```

毛球在叙事中体现这种延迟：
> 艾琳在旅店听到隔壁桌的商人低声说"听说皇帝驾崩了"——她的手指僵住了。
> 而与此同时，王都的皇太子已经在加冕仪式上坐了两个时辰。

| `/时间 now` | 查看当前世界书的完整时间状态 | |

### 0.9.10 时间树分支（v12.3 升级）

时间模块从线性时间升级为时间树——同一时间点可以分叉出不同世界线。
完整设计见 [存档分支与时间树系统.md](存档分支与时间树系统.md)。

**核心变更：**
- M16 不再只管理单一线性时间，而是管理一棵时间树
- 每个分支有独立的 timeline.json（分支内事件线）
- timeline-tree.json 记录跨分支的时间关系
- 分叉点前的时间线共享，分叉后各自独立
- 时间一致性检查仅在同一分支内进行

**新增指令：**
| 指令 | 功能 |
|------|------|
| `/时间树 show` | 可视化时间树 |
| `/时间树 node ID` | 查看节点详情 |
| `/分支 list` | 列出所有分支 |
| `/分支 switch 名` | 切换活跃分支 |
| `/分支 create 名 --from ID` | 从节点创建分叉 |
| `/分支 compare A B` | 对比两分支差异 |
