# M19 角色卡驱动模式 — 最小测试样例

> 用途：维护者在修改角色卡导入、控制台识别、存档隔离或 M19 规则后，必须用这 4 个样例复核识别结果。
> 当前版本：world-tree v12.19

## 样例 A：世界书 JSON

预期：识别为【世界书】，进入 worldbook 模式，存档前缀 `[W]`。

```json
{
  "spec": "worldtree-worldbook",
  "name": "测试世界书",
  "entries": [
    {
      "keys": ["王都", "城门"],
      "content": "王都城门由银甲卫队守备。",
      "priority": 80
    }
  ]
}
```

## 样例 B：ST 角色卡 JSON

预期：识别为【角色卡】，进入 character_card/M19 模式，存档前缀 `[C]`，不得注入 M2/M18。

```json
{
  "name": "艾琳",
  "description": "银发咖啡店店员，外冷内热。",
  "personality": "谨慎、嘴硬、会记住 user 的小习惯。",
  "scenario": "雨夜的街角咖啡店。",
  "first_mes": "……这么大的雨，你还真来了。坐吧，别把地板弄得都是水。",
  "mes_example": "<START>\n{{char}}: ……我才不是特意给你留的位置。"
}
```

## 样例 C：双重命中 JSON

预期：同时标注【世界书】和【角色卡】，必须提示用户选择加载类型，不得自动猜测。

```json
{
  "name": "艾琳与王都",
  "first_mes": "你终于来了。",
  "personality": "警惕但温柔",
  "entries": [
    {
      "keys": ["王都"],
      "content": "王都是故事主舞台。",
      "priority": 50
    }
  ]
}
```

## 样例 D：未知 JSON

预期：提示“无法识别格式”，不创建模组，不写 runtime/canon_state。

```json
{
  "title": "未完成资料",
  "notes": ["只有笔记，没有世界书 entries，也没有角色卡 first_mes/personality。"]
}
```

## 维护检查

- 双重命中只能提示选择，不能默认按世界书或角色卡加载。
- 角色卡模式不得显示好感度、关系数值面板、DM 提案或 M18 预测。
- 跨模式读档必须阻断：`[W]` 不能读入角色卡模式，`[C]` 不能读入世界书模式。
- 修改控制台识别逻辑后，运行 `scripts/world_tree_contract_check.py` 并手动用以上 4 个样例复核。
