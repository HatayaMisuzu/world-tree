# 分支差异叙事摘要

> world-tree v12.19 创新升级模块。定位：把 JSON diff 翻译成写作者能理解的“命运差异”。

## 输出目标

分支对比不只比较字段变化，还要回答：这条分支和主线相比，故事命运哪里变了？

## 三层摘要

| 层级 | 问题 | 示例 |
|------|------|------|
| 事实差异 | 哪些 confirmed 不同 | “卡修斯在 A 线存活，B 线失踪” |
| 关系差异 | 角色关系发生什么变化 | “艾琳在 A 线信任 user，在 B 线保持距离” |
| 命运差异 | 长期叙事走向不同 | “A 线走向结盟，B 线走向孤立潜入” |

## 建议输出

```json
{
  "base_branch": "main",
  "compare_branch": "艾琳线",
  "fact_delta": [],
  "relationship_delta": [],
  "fate_delta": [
    "这条分支把故事从政治博弈推向角色信任线。"
  ],
  "recommended_use": "适合继续写角色关系推进。"
}
```

## 规则

- 不用字段名堆砌报告，必须给出叙事意义。
- 不确定的差异标为“推测”，不得写入 confirmed。
- 对比结果默认只读，不修改任何分支。
