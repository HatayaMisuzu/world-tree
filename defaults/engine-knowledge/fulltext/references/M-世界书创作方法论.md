# 世界书创作方法论 v2.0

> 基于 M-世界书创作方法论 v1.0 + 艾尔德兰设定分析 + 共感娃娃复盘 + 万界复盘  
> 定位：世界树引擎 M-创作模块的核心流程文件。创作世界书时手动激活 `/模块 on M-创作`  
> 适用者：毛球（default profile）创作世界书时使用。writer profile 运行游戏时默认关闭

---

## 一、核心理念

| # | 理念 | 含义 |
|---|------|------|
| **P1** | **创作以主人为中心** | 素材+主人期望决定世界书形态，不预设「好感度」「抽卡」等体系 |
| **P2** | **叙事密度大于条目数量** | 30条高质量 > 100条注水。每一条目都能让人读完想了解更多 |
| **P3** | **留白与呈现同等重要** | 不把所有事写完。70%信息+30%悬念——读者自己拼凑的部分最有感染力 |
| **P4** | **可审查不可跳** | 每版必须列关键信息供主人过目，不等主人催 |
| **P5** | **世界书是写给AI看的** | 内容结构要符合AI的信息调用习惯和提示词工程逻辑。不是写"好文章"，是写"AI能高效读懂的指令"。简洁>优美，结构化>文学化，触发精准>描述全面 |

**P5 的具体含义：**

```
❌ 人类读物思维：「清晨的阳光斜斜地照在圣城洁白的石墙上，钟声在远处的神殿塔顶回荡…」
✅ AI指令思维：「圣城奥德希安：大陆中央，命运神殿所在地。特征：城墙白色，钟声每日清晨响起。」

世界书不是小说，不是设定集——是AI在生成回复时参考的信息库。
每条条目都是一条「如果触发条件满足，就把这段信息注入当前语境」的指令。
```

## 二、创作七步法

### 第1步：读素材（通读+标记）

一次读完所有可用素材，不做碎片化提取。

**标记三类信息：**
- **⌘ 概念点**：地名/角色名/势力名/规则/机制 → 必须转化为条目
- **★ 叙事密度点**：有情感张力的细节（一句话一个故事的那种）
- **? 留白空间**：素材中没说清/暗示但没展开的内容 → 未来的悬念钩子

**同时识别：**
- **小物件候选**：素材中有无「小而重的物体」可作角色的锚点物
- 素材的缺失维度：哪些方面需要向主人补充提问
- **角色四锚点框架**：如果素材中人物的物件/习惯动作/伤口/秘密不够，提问阶段重点追问

---

### 第2步：提问了解期望

**必问维度：**

```text
用途风格：  冒险 / 角色扮演 / 攻略 / 悬念 / 史诗叙事 / 日常 / 其他？
规模预期：  小场景（1-2个地点） / 中规模（5-10个地点） / 大世界观（10+地点）
游戏机制：  是否需要好感度 / 货币 / 抽卡 / 任务系统 / 成就系统？
目标体验：  剧情沉浸 / 自由度探索 / 角色养成 / 收集完成 / 情感代入？
结局偏好：  是否需要在世界书中预设多重结局分支？
突出设定：  主人心中有特别想突出的设定点或氛围吗？
约束条件：  有无特别限制？

角色锚点： 素材中角色的「四锚点」够清晰吗？核心角色有让人想留下的特质吗？
```

**追问规则：** 模糊→缩小范围再问 / 明确→直接推进 / 你看着办→合理推断后仔细审查

---

### 第3步：锚点设计

写条目之前先确定世界的**情感骨架**。

**3.1 确定主角锚点**

| 锚点类型 | 产出物 |
|---------|-------|
| 核心物件 | 一个具象的、可被看见/触摸的物体，承载最重的情感 |
| 开篇仪式 | 一个不可跳过的仪式感时刻，玩家进入世界的第一印象 |
| 情感温度 | 一个词描述核心情绪（如「隐忍的悲伤」「燃尽的温暖」） |

**3.2 关键角色四锚点**

```text
角色名：[名字]
  ▸ 锚点物件：随身小东西——不说用途，只描述
  ▸ 习惯动作：反复出现的微动作
  ▸ 一道伤口：身体或心灵旧伤——不说来由，只呈现
  ▸ 一个不说的事：所有人回避的话题或秘密
```

**3.3 结局架构预览**

```text
多重结局需求？ 单章级 / 全篇级 / 两者兼有
结局类型偏好： 英雄（全员活但代价） / 破灭（有人死但有意义） / 混沌（选择大于结果）
```

---

### 第4步：选原则

从 **M-创作工具箱.md** 的十大原则中选择适用的，记录选/不选的原因。

### 第5步：出初稿

按选定原则和必检维度生成世界书 JSON 条目。

**产出物：** worldbook.json（JSON格式，符合worldbook-json-schema.md规范）

**写作时自检（双重视角）：**

| 检查项 | 自问 |
|--------|------|
| **人类视角：叙事密度** | 去掉名字还剩什么？什么都没剩→太干了 |
| **人类视角：留白检查** | 删掉30%的说明会不会更有力？ |
| **人类视角：锚点核查** | 预定的四锚点在每个输出里都有体现吗？ |
| **AI视角：条理结构** | AI读到这段能秒懂核心信息吗？ | 
| **AI视角：触发精准** | 触发词能覆盖这条目被需要的场景吗？ |
| **AI视角：冗余检查** | 有没有可以删掉又不丢信息的修饰语？ |

**条目类型内容比例参考：**

| 类型 | 比例 |
|------|------|
| 规则/系统类 | 70%机制 + 30%叙事化 |
| 角色类 | 45%设定 + 55%细节锚点 |
| 地点类 | 50%描述 + 30%氛围 + 20%钩子 |
| 物品类 | 30%功能 + 70%来历/碎碎念 |
| 事件类 | 40%经过 + 30%结果 + 30%未解决的尾巴 |

---

### 第6步：列关键信息供审查

**不可跳过。** 审查信息模板：

```text
世界书名称：[名称]
条目总数：[N] 条
类型分布：[规则/地点/人物/事件/物品/组织 N条]

核心设定一句话：[一句话说清世界核心]

角色锚点一览：
  [角色名] → [锚点物件] + [不说的事]
  ...

重点条目预览（5-10条）：
1. [触发词] → [内容摘要50字]
...

留白空间（3-5处）：
  [条目A] 中留下了"XX"未展开
  ...

选用的创作原则：[清单]
未选用的原则及理由：[清单]
```

---

### 第7步：按需修改

| 类型 | 操作 |
|------|------|
| 微调 | 直接改，更新关键信息 |
| 增补 | 补充条目，更新关键信息+新增预览 |
| 重组 | 出一版新关键信息+修改对照表 |
| 重来 | 回到第2步重新提问 |

## 三、必检维度（素材提取检查表）

### A. 世界维度

A-1 核心设定 / A-2 历史 / A-3 政治体系 / A-4 种族 / A-5 宗教 / A-6 魔法/超自然 / A-7 经济 / A-8 文化 / A-9 组织 / A-10 角色（用四锚点框架） / A-11 地标 / A-12 外部威胁

### B. 角色维度

B-1 角色档案 / B-2 四锚点（每个角色至少2项） / B-3 角色关系 / B-4 成长弧线 / B-5 互动方式 / B-6 温度感

### C. 机制维度

C-1 核心驱动力 / C-2 资源体系（每条规则是否在讲故事） / C-3 成长曲线（每级是否有叙事代价） / C-4 规则边界（边界本身是否制造命运感） / C-5 结局状态（多重结局如何级联）

## 四、常见陷阱

| 陷阱 | 预防 |
|------|------|
| 预设偏见 | 先问主人期望，不默认好感度/抽卡 |
| 过度规则化 | 每写一条规则自问：这条本身在讲故事吗？ |
| 模板依赖 | 不同类型条目用不同内容比例 |
| 跳过审查 | 审查是流程硬步骤 |
| 叙事干瘪 | 去掉名字还剩什么？ |
| 留白缺失 | 删掉30%看是否更有力 |
| 锚点不够 | 每个关键角色至少2个四锚点元素 |
| 开局平淡 | 参考工具箱的开局设计卡 |
| **人类思维写AI条目** | **写成了小说段落而不是AI指令——每写一段问自己「AI读这个能秒懂吗」** |
