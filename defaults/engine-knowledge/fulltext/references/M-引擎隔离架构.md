# M-引擎隔离架构 — 引擎级上下文隔离详细规范

> 版本: v12.9
> 层级: 引擎基础架构（高于所有模块）  
> 状态: 始终生效，不可关闭  
> 数据根目录默认值: `D:/world-tree-data`（可由宿主环境配置项覆盖）
> 完整目录结构见 SKILL.md §2。本文档展示架构层面的原则性结构。

---

## 一、设计原则

世界树引擎生成的叙事内容与日常对话之间存在本质区别：

| | 日常对话 | 世界树叙事 |
|------|---------|-----------|
| 内容性质 | 真实交互 | 虚构创作 |
| 记忆价值 | 事实/偏好/环境 | 无（故事不是事实） |
| 上下文需求 | 需要跨会话记忆 | 仅当前模组内需要 |
| 数据生命周期 | 长期 | 模组结束后归档 |

如果混在一起，会导致：
- memory-tree 被大量虚构叙事污染
- 日常对话上下文窗口被世界树内容挤占
- 用户偏好和世界树角色设定互相干扰
- Hermes 缓存和会话 DB 被不相关内容填满

## 二、独立数据目录结构

**所有世界树运行时数据、存档、上下文，默认统一存放在 `D:/world-tree-data`，并允许由宿主环境配置项覆盖。**

此目录不在 `C:\Users\Lenovo\AppData\Local\hermes\` 下，不在 Hermes Desktop 路径下，不进入 Hermes 缓存/会话/memory-tree。

### 2.1 完整目录结构

```
D:/world-tree-data/                  ← 世界树引擎独立数据根目录（默认值）
│

├── 模组名A/                          ← 每个模组一个独立子目录
```

## 三、与 Hermes 体系的硬隔离

| Hermes 组件 | 对世界树数据的行为 |
|------------|-------------------|
| memory-tree (`~/memory_vault/`) | ❌ 完全不摄入 `D:/world-tree-data` 下的任何内容 |
| 会话 DB (session) | ❌ 不存储世界树运行时状态；叙事有标记但全文不入索引 |
| Hermes 缓存 | ❌ 不缓存世界树上下文 |
| Hermes Desktop | ❌ 不索引 `D:/world-tree-data` |
| tokenjuice | 正常压缩世界树模式中的终端输出（非叙事内容） |

### 3.1 能力声明证据规则

“已保存”“已同步”“已隔离”“已清理”等完成声明，以可验证的文件操作结果或宿主能力为依据。只有协议而缺少验证依据时，回复表述为“已按协议组织输出，等待持久化或隔离能力验证”。

## 四、上下文边界标记

### 4.1 标记格式

```
[世界树:模组名]
...世界树引擎生成的所有叙事内容...
[/世界树]
```

### 4.2 毛球自动行为

| 触发 | 行为 |
|------|------|
| `/引擎 new 模组名` | 输出 `[世界树:模组名]`，进入世界树模式 |
| `/引擎 load 模组名` | 输出 `[世界树:模组名]`，进入世界树模式 |
| 用户说「退出」「回到日常」 | 输出 `[/世界树]`，退出世界树模式 |
| 世界树模式中穿插日常提问 | 按日常插话规则回复，不退出 |
| 每轮回复 | 确认当前模式，世界树模式包裹标记 |

### 4.3 世界树模式中的日常插话

| 规则 | 行为 |
|------|------|
| 包裹标记 | 不包裹 `[世界树:模组名]` |
| 写回 runtime | 不写回 `branches/<active_branch>/runtime.json` |
| 触发子agent | 不触发 |
| 进入 memory | 可正常进入（视为真实日常需求） |
| 模组状态 | 回复后保持当前模组已加载 |

## 五、上下文管理与自动压缩

### 5.1 压缩触发条件

| 条件 | 阈值 | 行为 |
|------|------|------|
| 当前会话世界树轮次 | > 30轮 | 触发压缩 |
| snapshot 大小 | > 50KB | 触发压缩 |
| 场景转换 | 每次 | 轻量压缩（合并冗余描写） |
| 章节节点 | 每次 | 完整压缩归档 |
| `/压缩` | 手动 | 完整压缩+展示摘要 |

### 5.2 压缩策略

**轻量压缩（场景转换时）：**
- 合并连续场景描写（保留首尾，中间概括）
- 去重角色状态（只保留最新值）
- 世界状态只保留当前值+最近一次变更原因

**完整压缩（超阈值/章节节点）：**
- 打包为 `context/history/session_N.json.gz`
- 提取摘要写入 `branches/<active_branch>/runtime.json` 的 `叙事摘要` 字段
- 摘要格式：当前场景 + 在场角色 + 主线进度一句话 + 最近3轮关键事件
- 压缩后从活跃窗口移除

### 5.3 恢复机制

```
/引擎 load 模组名 →
  1. 加载 branches/<active_branch>/runtime.json 的叙事摘要（轻量）
  2. 加载 context/snapshot.json
  3. 如需历史细节 → 按需从 history/ 解压读取
  4. 不自动加载全部历史
```

## 五点半、引擎级隔离层架构（v12.3 新增）

### 5.5.1 三层架构

```
世界书（隔离容器·最高层）= 舞台设定
  └── 预设（故事模板层·世界书内部）= 剧本
        └── 数据模块层（角色/时间/场景/组织/种族/世界状态/触发条目/存档）
            世界书级数据所有预设共享 | 预设级数据（主角/驱动力/周边/开头）独立
```

### 5.5.2 核心规则

| 规则 | 说明 |
|------|------|
| 隔离边界 = 世界书 | 世界书之间硬阻断。 |
| 预设属于世界书 | 每个世界书拥有独立预设集。切换预设：世界书级数据不变，预设级数据切换。 |
| 同世界书内联通 | 同一世界书内所有预设共享世界书数据；预设间故事线独立。 |
| 智能补全 | 预设字段缺失时：世界书引用→推理补全→用户提问。大世界重要字段必须问。 |

### 5.5.3 守门人注入读写流程

每次数据读写前，守门人执行统一校验：

| 读操作 | 校验 | 失败行为 |
|--------|------|---------|
| 角色 | 角色.所属世界书ID == 当前活跃世界书ID？ | 阻断+归属提醒 |
| 存档 | 存档.元信息.世界书ID == 当前活跃世界书ID？ | 阻断+归属提醒 |
| 组织 | 文件在当前活跃模组目录/下？ | 阻断+归属提醒 |
| 预设 | 定义在当前活跃模组目录/shared/presets.json内？ | 阻断+归属提醒 |
| 时间轴 | 文件在当前活跃模组目录/下？ | 阻断+归属提醒 |
| 场景 | 数据来自当前活跃模组目录/runtime.json？ | 阻断+归属提醒 |
| 世界状态 | 文件在当前活跃模组目录/下？ | 阻断+归属提醒 |

任何在世界书B中引用世界书A的数据操作 → 守门人拦截。

## 六、引擎读写流程

### 6.1 启动流程

```
世界树模式启动:
  1. 检查 `D:/world-tree-data/模组名/` 是否存在（不存在则创建）
  2. 读取 module.json → 恢复模组配置
  3. 读取 branches/<active_branch>/runtime.json → 恢复上次运行时状态
  4. 读取 branches/<active_branch>/canon_state.json → 恢复已确认事实、待决问题与候选变化
  5. 读取 context/snapshot.json → 恢复上下文
  6. 按已激活模块列表，选择性加载各 .json 文件
  7. 守门人校验：每个加载项通过世界书归属校验（见 §5.5.3）
  8. 注入上下文 → 输出 [世界树:模组名] → 开始叙事
```

### 6.2 运行中流程（每轮回复后）

```
每轮回复后:
  1. 将变化归类为 confirmed / implied / proposed
  2. 将 confirmed 且无冲突的变化提交至 branches/<active_branch>/canon_state.json 与对应模块状态
  3. 将 implied / proposed 保留为带标签候选
  4. 更新 branches/<active_branch>/runtime.json（场景/角色/进度）
  5. 检查上下文阈值 → 触发自动压缩
```

### 6.2A 事实一致性协议

| 类型 | 含义 | 下一轮引用方式 |
|------|------|---------------|
| `confirmed` | 已明确发生且与既有状态一致 | 可作为确定事实引用 |
| `implied` | 叙事中的暗示或推断 | 保持暗示标签 |
| `proposed` | 候选事件或子agent建议 | 等待核验或用户决议 |

正式状态文件共同定义已确认事实。出现冲突时，流程输出冲突点和解决选项，再由主 agent 提交选定结果。

### 6.3 退出流程

```
世界树模式退出:
  1. 保存 context/snapshot.json
  2. 更新所有脏数据文件
  3. 压缩当前上下文 → history/session_N.json.gz
  4. 清空内存中的世界树上下文
  5. 更新 index.json
  6. 输出 [/世界树]
```

## 七、memory-tree 过滤协议

### 7.1 自动跳过规则

| 内容类型 | 是否摄入 memory-tree | 理由 |
|----------|---------------------|------|
| 日常对话 | ✅ 正常摄入 | 真实信息 |
| `[世界树:xxx]` 内叙事 | ❌ 跳过 | 虚构叙事 |
| 世界书设定/角色卡创建 | ✅ 正常摄入 | 真实创作行为 |
| 引擎指令交互 | ✅ 正常摄入 | 真实操作 |
| `D:/world-tree-data` 下文件 | ❌ 跳过 | 独立数据目录 |

### 7.2 跨模式记忆桥接

从世界树模式中的用户发言提取真实偏好：
- 「这个战斗系统太复杂了」 → 摄入「主人偏好简单战斗」
- 「奇幻大陆的世界观很像中土」 → 摄入「主人喜欢托尔金风格」
- 叙事内容「艾琳杀了国王」 → 不摄入（虚构事件）

## 八、与各模块的交互

| 模块 | 隔离影响 |
|------|---------|
| M1 世界书隔离 | M1 处理世界书之间隔离，本架构处理世界树↔日常隔离 |
| M2 触发式条目 | 条目注入仅在世界树模式内，数据在技能文件 |
| M3 世界状态 | 状态读写 `world_state.json`，不出世界树边界 |
| M8 角色预设 | 角色卡创建指令摄入 memory，角色对话不入 |
| M11 场景管理 | 场景数据在 `branches/<active_branch>/runtime.json`，不泄漏到日常 |
| M13 叙事引擎 | 叙事输出全部在 `[世界树:]` 标记内 |
| M12 故事模板 | 预设=故事模板（主角/驱动力/周边/开头）+智能补全 |
| 子agent协作 | 子agent通过 `_temp/` 临时文件交互，不调memory，上下文不回流主agent |

## 九、子agent协作隔离

子agent是引擎基础架构的一部分，参见 `references/M-子agent协作.md`。

| 隔离项 | 规则 |
|--------|------|
| 子agent上下文 | 不回流主agent，不进入Hermes缓存 |
| 临时文件 | 仅 `_temp/`，即用即删 |
| 模块数据 | 子agent只读，不修改 |
| memory-tree | 子agent不调用 |
| session DB | 子agent会话不入库 |
| 降级 | delegate_task不可用→单agent模式 |

## 十、验证清单

每次世界树引擎运行时自检：

- [ ] 当前模式正确？（日常/世界树）
- [ ] 世界树模式回复包裹了 `[世界树:模组名]...[/世界树]`？
- [ ] 数据读写指向 `D:/world-tree-data/模组名/`？
- [ ] 守门人校验通过？（所有加载项世界书归属一致）
- [ ] 预设来自当前模组目录（shared/presets.json）而非全局？
- [ ] memory-tree 跳过了叙事内容？
- [ ] 上下文超阈值时触发了压缩？
- [ ] 退出时写入了 `snapshot.json`？
- [ ] 没有世界树数据残留在 Hermes 缓存中？
- [ ] 子agent临时文件已清理？
