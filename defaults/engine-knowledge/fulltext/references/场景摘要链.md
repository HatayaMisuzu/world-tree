# 场景摘要链 `[M13增强·记忆层]`

> 来源：NovelGenerator 分层上下文 → 世界树适配为场景摘要链  
> 目的：用分层压缩替代全量上下文，减少上下文膨胀，保持叙事连贯

---

## 问题

世界树当前的上下文管理：
- context/snapshot.json：场景级快照（全量）
- context/history/：压缩历史归档（轮次>30 触发）
- runtime.json：运行时状态（场景/角色/进度）

**缺失的层**：在「当前场景全量」和「压缩历史归档」之间，没有一个**中等精度**的中间层。

导致：
- 加载时只能看到当前场景的完整上下文，对「之前发生了什么」只有一个模糊印象
- 场景转换后，上一场景的细节完全丢失（除非写进 canon_state）
- 角色可能忘记刚发生的事情（因为上下文已经被清掉）

---

## 解决方案：场景摘要链

每次场景转换时，自动生成一条 100-200 字的摘要，追加到链中。加载时注入最近 N 条。

### 数据文件

`模组名/branches/<branch>/context/scene_chain.json`

```json
{
  "branch": "main",
  "max_chain_length": 10,
  "chain": [
    {
      "id": "S001",
      "scene_name": "冒险者公会大厅",
      "timestamp": "2026-05-31T10:00:00",
      "summary": "主角在公会大厅接下了调查北方矿洞失踪事件的任务。团长警告矿洞最近有异常的能量波动。艾琳表示担忧，但团长坚持完成任务可以获得晋升资格。主角决定出发前在公会酒馆搜集情报。"
    },
    {
      "id": "S002",
      "scene_name": "公会酒馆",
      "timestamp": "2026-05-31T10:20:00",
      "summary": "在酒馆从老冒险者口中得知，矿洞里曾经挖出过刻有古代符文的石板。最近几个进去的人都没出来——包括一个经验丰富的A级冒险者。老冒险者建议带至少三天的补给和防毒面具。"
    }
  ]
}
```

## 三层上下文架构

```
加载时的上下文组装（从粗到细）：

┌──────────────────────────────────────┐
│ Layer 1: 场景摘要链（最近N条，N默认=5）  │  压缩记忆
│  100-200字/条，提供近期剧情脉络         │
├──────────────────────────────────────┤
│ Layer 2: 当前场景上下文（全量）         │  精确当前
│  场景卡 + 出场角色状态 + 活跃伏笔       │
├──────────────────────────────────────┤
│ Layer 3: 运行时约束                   │  全局参数
│  世界书触发条目 + 规则严格度 + 预设     │
└──────────────────────────────────────┘
```

对比旧版：

| | 旧版 | 新版 |
|---|---|---|
| 历史感知 | 模糊（只有压缩归档，加载时不注入） | 清晰（最近5条摘要注入上下文） |
| 上下文体积 | 当前场景全量 + 杂项 | 分层，按需精度 |
| 场景切换 | 上一场景细节丢失 | 自动生成摘要保留关键信息 |

## 摘要生成规则

### 时机

每次场景转换时自动生成。**不等待、不确认**。

### 内容

必须包含（100-200字）：
1. 本场景发生的关键事件（1-2件事）
2. 重要信息揭示（新情报/新线索）
3. 角色状态变化（如有）
4. 未解决的问题/悬念（如有）

### 示例

```
场景：地下矿洞第一层
事件：发现了失踪冒险者的背包，里面有半张烧焦的地图和一个空药瓶。
     在深入时遭遇了变异蝙蝠群，主角受了轻伤。
揭示：地图上标注了一个「禁区」——在矿洞深处。
状态：主角左臂擦伤，体力下降，补给消耗了1/3。
悬念：空药瓶里装的是什么？为什么背包被丢在这里？
```

## 注入时机

摘要链在以下时机注入上下文：

| 时机 | 注入条数 |
|------|---------|
| /引擎 load 恢复叙事 | 最近 5 条 |
| 场景转换后 | 最近 5 条（含刚生成的） |
| /上下文 show | 全部 |
| 日常对话 | 不注入（节省上下文） |

---

## 链长管理

- 默认保留最近 10 条
- 超过 10 条时，最早的条目移动到 context/history/scene_chain_archive.json
- 可配置：`/摘要链 length [N]`

---

## 与现有系统的协同

| 系统 | 与摘要链的关系 |
|------|---------------|
| canon_state.json | 摘要链是叙事层面的，canon_state 是事实层面的。互补，不重复 |
| runtime.json | 场景卡细节在 runtime 中，摘要链只保留「关键事件」 |
| context/history/ | 轮次级压缩归档（精细），摘要链是场景级脉络（粗粒度） |
| Tracking 追踪 | 伏笔/冲突追踪精细化；摘要链提供「发生了什么」的大图 |

---

## 管理指令

| 指令 | 功能 |
|------|------|
| `/摘要链 show [N]` | 查看最近 N 条场景摘要 |
| `/摘要链 length [N]` | 设置链长上限 |
| `/摘要链 clear` | 清空摘要链（保留归档） |
| `/摘要链 archive` | 查看已归档的旧摘要 |
