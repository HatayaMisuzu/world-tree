## 〇.五、多世界书隔离架构（Multi-World Isolation）`[M1·核心]`

### 0.5.1 核心原则

**一个世界书就是一个完整独立的宇宙。** 世界书之间互不知晓、互不干扰。

```
世界书A（奇幻冒险）       世界书B（科幻未来）       世界书C（现代校园）
├── 预设集（独立）         ├── 预设集（独立）         ├── 预设集（独立）
├── 设定条目               ├── 设定条目               ├── 设定条目
├── 角色组（绑定）         ├── 角色组（绑定）         ├── 角色组（绑定）
├── 组织/种族              ├── 组织/种族              ├── 组织/种族
├── 时间轴                 ├── 时间轴                 ├── 时间轴
├── 世界状态               ├── 世界状态               ├── 世界状态
├── 场景                   ├── 场景                   ├── 场景
└── 存档                   └── 存档                   └── 存档

三者完全隔离。A的角色不会出现在B中，B的预设不会影响C的风格，A的时间线不会推进B的故事。
```

### 0.5.2 世界书容器结构

每个世界书是一个独立容器，包含：

```yaml
世界书容器:
  世界书ID: "fantasy-adventure-01"   # 唯一标识
  世界书名称: "艾瑟琳的奇幻大陆"        # 人类可读名
  世界类型: 奇幻/科幻/都市/古风/...   # 世界观分类
  创建时间: "2026-05-25"
  
  设定条目列表:                      # 该世界专属的触发式设定
    - 条目1
    - 条目2
    
  绑定角色列表:                      # 只属于这个世界书的角色
    - 艾琳
    - 团长
    
  世界书记忆:                        # 该世界的专属历史/事件
    - 已触发事件
    - 剧情进度
```

### 0.5.3 隔离铁律（最高优先级）

> 以下规则优先级高于所有其他规则，不可被任何预设/角色/场景覆盖。

**铁律1 — 活跃世界书唯一性：** 同一时刻，有且只有**一个**活跃世界书。

**铁律2 — 全维度绑定不可跨世界：** 角色、组织、时间轴、场景、存档在创建时绑定所属世界书，不可跨世界书访问。角色A属于世界书X，则角色A永远不能在世界书Y中出现或对话。世界书X的时间线不能推进世界书Y的事件。

**铁律3 — 设定隔离：** 当前活跃世界书的设定条目只在该世界书内触发。切换世界书后，旧世界书的所有激活条目全部失效。

**铁律4 — 切换即全维度重置：** 切换世界书时，角色/时间轴/场景/组织/种族/世界状态/触发条目/预设全部从内存清空，守门人校验后从目标世界书目录重新加载。

**铁律5 — 串台自动阻断：** 如果主人试图在世界书B中提及世界书A的角色/设定，毛球必须提醒：
> 「主人，【角色名】属于【世界书A】，不在当前世界书【世界书B】中哦～要切换到那个世界书吗？」

### 0.5.4 世界书集管理指令

| 指令 | 功能 | 示例 |
|------|------|------|
| `/世界书集 list` | 列出所有已创建的世界书及概要 | |
| `/世界书集 create 名称 [类型]` | 创建新的独立世界书容器 | `/世界书集 create 赛博东京 科幻` |
| `/世界书集 switch 名称` | 切换到指定世界书（活跃） | `/世界书集 switch 赛博东京` |
| `/世界书集 delete 名称` | 删除世界书及其所有绑定角色 | `/世界书集 delete 赛博东京` |
| `/世界书集 now` | 显示当前活跃世界书信息 | |
| `/世界书集 rename 旧名: 新名` | 重命名世界书 | |

### 0.5.5 世界书切换完整流程（全维度）

当执行 `/世界书集 switch 目标世界书` 时：

```
1. 保存并卸载当前世界书所有维度数据：
   角色表 / 组织表 / 时间轴 / 世界状态 / 随机状态 / 场景 / 预设参数 / 触发条目
2. 清空内存中所有维度数据（上下文窗口全清）
3. 加载目标世界书数据（按顺序）：
   module.json → shared/presets.json → 角色/组织/时间/场景/世界状态/触发条目
4. 守门人逐项校验：
   每个加载项所属世界书ID == 当前活跃世界书ID？
   任一失败 → 阻断 + 世界书归属提醒
5. 恢复上次离开该世界书时的场景/剧情进度
6. 确认：「已切换到【目标世界书】，角色/场景/预设均来自该世界书」
```

### 0.5.5A 守门人校验规则

引擎级守门人在所有数据读写前执行统一校验：

| 操作 | 校验 | 失败行为 |
|------|------|---------|
| 读角色 | 角色.所属世界书ID == 当前活跃世界书ID？ | 阻断+归属提醒 |
| 读存档 | 存档.元信息.世界书ID == 当前活跃世界书ID？ | 阻断+归属提醒 |
| 读组织 | 组织文件在 当前活跃模组目录/下？ | 阻断+归属提醒 |
| 读预设 | 预设定义在 当前活跃模组目录/shared/presets.json 内？ | 阻断+归属提醒 |
| 读时间轴 | 时间轴文件在 当前活跃模组目录/下？ | 阻断+归属提醒 |
| 读场景 | 场景数据来自 当前活跃模组目录/ 的 runtime.json？ | 阻断+归属提醒 |
| 读世界状态 | 状态文件在 当前活跃模组目录/下？ | 阻断+归属提醒 |

主人在世界书B中引用世界书A的角色、场景、预设、组织、时间线、世界状态等任何维度 → 守门人拦截。

### 0.5.5D 会话级隔离盲区（Hermes Session Blindspot）

> ⚠️ M1 隔离机制覆盖数据目录级（角色/时间轴/场景/存档/预设均绑定模组目录），
> 但**不覆盖 Hermes 会话级上下文**。这是世界树引擎与 Hermes 对话系统之间的架构边界。

**盲区说明：**

世界树的铁律1-5在「引擎加载世界书后」的层面生效——数据读写时守门人校验归属。
但在「用户对 Hermes 说'加载XX'」这一步，Writer agent 收到的 **不是** 干净的系统提示，
而是 **当前会话的全部历史消息**。

如果用户在同一个 Hermes 会话中先后说：
```
加载共感娃娃  →  (交互几十轮...)  →  加载万界
```

第二条指令到达 Writer agent 时，会话历史已经包含：
- "共感娃娃"世界书的全部设定注入
- 相关角色的对话/叙事内容
- 前一次加载指令和引擎响应

模型很可能在响应"加载万界"时带入旧世界书的信息。

**影响场景：**
- 控制台「对话」Tab 中切换世界书但未新建会话（最常见）
- Gateway 中同一个 chat_id 下先后加载不同模组
- 同一会话中 `/引擎 load A` → `/引擎 exit` → `/引擎 load B`

**缓解措施（引擎层面无法根本解决，需操作规范）：**

1. **控制台用户：** 切换世界书/模组时，先点「新建会话」再加载（见 SKILL.md §8.5A）
2. **Gateway 用户：** 在平台端新建对话线程，或使用 `/new` 指令清空会话
3. **引擎提示：** `/引擎 load` 成功后应主动提醒「如需完全隔离上下文，建议使用 /new 后再加载」
4. **长远方案：** 控制台在用户选择不同世界书加载时自动触发新建会话

**排查验证：**
- 检查 writer profile 的 `agent.log`，搜索同 session_id 下是否出现多个不同模组名
- 检查 session DB 中同一 session 的 message 列表，看是否有跨世界书的对话串扰
- 详细案例见 `references/控制台会话隔离排查.md`

当执行 `/预设 use 目标预设` 时（同世界书内）：

```
1. 保存当前预设数据：presets/当前预设名/runtime.json + timeline.json
2. 卸载内存中预设级数据：主角/驱动力/周边/时间线/场景/开头
3. 加载目标预设：
   读取 presets/目标预设名/preset.json → 主角/驱动力/周边/开头
   读取 presets/目标预设名/runtime.json → 场景/进度
   读取 presets/目标预设名/timeline.json → 故事时间线
4. 智能补全检查：缺失字段触发提问
5. 守门人校验：预设属于当前活跃世界书？
6. 确认：「已切换到预设【目标预设】，主角：XXX，故事类型：XXX」
7. 世界书级数据（世界观/规则/组织/角色库/历史/世界状态）完全不变
```

### 0.5.6 默认世界书

引擎初始化时自动创建：

```yaml
世界书ID: "default"
世界书名称: "默认日常"
世界类型: 现代日常
绑定角色: [毛球]（毛球自身人设始终属于默认世界书）
```

如果主人不创建任何世界书，所有互动都在"默认日常"世界书中进行。
