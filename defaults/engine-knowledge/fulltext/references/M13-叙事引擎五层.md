## 五、叙事引擎五层架构 `[M13·核心]`

继承自erotic-writing skill §十，本引擎完善并泛化该架构。**五层架构在每个世界书中独立运行。**

```
┌─────────────────────────────────────────────────────┐
│                   叙事引擎核心                        │
├──────────┬──────────┬──────────┬────────────────────┤
│ ①角色层  │ ②环境层  │ ③剧情层  │ ④语气/风格层       │
│ 角色卡    │ 场景卡    │ 事件线    │ 预设参数           │
│ 关系网络  │ 世界书    │ 因果链    │ 节奏/齁度/视角     │
│ 声纹库    │ 道具态    │ 伏笔表    │ 选项/分支          │
├──────────┴──────────┴──────────┴────────────────────┤
│ ⑤记忆/上下文层                                      │
│ 存档点 | 关键事件 | 角色状态 | 世界书激活记录        │
│ 场景摘要链(v12.7) | 活跃伏笔追踪                      │
└─────────────────────────────────────────────────────┘
```

### 5.1 角色层

角色层使用角色预设系统(§二)中的角色卡。多个活跃角色同时存在时：
- 每人独立追踪体力/情绪/装备/位置
- 用声纹区分对话（读者不看名字也能分辨谁在说话）
- 出场/退场有明确标记

### 5.2 环境层

环境层由场景卡 + 世界书激活条目组成：
- 场景卡提供当下物理环境
- 世界书提供世界观层面的设定约束
- 道具状态跨场景追踪（跳蛋在哪、档位、电量）

### 5.3 剧情层

剧情层追踪：
- 当前主线/支线
- 已触发事件
- 待解决冲突
- 未使用伏笔
- 时间线（避免"昨天的事今天还没发生"）

### 5.4 语气/风格层

使用风格预设(§三)控制输出参数。切换预设即时生效。

### 5.5 记忆/上下文层

**v12.7新增：分层上下文架构**

采用三层上下文组装（详见 [场景摘要链.md](场景摘要链.md)）：
- Layer 1：场景摘要链（最近N条，100-200字/条）—— 压缩记忆
- Layer 2：当前场景上下文（全量）—— 精确当前
- Layer 3：运行时约束（世界书触发条目 + 规则 + 预设）—— 全局参数

- 存档点机制（从erotic-writing继承）
- 关键事件精简存储
- 世界书激活记录
- 跨会话只需记住：当前模组名/角色名单/剧情节点/世界书文件名 + 最近场景摘要链

**v12.7新增：场景摘要链**

每次场景转换时自动生成100-200字结构化摘要，存入 `context/scene_chain.json`。

上下文注入分层：
```
Layer 1: 当前场景全量上下文（runtime.json）
Layer 2: 前N场景摘要链（最近5条，每条100-200字）
Layer 3: 活跃伏笔 + 追踪标记（Tracking系统）
Layer 4: 世界状态摘要（M3）
```

参见 [场景摘要链.md](场景摘要链.md)
