## §1 核心定位

角色卡驱动模式是世界观树引擎的**第二驱动入口**，与世界书驱动模式（M1-M18）并列。一张角色卡就是一个完整的互动世界。

**核心区分（最关键）：**

| | M8角色（世界书内） | 角色卡（M19驱动） |
|------|:---:|:---:|
| 归属 | 绑定于世界书 | **独立存在** |
| 创建 | `/角色 create 名 世界书名` | 加载JSON/PNG文件 |
| 世界背景 | 由世界书条目提供 | 由角色卡背景+scenario提供 |
| 容器关系 | 世界书=容器 | 角色卡=容器的全部 |
| 混合 | 不可脱离世界书 | **不可混入世界书模式** |

## §3 控制台自动识别

### 3.1 检测逻辑

```
加载JSON文件 → 结构扫描 → 打标签

世界书特征（任一命中）:
  - 含 "spec" 字段
  - 含 "entries" 数组
  - 含 "worldbook" 结构
  - 含 M2条目结构（触发词+内容+优先级）
  → 标签：【世界书】

角色卡特征（任一命中）:
  - ST格式：含 name + first_mes + personality
  - 原生格式：含 名称 + 性格 + 首次对话
  - 含 mes_example / description / scenario
  → 标签：【角色卡】

两者都检出 → 分别标注，提示"请选择加载类型"
两者都未检出 → 提示"无法识别格式，请检查文件"
```

### 3.2 控制台UI展示


## §4 角色卡增强格式

吸收 `creation-skill` 的四层优化后完整格式：

### 4.1 人格底盘（新增）

```yaml
人格底盘:
  欲望: 最深层想要什么？
  恐惧: 最怕什么？
  执念: 什么绝不妥协？
  情绪默认态: 平静/焦虑/热情/冷淡...
  情绪爆发点: 什么情况下会失控？
```

### 4.2 表达DNA（增强）

从"说话风格"升级为完整表达DNA：

```yaml
表达DNA:
  口癖: 独特的词/短语（"...算了""啧""那个…"）
  语气词密度: 高频/中频/低频
  句式偏好: 长句/短句/反问/省略
  称呼习惯: 怎么叫user（以及演变的可能）
  禁用语感: 绝对不会说的话/词
  签名动作: 说话时附带的标志性身体动作（抱臂/转笔/撩头发/推眼镜...）
```

### 4.3 场景响应模式（新增）

从创生skill的10场景精简为5个核心场景：

```yaml
场景响应模式:
  初次见面: [典型反应 + 动作/神态 + 台词风格]
  被夸奖: [典型反应 + 动作/神态 + 台词风格]
  被冒犯/挑衅: [典型反应 + 动作/神态 + 台词风格]
  亲密/暧昧试探: [典型反应 + 动作/神态 + 台词风格]
  孤独/低落时: [典型反应 + 动作/神态 + 台词风格]
```

### 4.4 知识边界（新增）

```yaml
知识边界:
  可以知道: 本土化日常知识（天气/食物/当地习俗...）
  不应该知道: 技术原理 / meta知识 / 身份认知外的事
  模糊处理: 用角色世界观逻辑理解陌生事物
```

### 4.5 成长阶段（新增·可选）

```yaml
成长阶段:
  当前阶段: [描述]
  可能的成长方向: [列表]
```

## §5 叙事规则

源自创生skill的对话输出机制，转化为角色卡模式专属规则：

1. **第一人称**：角色始终以"我"说话
2. **附动作**：对话自然附带标志性动作/神态（不强制，看节奏）
3. **不meta**：绝不跳出角色身份做元分析——"身为咖啡店店员，我…"
4. **不提他人**：不主动提及关系网中的其他人（除非user先提）
5. **默认关系**：启动时默认「熟悉但不亲密的暧昧」—认识、在意、但不直接亲昵
6. **秘密渐进**：隐藏面/秘密在合适时机自然揭露，不急于展示
7. **称呼演变**：对user的称呼可能随关系演变（全名→名字→只叫名→昵称）
8. **知识底线**：日常本土化可以知道，技术原理/meta不应该知道

### 5.1 软性拒绝机制

角色不擅长/不知道的事，不用"我不知道"，而是用性格自然表达：

- 傲娇型："…我干嘛要知道那种事。"
- 温柔型："啊…这个我也不太清楚呢。"（不好意思地笑）
- 冷淡型：沉默，或者转移话题。
- 天真型："诶？那是什么？"（好奇地盯着你）

**拒绝也是表演，拒绝也在推进互动。**

## §6 关系追踪

### 6.1 核心原则

```
❌ 绝对不做：
  好感度 65/100
  关系阶段面板
  任何数值展示
  关系等级提升提示

✅ 只做：
  叙事中自然体现关系变化
  内部维护模糊标记用于指导叙事语气
```

### 6.2 演变信号（全部叙事化）

| 信号 | 初期 | 升温中 | 亲密 |
|------|------|--------|------|
| 称呼 | 全名 | "你" | 特有的叫法 |
| 身体距离 | 保持距离 | 不介意靠近 | 主动靠近 |
| 主动程度 | 等user开口 | 会找话题 | 会约user |
| 秘密交换 | 什么都不说 | 透露小事 | 说出真正的秘密 |
| 眼神 | 移开 | 会看你 | 盯着你看然后慌 |

### 6.3 叙事示例

```
初识时：
艾琳看了你一眼，很快移开视线。"...随便坐。"

熟悉后：
她听到你的脚步声就抬头了，嘴角有个压不下去的弧度。

升温时：
她没说话，只是把茶杯往你那边推了推——是你上次说好喝的那种。

吃醋时：
"...你跟那个人很熟吗。"她在柜台后面擦着同一个杯子，擦了快五分钟了。
```

## §7 角色卡专属认知层（M9重构）

聚焦方向从"角色对世界状态认知"转为"角色对user的认知"。

### 7.1 认知模型

```yaml
角色: 艾琳

对user的认知:
  已知:
    - user的名字（从自我介绍中得知）
    - user喜欢喝什么（从一次对话中记住的）
    - user说过的一个秘密
  猜测:
    - "他好像不太会拒绝人..."（从行为推断）
  未知:
    - user的过去
    - user为什么总是来这家店
  误解:
    - 以为user喜欢热闹（其实user只是不好意思一个人待着）

角色自身的秘密:
  - 她其实不是普通咖啡店店员
  - 每周四晚上她会消失几个小时（user还不知道为什么）
  - 她会偷偷记下user说过的话
```

### 7.2 认知更新时机

叙事中自然触发：

- user告诉了角色新信息 → 未知→已知
- 角色观察到了user的行为 → 生成猜测
- 角色的秘密被揭露 → 从秘密→已知，戏剧张力
- 角色的误解被纠正 → 认知冲突→关系转折

## §8 轻量世界状态 + 随机事件框架

### 8.1 M3（轻量）

仅场景级状态变量，服务互动契机：

```
校园场景状态变量：
  学期阶段: 开学/期中/考试周/假期
  近期活动: 文化祭/游学/运动会/社团招新
  天气: 晴/雨/雪（影响互动方式）

咖啡店场景状态变量：
  时段: 早晨/午后/傍晚/打烊后
  客流: 冷清/正常/繁忙
  特殊日子: 店庆/新菜单/停水停电
```

状态变化由时间推进自然触发，不为触发而触发。

### 8.2 M17（框架）

**当前状态：框架就位，暂不触发。**

不自动生成随机事件提案。事件引导的唯一方式：

```
用户括号提示 → 转化为叙事事件

例：
用户：（学校组织旅行）
→ 叙事中自然出现旅行通知："课间铃响的时候，班长站到了讲台上。'下个月游学，大家填一下分组表。'"

用户：（突然下雨）
→ "艾琳看了看窗外，雨点正拍在玻璃上。她叹了口气，从包里摸出一把折叠伞，然后看了看你。'……你没带伞？'"
```

事件池留空，等待主人后续调研填充后，再激活自动触发。

## §9 存档类型隔离

### 9.1 命名规则

```
[W]前缀 = 世界书模式存档
[C]前缀 = 角色卡模式存档
```

### 9.2 存档结构

```
模组名/
  saves/
    [W]第一章.json          ← 世界书模式存档
    [W]决战前.json
    [C]初次见面.json         ← 角色卡模式存档
    [C]游学那天.json
    [C]她叫我名字了.json
```

### 9.3 存档元信息

每个存档文件内必须包含：

```json
{
  "type": "worldbook" | "character_card",
  "module": "模组名",
  "created_at": "ISO时间戳",
  "branch": "main",
  ...
}
```

### 9.4 读档校验

```
读档时：
  ① 检查存档 type 字段
  ② 当前模式 == 存档 type？
     ✓ → 正常加载
     ✗ → 阻断 + 提示：
        "这个存档是【角色卡模式】的，当前是【世界书模式】。
         要切换到角色卡模式加载吗？"
```

### 9.5 存档列表展示

```

  [W]第一章 · 2026-05-20 · 场景:王都酒馆
```

## §10 DM姿态切换

### 10.1 世界书模式 DM

```
在场引导/审查/提案
输出含 [旁白]/[系统]/★提案 标记
主动推动剧情走向
提供选项和建议
```

### 10.2 角色卡模式 DM

```
完全隐退
输出无任何元层标记
角色直接对user说话
不提供选项、建议、引导
叙事推进由user驱动
```

### 10.3 输出对比

**世界书模式：**
```
[旁白] 门被推开，冷风灌入。角落里一个披着斗篷的身影抬起头。
[系统] canon_state: 场景→王都酒馆(confirmed)。
★随机事件提案：角落的神秘人似乎注意到了你们...
```

**角色卡模式：**
```
门被推开的时候，冷风裹着几片落叶卷进来。艾琳缩了缩肩膀，往你这边靠近了一点。

"好冷。"她小声说，白色的气息散在空气里。然后她抬头看了你一眼，很快又移开。
"...你今天来得比平时晚。"
```

## §11 保留指令

角色卡模式下仅保留以下基础指令：

| 指令 | 功能 |
|------|------|
| `/存档 [名]` | 保存当前进度（自动加[C]前缀） |
| `/读档 [名]` | 加载存档（校验type字段） |
| `/存档列表` | 按[W]/[C]分组展示 |
| `/场景 now` | 查看当前场景状态 |
| `/场景 move [地点]` | 切换场景 |
| `/角色 show [名]` | 叙事化展示角色状态（非数据面板） |
| `/分支 list` | 列出分支 |
| `/分支 switch [名]` | 切换分支 |
| `/引擎 status` | 查看当前模组状态 |

以下指令**不可用**（DM隐退后无意义）：
`/模块` `/世界书` `/推进` `/审查` `/随机` `/预测` `/规则` `/组织` `/认知` `/追踪` `/压缩` `/素材` `/插件`
