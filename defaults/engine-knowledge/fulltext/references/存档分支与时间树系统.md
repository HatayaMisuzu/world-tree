# 世界树引擎 — 存档分支与时间树系统

> 存档分支/时间树/数据隔离/跨分支对比/回档规范 v1.0
> 升级范围：M16时间模块 + §八存档系统 + Tracking追踪系统 + 数据目录结构

---

## 设计目标

将世界树从"单线叙事"升级为"时间树分支叙事"：

```
之前（单线）:                    之后（时间树）:
                                  root（模组创建）
root → 第1轮 → ... → 第20轮          │
        ↑                    [存档: 王都门前·第15轮]
        只能往前或回退               ╱         ╲
                              分支A: 帮艾琳    分支B: 放任不管
                                  │              │
                              [存档: 结盟]   [存档: 独自前行]
                                  │              │
                                ...            ...
```

用户可以在任何存档节点分叉出新的世界线，每条世界线有独立的状态、参数、冲突和伏笔追踪，互不污染。

### 节点类型 (node.type)

| 类型 | 说明 | 示例 |
|------|------|------|
| `origin` | 模组创建时的根节点 | 模组初始化 |
| `archive` | 用户创建的存档节点 | `/存档 王都门前` |
| `checkpoint` | 引擎自动标记的检查点 | 场景转换、章节节点 |
| `auto_save` | 自动存档节点 | 退出世界树模式时 |
| `merge_point` | 分支合并点（预留） | 两条分支汇合 |

### 分支状态 (branch.status)

| 状态 | 说明 |
|------|------|
| `active` | 活跃分支，可以继续叙事 |
| `paused` | 暂停的分支，存档后切到其他分支 |
| `completed` | 已完成的分支（故事收束） |
| `abandoned` | 废弃的分支 |
| `planned` | 已创建但尚未开始叙事的分支 |

## 二、分支数据隔离结构

### 核心设计：跨分支不变 vs 分支独立

```
共享层 (shared/)          ← 角色基础设定、世界书条目、组织基础结构
   │                      （跨所有分支不变，分叉时复制引用）
   │
分支层 (branches/xxx/)     ← canon_state / runtime / characters状态 / tracking
   │                      （每个分支完全独立，互不污染）


具体目录结构：
D:/world-tree-data/模组名/
├── module.json                     ← 引擎配置（跨分支共享）
├── timeline-tree.json              ← 时间树（所有节点和分支的索引）
│
├── shared/                         ← 跨分支共享数据
│   ├── worldbook.json              ← 世界书条目（触发词/内容）
│   ├── characters_base.json        ← 角色基础设定（外貌/性格/背景/目标）
│   ├── organizations.json          ← 组织基础结构（名称/层级/文化）
│   ├── presets.json                ← 自定义预设
│   └── modules_config.json         ← 已激活模块列表和配置
│
├── branches/                       ← 分支独立数据
│   ├── main/                       ← 主线分支
│   │   ├── canon_state.json        ← 本分支的已确认事实
│   │   ├── runtime.json            ← 本分支的运行时状态
│   │   ├── world_state.json        ← 本分支的世界状态（M3）
│   │   ├── characters_state.json   ← 本分支的角色动态状态（位置/体力/情绪/认知）
│   │   ├── timeline.json           ← 本分支的时间轴（M16）
│   │   ├── random_state.json       ← 本分支的随机事件状态（M17）
│   │   ├── archive/                ← 本分支的存档目录
│   │   │   └── 王都门前_main.json
│   │   └── tracking/               ← 本分支的追踪数据
│   │       ├── change-log.jsonl
│   │       ├── foreshadowing.json
│   │       ├── conflicts.json
│   │       ├── parameter-snapshots.json
│   │       ├── parameter-history.jsonl
│   │       ├── checkpoints.json
│   │       └── tracking-index.json
│   │
│   ├── branch-A/                   ← 分支A
│   │   ├── canon_state.json        ← 独立副本
│   │   ├── runtime.json            ← 独立副本
│   │   ├── world_state.json
│   │   ├── characters_state.json
│   │   ├── timeline.json
│   │   ├── random_state.json
│   │   ├── archive/
│   │   └── tracking/
│   │
│   └── branch-B/                   ← 分支B
│       └── ...（同上结构）
│
└── context/                         ← 上下文管理（当前活跃分支的快照）
    ├── snapshot.json               ← 当前分支上下文快照
    └── history/                    ← 当前分支历史归档
```

### 数据分层规则

| 数据 | 存放位置 | 分叉时行为 |
|------|---------|-----------|
| 世界书条目 | shared/worldbook.json | 不复制，所有分支共享同一份 |
| 角色基础设定 | shared/characters_base.json | 不复制，所有分支共享 |
| 组织基础结构 | shared/organizations.json | 不复制，所有分支共享 |
| 模块配置 | shared/modules_config.json | 不复制，所有分支共享 |
| 已确认事实 | branches/xxx/canon_state.json | 分叉时从父分支复制当前值 |
| 运行时状态 | branches/xxx/runtime.json | 分叉时从父分支复制当前值 |
| 角色动态状态 | branches/xxx/characters_state.json | 分叉时从父分支复制当前值 |
| 世界状态(M3) | branches/xxx/world_state.json | 分叉时从父分支复制当前值 |
| 时间轴(M16) | branches/xxx/timeline.json | 分叉时复制+添加分叉事件 |
| 随机事件(M17) | branches/xxx/random_state.json | 分叉时从父分支复制当前值 |
| 追踪数据 | branches/xxx/tracking/ | 分叉时从父分支复制当前状态 |
| 预设参数 | shared/presets.json | 不复制，但分支可覆盖 |

### 共享数据的修改规则

修改 shared/ 的数据（如角色基础设定、世界书条目）会影响所有分支。因为这是"世界设定"，不是"世界状态"。

如果某个分支需要角色有不同的基础设定（如性格在分支A发生了根本变化），可以：
1. 先在 shared/characters_base.json 修改基础设定（影响所有分支）
2. 或者用 `/角色 edit` 修改角色状态（只影响当前分支的动态状态文件）

## 三、存档操作适配

### 普通存档（当前分支内）

```
/存档 王都门前
→ 在当前分支的时间线上创建 archive 节点
→ 将当前分支数据写入 branches/{当前分支}/archive/
→ 更新 timeline-tree.json 的 nodes 和 branches.current_node
→ 创建参数快照到 tracking/parameter-snapshots.json
```

### 分支存档（从某节点分出新分支）

```
/存档 王都门前 --new-branch 帮助艾琳
→ 以当前节点为分叉点，创建新分支
→ 复制分叉点的所有分支数据到 branches/新分支名/
→ 在 timeline-tree.json 中注册新节点、新分支、分叉信息
→ 切换到新分支，active_branch = "新分支名"
→ 后续所有叙事和存档都在新分支内进行
```

### 从历史节点分叉

```
/分支 create 帮助艾琳 --from node-001
→ 在 node-001 处创建新分支
→ 加载 node-001 时刻的数据状态 → 复制到新分支目录
→ 用户从分叉点重新开始叙事
```

### 切换分支

```
/分支 switch main
→ 保存当前分支的 runtime.json + canon_state.json + context/snapshot.json
→ 读取目标分支的 runtime.json + canon_state.json + context/snapshot.json
→ 更新 timeline-tree.json 的 active_branch
→ 注入目标分支上下文 → 开始叙事
```

## 四、时间模块 (M16) 适应时间树

### M16 原有功能保留

- 个人时间驱动力
- 组织时间驱动力
- 世界时间轴
- 未来管控（预言限制/时间跳跃禁止/命运不存在）

### M16 新增功能：分支时间

```
每条分支有独立的：
  - 分支内世界时间（从分叉点开始计数，如 "分支A第3天"）
  - 分支内事件线
  - 分支的人/组织时间戳

timeline-tree.json 负责跨分支的时间关系：
  - 分叉点 = 时间树上的分裂节点
  - 分支A和分支B在分叉点之前共享同一个时间线
  - 分叉之后各自独立
```

### 时间树可视化

```
/时间树 show
→ 输出：

root (模组创建, Round 0)
 └─ node-001 [存档] 第15轮·王都门前 (main)
     ├─ node-A1 [存档] ★ 分支A: 帮助艾琳·第25轮 (active)
     └─ node-B1 [存档] 分支B: 放任不管·第20轮

活跃分支: 分支A (帮助艾琳)
```

### 时间一致性检查（跨分支不检查）

- 同一分支内做时间一致性检查
- 不同分支之间不做时间一致性检查（它们本来就是不同世界线）
- 但共享层（shared/）的数据修改会影响所有分支，需谨慎

## 五、冲突/伏笔隔离

### 每个分支独立的 tracking/

```
branches/main/tracking/
├── foreshadowing.json     ← 主线的伏笔
├── conflicts.json         ← 主线的冲突
└── ...

branches/branch-A/tracking/
├── foreshadowing.json     ← 分支A的伏笔（与主线完全独立）
├── conflicts.json         ← 分支A的冲突（与主线完全独立）
└── ...
```

分叉时行为：
- 分叉点处的伏笔/冲突状态复制到新分支
- 之后各分支独立追踪
- 不会出现"分支A回收了主线埋的伏笔"这种情况（隔离了的）

## 六、穿线防护机制

### 六条隔离铁律（新增分支隔离）

1. **分支数据不共享**：branches/ 下的数据文件跨分支完全独立
2. **活跃分支唯一**：同一时刻只有一个活跃分支
3. **分叉操作显式化**：所有分支创建都必须用户明确指令
4. **共享层只读保护**：叙事过程中 shared/ 的数据不自动修改，只有用户指令才能改
5. **上下文不泄漏**：切换分支时完整的上下文重置
6. **存档带分支标记**：所有存档文件包含 branch_id 字段

### 穿线检测

每次操作前自动检查：
- 当前操作的模组和分支是否匹配？
- 读取的数据是否来自正确的分支目录？
- 写入的数据是否写入当前分支？（防止误写其他分支）

## 七、指令升级

### 存档指令（升级）

| 指令 | 功能 |
|------|------|
| `/存档 [名称]` | 在当前分支保存（等同于原行为） |
| `/存档 [名称] --new-branch 分支名` | 从当前节点分出新分支 |
| `/读档 [节点ID\|存档名]` | 加载指定节点（在当前分支内） |
| `/读档 [节点ID] --new-branch 分支名` | 从历史节点分出新分支 |
| `/存档列表` | 列出当前分支的存档（升级：标注分支名） |
| `/存档列表 --all` | 列出所有分支的所有存档 |

### 分支指令（新增）

| 指令 | 功能 | 示例 |
|------|------|------|
| `/分支 list` | 列出所有分支及其状态 | `/分支 list` |
| `/分支 switch 分支名` | 切换到另一个分支 | `/分支 switch branch-B` |
| `/分支 create 分支名 --from 节点ID` | 从指定节点创建分叉 | `/分支 create 反派线 --from node-001` |
| `/分支 info 分支名` | 查看分支详情 | `/分支 info branch-A` |
| `/分支 compare 分支A 分支B` | 对比两分支的差异 | `/分支 compare main branch-A` |
| `/分支 delete 分支名` | 删除分支（不可恢复） | `/分支 delete branch-B` |
| `/分支 rename 旧名 新名` | 重命名分支 | `/分支 rename branch-A 艾琳线` |

### 时间树指令（新增）

| 指令 | 功能 |
|------|------|
| `/时间树 show` | 可视化显示完整时间树 |
| `/时间树 node 节点ID` | 查看节点详情 |
| `/时间树 path 节点A 节点B` | 查看两个节点的路径关系 |
| `/时间树 stats` | 时间树统计信息 |

## 八、版本迁移说明

### v12.2（当前）→ v12.3（本升级）

v12.2 的模组数据迁移到 v12.3 的时间树结构：

```
v12.2 结构:
D:/world-tree-data/模组名/
├── canon_state.json       → 迁移到 branches/main/canon_state.json
├── runtime.json           → 迁移到 branches/main/runtime.json
├── world_state.json       → 迁移到 branches/main/world_state.json
├── characters.json        → 拆分：
│                           基础设定 → shared/characters_base.json
│                           动态状态 → branches/main/characters_state.json
├── organizations.json     → 迁移到 shared/organizations.json
├── timeline.json          → 迁移到 branches/main/timeline.json
├── random_state.json      → 迁移到 branches/main/random_state.json
└── archive/               → 迁移到 branches/main/archive/

新增:
├── timeline-tree.json     ← 新创建
├── shared/                ← 新目录
└── branches/              ← 新目录（含main/）
```

迁移时自动创建时间树根节点和主线分支，将现有数据迁移到 branches/main/。
