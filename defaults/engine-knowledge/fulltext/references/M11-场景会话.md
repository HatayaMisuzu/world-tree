## 四、场景会话管理 `[M11·核心]`

### 4.1 场景概念

一个"场景"是一个有明确时空边界的故事片段，**严格属于当前活跃世界书**。切换世界书后场景完全重置。

**v12.0新增：场景内支持嵌套子场景（Stage），实现场景的层级结构。**

引擎维护当前场景的状态卡：

```yaml
场景卡:
  场景名: 
  场景级别: 场景 | 子场景        # v12.0新增，子场景嵌套在父场景内
  父场景: (如果是子场景)         # 所属父场景名称
  时间: (清晨/上午/下午/黄昏/深夜)
  地点: 
  在场角色: [角色A, 角色B, ...]
  环境描写: (2-3句)
  当前氛围: (轻松/紧张/浪漫/危险/...)
  场景类型: 普通 | 关键时刻    # 关键时刻自动激活慢镜头叙事
  感官锚点: (选1-3个主导感官，如: 听觉·远处狗吠 + 触觉·潮湿的墙壁)
  对话氛围: (信息交换/权力博弈/情感试探/日常寒暄)
  活跃道具: (正在使用的特殊物品)
  暴露风险: (仅色色场景: 安全/半公开/危险)
  访问计数: 0                    # v12.0新增，记录此场景被访问的次数
  首次访问时间:                  # v12.0新增
  上次离开时间:                  # v12.0新增
  场景内事件记录: []             # v12.0新增，记录本场景中发生的关键事件
```

### 4.2 嵌套子场景（Stage）v12.0新增

场景内可以有子场景，实现更精细的空间管理：

```
示例：父场景「冒险者公会」包含子场景：

  冒险者公会（父场景·建筑）
  ├── 公会大厅（子场景·接待区）
  │   ├── 任务公告栏
  │   └── 接待台（艾琳通常在这里）
  ├── 公会酒馆（子场景·社交区）
  │   ├── 吧台
  │   └── 角落的暗座
  ├── 地下训练场（子场景·功能区域）
  └── 二楼办公室（子场景·非公开区域）
      └── 团长办公室

场景层级规则：
- 父场景的设定和氛围自动继承到子场景
- 子场景可以有独立的氛围和感官锚点
- 角色在子场景间移动：/场景 move 酒馆（从大厅移动到酒馆）
- 角色离开父场景：/场景 move 王都广场（完全离开公会建筑）
- 父场景的访问计数 = 所有子场景访问计数之和
```

**访问计数（Visited Counter）的叙事应用：**

```yaml
访问计数 = {
  公会大厅: 12,
  公会酒馆: 8,
  地下训练场: 3,
  团长办公室: 1,
  王都广场: 5
}
```

毛球根据访问计数调整叙事细节：
- 首次访问：详细环境描写，角色对周围充满好奇
- 第2-3次：角色开始熟悉布局，环境描写精简
- 第5次以上：角色对环境熟视无睹，不再描写基础布局，只提变化
- 第10次以上：可能会有"每天都来的老地方"的倦怠感/归属感

```
第1次来酒馆：
「你推开厚重的橡木门，麦酒和烤肉的气味扑面而来。
  角落里坐着几个佣兵在玩牌，吧台后面老板在擦杯子。」

第8次来酒馆：
「佣兵们还是老位置。老板冲你点点头——已经知道你要点什么了。」
```

### 4.3 场景管理指令

| 指令 | 功能 |
|------|------|
| `/场景 new 场景名` | 开启新场景 |
| `/场景 now` | 显示当前场景状态 |
| `/场景 move 地点` | 角色移动到新地点 |
| `/场景 time 时间` | 推进/改变时间 |
| `/场景 join 角色名` | 角色加入当前场景 |
| `/场景 leave 角色名` | 角色退出当前场景 |
| `/场景 save 场景名` | 保存当前场景快照 |

### 4.4 场景切换流程

**场景内移动：**
切换场景时毛球自动：
1. 保存当前场景快照
2. 检查世界书中有无新地点相关的触发条目
3. 重新描述新环境（2-3句）
4. 更新在场角色位置
5. 恢复道具/身体状态追踪（如跳蛋还在体内、档位不变）
6. **场景走向预测（M18 激活时）**：基于当前局势生成 2-3 个可能走向 → 注入上下文作为叙事惯性 → 详见 [M18-场景走向预测.md](M18-场景走向预测.md)
7. **场景摘要（M13 记忆层增强）**：生成100-200字场景摘要追加到场景摘要链 → 详见 [场景摘要链.md](场景摘要链.md)
6. **生成场景摘要**（追加到 `context/scene_chain.json`，100-200字）`[v12.7新增]`
7. **生成场景走向预测**（M18激活时：2-3个可能方向，标注风险收益）`[v12.7新增]`

**跨世界书切换（/世界书集 switch）：**
1. 保存当前世界书的场景快照和角色状态
2. 清空当前场景卡、在场角色、激活条目
3. 加载目标世界书的设定条目
4. 加载目标世界书的绑定角色列表
5. 恢复目标世界书上次离开时的场景/剧情进度（如有存档）
6. 重新初始化场景卡
7. 毛球确认切换并告知当前可用角色

### 4.5 场景叙事指导

**关键时刻·慢镜头：** 场景类型标记为「关键时刻」时，毛球自动放慢叙事节奏——
- 将关键动作拆为微步骤序列（准备→执行→意外→调整→完成）
- 在动作间隙插入角色的判断、预感、身体反应
- 叙事时间 > 实际时间，让冲击力充分释放

**感官锚点：** 场景描写至少覆盖两到三种感官，避免"只有视觉的平面场景"——
- 紧张场景：加触觉（掌心出汗）和听觉（自己的心跳声）
- 浪漫场景：加嗅觉（对方身上的气味）和听觉（环境音变柔）
- 恐怖场景：加触觉（后背发凉）和嗅觉（说不清的腐败味）

**对话层次：** 角色对话不只是交换信息，同时在做——
- 动作中断（说话时喝水/翻文件/看手机，透露真实状态）
- 潜台词（说A想B，从上下文推断真实含义）
- 权力博弈（谁先开口、谁沉默、谁转移话题，都是关系微调）

**暗线穿插：** 当前场景中可以插入一笔暗线/伏笔，一两句即可——
- 配角的一个反常举动（"他今天泡茶的手法比平时慢了很多"）
- 环境中一闪而过的信息（"电视新闻里闪过一个熟悉的名字"）
- 不展开，不解释，为后续场景留下种子
