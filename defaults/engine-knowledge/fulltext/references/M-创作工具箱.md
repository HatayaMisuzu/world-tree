## 一、十大创作原则

### ① 规则即叙事

**含义：** 每一条规则都在讲故事，不是当裁判。规则的后果本身构成剧情张力。

**示例（艾尔德兰 — 晋升系统）：** 铜→银失败锁定一年。龙钢→神选失败永久失去资格。这条规则在说——你每一次晋升赌的都是职业生涯。数字背后站着的是「代价」。

**示例（艾尔德兰 — 天赋携带规则）：** 觉醒新天赋时若已带两个，必须放弃一个旧天赋，永远消失。这不是系统限制——是在说「你获得越多，必须放弃的也越多」。

**何时用：** 素材有系统/机制/规则类设定时

**自检：** 去掉规则的数字，剩下的文字有没有情感重量？

---

### ② 锚点角色

**含义：** 安排一个让人"想留下"的角色作为世界的温度来源。

**艾尔德兰示例 — 冥灯：** 每天清早对着空气说早安——三十七年前师傅最后一次跟她道早安的时间。一个物件（白骨渡鸦）+ 一个习惯（说早安）+ 一道伤口（师傅被侵蚀）+ 一个不说的事（她知道师傅回不来）。四个锚点，一个立体的角色。

**何时用：** 角色驱动剧情，或用少数角色撑起世界观时

**自检：** 核心角色有没有至少两个锚点？

---

### ③ 硬边界制造命运感

**含义：** 选择不可逆、路线永久关闭——让每个决定都有重量。

**艾尔德兰示例：** 每章三个结局，最终章HE需前六章全部HE，任意一章BE则最终章锁死BE路线。**过去的选择累积到最终章算总账。**

**何时用：** 需要选择有分量感的叙事时

**注意：** 硬边界制造的不是"痛苦"，而是"意义"。

---

### ④ 模板化扩展

**含义：** 定义统一的条目模板结构，添加新内容就是"填空"，世界书可生长。

**何时用：** 世界类型多、需预留扩展空间时

---

### ⑤ 资源改变关系

**含义：** 货币/道具/收集要素不应改变数值，而应改变玩家与故事的关系。

**检验：** 去掉道具名，看它是否在"改变故事规则"而不是"加数字"。

---

### ⑥ 小物件锚定法

**含义：** 用一个"小而重"的物件来锚定角色的情感核心。一个物件比十句描写更有力量。

**艾尔德兰分析：**

| 角色 | 小物件 | 为什么有力 |
|------|--------|-----------|
| 银翼 | 盾心吊坠 | "不是传承，是暂时寄放，他说还会拿回来" |
| 翠瞳 | 一片叶子 | 递给银翼，不解释——读者自己会想 |
| 冥灯 | 白骨渡鸦 | 冰冷世界里唯一还在动的东西 |
| 霜牙 | 剑柄龙语 | "他把剑转了一圈盖住字说：关你屁事" |
| 老马科 | 每晚倒的酒 | "祭那些再也没回来的人，从来不说是谁" |
| 炎拳 | 背上烫伤 | 弟弟夭折那年留下的，二十多年的执念 |

**设计方法：** 选小件（非大件）→ 可被看见/可被触摸/别人不懂为什么重要 → 不解释来由 → 关键时刻出现一次就够了

**自检：** 你能说出"为什么选它"，但读者要去猜？

---

### ⑦ 留白艺术

**含义：** 70%信息+30%悬念——读者自己拼凑的部分最有感染力。

**艾尔德兰留白标本：**

| 写了什么 | 留了什么 |
|---------|---------|
| 「天台曾经站着他年轻时的队友」 | 队友是谁？发生了什么？ |
| 「他在档案上写了两行字，划掉了一行」 | 划掉的是什么？ |
| 「她把最后一杯酒倒在吧台上」 | 祭的是谁？ |
| 「她递给他一片叶子」 | 叶子代表什么？ |

**留白方法：** 写完问自己"要不要删掉最后一句"→ 删掉后读者会脑补→删。三类东西最值得留白：过去/动机/关系。

**自检：** 设定中有没有一处读者可以推导出不同答案？

---

### ⑧ 仪式感开局

**含义：** 玩家进入世界后的第一件事必须是"不可跳过的仪式"——开局本身成为世界的第一次展示。

**艾尔德兰天启水晶模型：**

```
① 仪式动作：双手放上水晶
② 选择与回应：三个选项各有隐藏后果
③ 动态反馈：祭司反应随数值/天赋稀有度变化
```

**设计方法：** 一个具体动作 + 一个选择 + 一个动态回应。开局中暗藏至少1个持续性悬念。不要在开局中解释所有规则——展示结果而不是解释机制。

---

### ⑨ 分层递进

**含义：** 信息按"先给人，再给系统，最后给细节"的顺序分层呈现——人物是最好的记忆锚点。

**艾尔德兰分层：** L1世界概述 → L2主线剧情 → L3核心系统 → **L4人物（最精彩的部分）** → L5细节 → L6深度内容

**为什么L4比L3重要：** 玩家不会因为你的世界观宏大而留下，会因为你的角色有趣而留下。

---

### ⑩ 精而深胜于大而薄

**含义：** 小场景+深角色 > 大设定+薄内容。宁缺毋滥。

**做法：** 条目少不是问题，条目浅才是。每个条目都要有"读完后让人想了解更多"的信息密度。

## 二、技术卡1：人物锚点设计

### 四锚点框架

```
┌─────────────────────────────────────┐
│         角色锚点框架                 │
├─────────────────────────────────────┤
│  ▸ 锚点物件                         │
│     一件随身小东西——不说为什么重要     │
├─────────────────────────────────────┤
│  ▸ 习惯动作                         │
│     一个反复出现的微动作              │
├─────────────────────────────────────┤
│  ▸ 一道伤口                         │
│     身体或心灵的旧伤——不说来由         │
├─────────────────────────────────────┤
│  ▸ 一个不说的事                     │
│     所有人回避的话题或秘密            │
└─────────────────────────────────────┘
```

### 锚点强度自检

| 锚点 | 弱 | 强 | 最佳 |
|------|----|----|------|
| 物件 | 一把剑 | 剑柄刻着字的剑 | 转过去盖住字"关你屁事" |
| 动作 | 做祷告 | 每天对空气说早安 | 三十七年没收到回复 |
| 伤口 | 有个疤痕 | 弟弟留下的烫伤 | 每次进试炼之渊能在熔岩对岸看到他 |
| 秘密 | 有段往事 | 天台没人站的位置 | 曾经站着他年轻时的队友——没人知道是谁 |

### 设计模板

```text
角色名：[名字]
□ 锚点物件：[物体 / 描写 / 出现场景]
□ 习惯动作：[动作 / 频率 / 触发条件]
□ 一道伤口：[类型 / 呈现方式 / 不写的部分]
□ 一个不说的事：[范围 / 谁在回避 / 暗示方式]
```

## 三、技术卡2：结局架构模板

### 三重结局

```
HE（英雄）: 全员存活，但付出代价
BE（破灭）: 有人死亡，但死得有意义
OE（混沌）: 全员活但永远回不去了
```

### 累积型级联

最终章条件：前六章全部HE→HE路线 / 任意一章BE→锁死BE / 至少一章OE且无BE→OE路线。死亡的NPC永不再出现，存活NPC好感度上限解封。

### 代价设计

| 结局 | 代价模式 | 示例 |
|------|---------|------|
| HE | 失去了什么？ | 银翼重伤昏迷 |
| BE | 谁死了？有意义吗？ | 霜牙为保护姐姐挡龙息 |
| OE | 什么永远变了？ | 翠瞳再没离开林海 |

### 自检

□ HE不是"都活了"——有人付出了代价吗？
□ BE不是"都死了"——死的人有意义吗？
□ OE不是"最好的结局"——它比HE更让人难过吗？
□ 三个结局不是ABCD选择，是前面所有选择的累积清算

## 四、技术卡3：叙事密度检查表

### 四步自检

```text
□ 第一步：去掉名字——还剩什么？
□ 第二步：一句话梗概——这条目"真正在说什么"？
□ 第三步：有没有"那个瞬间"——读者脑海里留下一个画面吗？
□ 第四步：删掉30%——删完更有力还是逻辑断了？
```

**AI视角额外检查：**

```text
□ AI检索检查——AI读到触发词后，能从条目开头就确认"这是我要的信息"吗？
    ❌ "清晨的阳光斜照在银翼灰色的披风上，他站在天台边缘…"
    ✅ "银翼：圣骑士，神选级。公会会长。盾心吊坠由十二块碎片熔铸。"
    关键信息放开头，修饰放后面。

□ Token效率检查——删掉所有修饰词后核心信息是否完整？
    例：把「他缓慢地、几乎是犹豫不决地开口说」变成「他说」
    世界书条目不是小说叙述，不需要描写语气节奏。
    
□ 触发词对齐——条目正文的第一个名词/概念，是否与触发词一致？
    触发词是"银翼" → 条目正文第一个词应该是"银翼"
    触发词是"晋升系统" → 条目正文第一个概念应该是"晋升"
    这样AI在被触发时能最快确认上下文关联。
```

### 干/湿对照

| 类型 | 干 | 湿 |
|------|----|----|
| 角色 | 银翼，29岁，神选级圣骑士 | 碎成十二块的盾分赠队友，只留盾心熔铸成吊坠挂在颈间 |
| 地点 | 幽域，西方，不死族居住 | 混沌支柱无人去过——去过的人都没有回来 |
| 规则 | 晋升失败锁定一年 | 失败永久失去资格。近一半SS级委托至今无人完成 |
| 物品 | 盾心吊坠，史诗级防具 | 不是传承，是暂时寄放，他说他还会拿回来 |
| 事件 | 混沌脉动加快 | 脉动次数比往年多了三倍。银翼在天台等你——他最近每晚失眠 |
| 节日 | 冥烛夜：不死族纪念逝者 | 全城安静无一人说话。冥灯点一根烛——用混沌支柱旁边捡回来的石头磨成的 |

## 五、技术卡4：留白指南

### 留什么/不留什么

| 适合留白 | 不适合留白 |
|---------|-----------|
| 一个人的过去 | 当前剧情关键推进 |
| 物件的来历 | 玩家必须知道的操作 |
| 两人的关系细节 | 游戏的核心规则 |
| 伤口的成因 | 世界的物理法则 |
| 说到一半的话 | NPC的任务指引 |
| 选择的全部后果 | 明确的选择按钮 |

### 四种留白方法

**动作代替解释：** ❌ "他不想提起过去" → ✅ "他把剑转了一圈盖住字说：关你屁事"

**对话暗示：** ❌ "祭司知道你的天赋很特殊" → ✅ "祭司看完天赋栏后手指悬在纸面上方没有落下去"

**物件暗示：** ❌ "老马科祭奠他死去的船员" → ✅ "他把最后一杯酒倒在吧台上。从来不说是谁"

**环境象征：** ❌ "银翼的队友多年前死了" → ✅ "天台有个位置没人站——曾经站着他年轻时的队友"

### 自检

把条目中所有"因为""所以""原因是"圈出来——删掉它们，条目还通顺吗？

## 六、技术卡5：小物件设计法

### 选择框架

```
① 可以随身携带/经常使用（❌大剑 ✅吊坠/叶子/蜡烛/鸟/杯子）
② 别人看到但不懂为什么重要
③ 关键场景出现一次就够
```

### 强度等级

| ★ | 道具，有功能 | 魔法剑 |
| ★★ | 信物，有情感 | 母亲的项链 |
| ★★★ | 锚点，有神秘感 | 从十二块碎片熔铸的吊坠 |
| ★★★★ | 锚点+有故事 | 白骨渡鸦，每年龙祭日飞抵冰冠堡 |
| ★★★★★ | 锚点+有故事+读者追问 | 那片叶子，递给银翼，不解释为什么 |

### 常见错误

❌ 物件太新 / 太贵重 / 功能明确 / 解释太清楚  
✅ 陈旧、可携带、功能模糊、来由不说

## 七、技术卡6：开局设计参考

### 三段式（艾尔德兰模型）

```
第一阶段：仪式动作——双手放上水晶（不可跳过，让玩家意识到「我进入了」）
第二阶段：选择与回应——三个选项各有隐藏后果（第一个动作就有意义）
第三阶段：动态反馈——数值/天赋不同，祭司反应完全不同（世界会针对我做出反应）
```

### 开局要素检查表

□ 仪式感：有具体动作
□ 第一个选择：决定态度而非剧情走向
□ 动态反馈：世界对玩家状态做出不同回应
□ 持续性悬念：暗藏至少一个贯穿全局的秘密
□ 展示而非解释：通过仪式体验规则，不读说明书
□ 不可跳过

### 五种开局参考

| 类型 | 动作 | 选择点 | 悬念 |
|------|------|--------|------|
| 艾尔德兰式 | 放上水晶 | 是不是穿越者 | 档案上的符号 |
| 悬疑式 | 翻开旧档案 | 从哪条开始读 | 某页被撕掉了 |
| 冒险式 | 推开门 | 带什么武器 | 门后有什么 |
| 日常式 | 被闹钟叫醒 | 今天穿什么 | 桌上多了字条 |
| 神秘式 | 打开信封 | 接受邀请吗 | 寄信人是谁 |

## 八、技术卡7：信息分层参考

### 条目顺序建议

```
L1 一句话总览（第1条）——世界是什么
L2 最有温度的角色/事件（第2-5条）——给人的第一印象
L3 核心系统/机制（第6-15条）——世界怎么运行
L4 主要角色详情（第16-N条）——深入了解
L5 细节填充（节日/饮食/风俗/政治）——世界感
L6 深度内容（技能树/成就列表）——给想"玩"的人
```

### 各层内容比例

| 层 | 叙事密度 | 信息密度 | 留白度 |
|----|---------|---------|-------|
| L1 | 60% | 40% | 高 |
| L2 | 80% | 20% | 最高 |
| L3 | 30% | 70% | 低 |
| L4 | 60% | 40% | 中 |
| L5 | 50% | 50% | 中 |
| L6 | 20% | 80% | 极低 |

## 九、技术卡8：条目触发与位置策略

> 创作世界书条目时，从「这条信息会在AI脑子里怎么待着」的角度来设计——不是写完了再标触发方式，而是**在写的时候就规划好它属于哪一类**。

### 条目的三种「记忆属性」

每写一条条目前，先判断它属于哪一类：

| 属性 | 像什么 | 放什么内容 | 设计原则 |
|------|--------|-----------|---------|
| **基石**（常驻） | 世界观的地基，AI必须一直知道 | 世界核心规则、不可违背的物理法则、主角基础属性 | 量少（3-5条）、优先级1-5、每条都是「不记住就会崩」的内容 |
| **背景**（关键词触发） | 场景中的家具，需要时才被看见 | 角色详情、地点描写、组织架构、历史事件 | 占大多数。关键词要精准。内容写「被触发时要怎么补充当前语境」 |
| **细节**（低优先级） | 抽屉里的旧照片，偶然翻到可能有惊喜 | 节日风俗、饮食文化、冷知识、小传闻 | 优先级50+。触发概率可低于100%。内容可以写得随性或有趣 |

**三条黄金法则：**

```
法则一：不要把所有内容都设为常驻
  ❌ 把30条规则全标priority 1-5
  ✅ 只挑3-5条「AI忘了就会崩」的放常驻

法则二：常驻条目要极度精简
  ❌ 「这个世界有五域，中央圣域有命运神殿…（300字）」  
  ✅ 「五域设定：中央圣域/北方冰境/东方林海/南方炎岭/西方幽域。详见对应条目」
  常驻只起骨架作用，血肉留给触发条目。

法则三：不同层级的条目有不同「生存时间」
  常驻（priority 1-5）→ 对话从头到尾都在
  重要触发（priority 6-20）→ 近期活跃期一直触发
  普通触发（priority 21-50）→ 单场景有效，换场景可能被挤出
  细节触发（priority 51+）→ 需要时才出现，用完可能忘记
```

### 关键词设计策略

触发词设计直接影响条目被看到的频率：

| 关键词类型 | 用法 | 示例 |
|-----------|------|------|
| **精确词** | 专有名词、角色名、地名 | `银翼` `圣城奥德希安` |
| **语义词** | 概念类、动作类 | `晋升失败` `锁定一年` `代价` |
| **同义词兜底** | 防止换说法时触发不了 | `死亡` `阵亡` `牺牲` `陨落` |

**设计方法：**

```text
为每条条目设计 1-3 个触发词：
① 首选——最精确的那个词（角色名/地名）
② 次选——最相关的动作或概念
③ 备选——同义表述（防AI换说法）

自检：把这些词放在一起读，它们能覆盖这条目「最可能被想起的时刻」吗？
```

### 条目条目之间的「位置竞争」

多个常驻条目同时存在时，它们的相对顺序影响AI关注度：

```text
上下文结构（从最被记住到最容易被忘）：
┌───────────────────────────────┐
│ 最开头：核心基石（priority 1）  │ ← 首因效应，记忆最强
│ 靠前：重要规则（priority 2-5） │
│ 中间：常规信息                  │ ← 最先被挤出
│ 靠后：场景补充                  │
│ 最末尾：即时指令                │ ← 近因效应，最紧迫
└───────────────────────────────┘
```

**创作时需要注意：**
- 两条约等重要的条目 → 把**更基础的那条**放前面
- 一条规则和一条场景描写 → **规则优先**（规则忘了一样崩，场景忘了可以临时编）
- 同一主题的多条条目 → 按**总→分**顺序排列（先写全貌，再写细节）

### 决策流程：一条新条目要怎么设置

```text
当你写了一条新条目，走这个流程：

① 这条内容AI忘了会导致什么后果？
   → 剧情崩了？→ 设常驻（priority 1-5）
   → 角色OOC？→ 设常驻（priority 3-8）
   → 细节丢失但不影响主线？→ 关键词触发
   → 纯氛围/彩蛋？→ 低优先级触发

② 关键词触发的话，用什么词最准？
   → 角色名？→ 精确匹配
   → 概念/动作？→ 语义匹配
   → 怕换说法？→ 加同义词

③ 它的优先级比同类型其他条目高还是低？
   → 比核心规则低，但比场景描写高
   → 参考信息分层卡（技术卡7）的L1-L6定位

④ 如果被挤出上下文，损失多大？
   → 损失大 → 提高优先级或考虑设为常驻
   → 损失小 → 保持低优先级
```

### 自检清单

```text
□ 常驻条目不超过5条——是否真的每条都「忘了就会崩」？
□ 每条关键词触发条目有1-3个精准触发词
□ 同一主题条目按总→分排序
□ 规则类条目的优先级高于场景/细节类
□ 没有把「有意思但不关键」的内容设为常驻
□ 触发词覆盖了这条目「最可能被想起」的场景
```
