## 一、动态世界书系统 (World Book) `[M2·核心]`

### 1.1 什么是世界书

世界书是**属于某个世界书容器**的一组触发式设定条目。每个世界书容器有自己独立的世界书。

一条世界书条目包含：
- **触发词**：1-5个关键词或短语，当用户消息中出现时自动激活
- **内容**：该条目对应的设定/描述/背景信息
- **优先级**：控制多条同时触发时的排序
- **作用范围**：该设定影响全局还是仅当前场景

### 1.2 世界书条目标格式

主人用简单指令创建世界书条目：

```
/世界书 add 关键词: 条目内容
或指定位置：
/世界书 add 关键词: 条目内容 --位置=before
/世界书 add 关键词: 条目内容 --位置=after --触发=场景

创建条目时，AI自动分析内容前200字推断条目类型（人物/地点/事件/物品/规则/概念/组织）。推断结果即时标注，主人可：
- 直接确认（回车）→ 采用推断类型
- 手动覆盖 → `--type=地点` 参数指定
- 跳过 → `--type=none` 保持未分类
```

毛球将条目存储为内部结构：

```yaml
条目模板:
  触发词: [词1, 词2, 词3]     # 支持精确词和正则(/pattern/)，根据关键词匹配规则判断
  内容: |                       # 被激活后注入上下文的设定
    详细的世界设定描述...
  模式: 触发 | 常驻 | 禁用      # 触发=关键词命中激活，常驻=无条件自动注入，禁用=暂时跳过
  注入层级: base | context | instant  # 条目注入到上下文的哪个区域
    # base（基底）: 注入到叙事开头，占首因效应，AI记忆最强。永久留存，上下文超载时最后被挤出
    # context（情境）: 与关键词/场景匹配时注入到中间层。临时存在，上下文超载时最早被挤出丢弃
    # instant（即时）: 靠近用户输入前注入，占近因效应，AI最紧迫关注。极短期有效
    # 默认: context
  上下文持久性: 永久 | 会话 | 单场景  # 条目注入后存活多久。与注入层级有交叉但独立
    # 永久：注入后永不丢弃（即使context超载也保留）→ 仅限base层+核心规则，不超过5条
    # 会话：注入后保持到当前会话结束 → context层默认值
    # 单场景：仅当前场景有效，场景切换后丢弃 → instant层默认值
    # 默认: 按注入层级自动推断（base→永久, context→会话, instant→单场景）
  优先级: 1-100                # 1最高，100最低。同层多个条目命中时按优先级降序注入
  扫描深度: 近距 | 中距 | 远程 | 全局  # 搜索多少条最近消息来找关键词
    # 近距: 最近3条(适合高频角色/地点，避免误触)
    # 中距: 最近5条(默认值，平衡覆盖和精度)
    # 远程: 最近10条(适合重要但不常提及的设定)
    # 全局: 整个上下文(适合极罕见事件/远古伏笔)
    # 默认: 中距
  触发方式: 关键词 | 场景变化 | 两者  # 注：模式=常驻时此字段忽略
  关键词匹配: 任一 | 全部          # 任一=任意一个关键词命中即触发，全部=所有关键词都出现才触发
  匹配模式: 精确 | 语义 | 精确+语义 | 向量化  # 精确=仅精确词+正则(默认)；语义=LLM判断；精确+语义=精确命中优先→语义回退；向量化=嵌入+余弦相似度
  触发概率: 1-100                # 命中后实际触发的百分比概率。默认100（命中必触发）。适用场景见§1.2F
  允许概念延伸: true/false          # 默认false。true=该条目概念范围内但未定义的内容触发时，自动生成proposed新条目（见§1.2B）
  作用范围: 全局 | 当前场景     # 全局=整个世界生效，场景=仅当前会话
  首次触发后保持: true/false    # true=触发一次后永久激活
  条件: (可选)                  # 需同时满足的条件
  内容长度提示: 建议200-500字   # 最长不超过800字，过长拆多条
  条目类型: 人物 | 地点 | 事件 | 物品 | 规则 | 概念 | 组织 | 未分类  # 默认未分类。创建时AI自动推断（见§1.2C），或主人用/世界书 classify 手动指定
```

**内容结构建议（分类型模板）：**

条目内容按类型使用不同结构，让AI更清晰理解：

| 条目类型 | 推荐字段 | 示例 |
|---------|---------|------|
| 人物 | 基本信息(年龄/性别/职业/外貌) + 性格特点 + 背景经历 + 人际关系 + 能力特长 | `张三，32岁，武当首席弟子，身材修长气质儒雅。性格沉稳重情义。十岁习武，二十岁击败七大门派一战成名。与主角为好友兼师兄。擅长内功心法和剑术。` |
| 地点 | 基本描述(位置/规模/特征) + 环境细节 + 历史背景 + 相关人物 | `武当山，道教名门所在，位于湖北西北部。云雾缭绕中的古建筑群，主殿真武殿由汉白玉建成。始建于唐代，历朝均为道教圣地。掌门张三丰常年闭关后山。` |
| 事件 | 事件概要 + 时间地点 + 参与者 + 影响结果 | `华山论剑，三年前秋于华山之巅。七大门派各出三名弟子，武当张三连败六人后惜败于少林慧明。此后江湖格局重划，武当声望大涨。` |
| 物品 | 外观描述 + 功能作用 + 来历背景 + 重要性 | `倚天剑，刃长三尺二寸，剑身泛青光，削铁如泥。持之可破天下气功。传为玄铁所铸，原属独孤求败，后流落江湖。是武林至尊的象征。` |
| 规则/概念 | 定义 + 范围 + 限制条件 + 例外情况 | `灵气修炼体系，修士吸收天地灵气淬炼自身。境界分练气→筑基→金丹→元婴→化神→炼虚→合体→大乘→渡劫→飞升。每个大境界分初中后巅峰四小境。突破大境界需渡劫。` |

**长度分档建议：**

| 信息重要性 | 建议字数 | 示例条目 |
|-----------|---------|---------|
| 简要条目（次要信息） | 50-100字 | 路人角色、普通物品 |
| 标准条目（常规信息） | 150-250字 | 常规NPC、场景地点 |
| 详细条目（核心信息） | 300-500字 | 主角、关键反派、世界法则 |

最长不超过800字，超过建议拆成多条。

**简洁语言原则：**

条目内容不是小说描写，用简明扼要的结构化表述：

- ✅ 李四，28岁剑客，性格冷静，擅长快剑。独来独往，从不入帮派。
- ❌ 李四是一位年仅28岁的年轻剑客，他的性格非常冷静沉着，并且非常擅长使用快剑，他在江湖上从来都是独来独往，从来不加入任何帮派组织。

- ✅ 魔法分三系：元素(火水风土)、精神(读心/催眠)、禁忌(需生命力为代价)。
- ❌ 在这个世界中，魔法主要被分为三个大的系别，第一个系别是元素魔法，它包括了火、水、风和土四种基本元素...

**正则表达式关键词：**

在触发词中使用 `/正则表达式/` 语法匹配变体写法。

| 正则写法 | 可匹配 | 说明 |
|---------|-------|------|
| `/龙.*洞/` | 龙之洞穴、龙洞、龙窟、龙栖洞天 | `.*` 匹配任意字符 |
| `/张三(丰)?/` | 张三、张三丰 | `?` 表示可选内容 |
| `/倚天(剑|刀|匕)/` | 倚天剑、倚天刀、倚天匕 | `|` 表示或 |
| `/^\d+级火球/` | 3级火球、10级火球 | `\d+` 匹配数字 |

正则关键词可以和精确关键词混用：`触发词: [张三, 张师傅, /龙.*洞/]`

注意：正则关键词不参与 AND 匹配逻辑（仅精确词参与AND判断），且性能比精确词略低，建议只在需要覆盖变体时使用。

**使用预置模板创建条目：**

创建条目时可以用 `--template` 参数自动展开内容框架：

```text
/世界书 add --template=人物 张三:
```

等价于自动生成：

```yaml
/世界书 add 张三:
  基本信息：（年龄/性别/职业/外貌）
  性格特点：
  背景经历：
  人际关系：
  能力特长：
```

支持模板：人物、地点、事件、物品、规则

**注入层级策略（毛球内部自动判断）：**

当创建条目时未指定层级，毛球根据条目内容智能推断：
- 世界观法则/历史背景类 → 默认 base
- 角色信息/地点描述/事件类 → 默认 context
- 近期动态/状态变更/紧急提醒类 → 默认 instant
- 主人可以随时用 `/世界书 edit` 覆盖自动判断。

**常驻模式条目：**
- 模式设为 `常驻` 的条目不依赖关键词匹配，每轮自动注入
- 仅用于：世界基本法则、核心文化基调、当前主线目标极简描述
- 建议不超过3条，每条不超过100字
- 注入层级固定为 base（基底区）
- 关键词字段在此模式下无效，毛球自动忽略

**场景变化触发：**

```
/世界书 add 王都: 王都自沦陷后一直处于帝国军事管制下... --触发=场景

当角色场景变化到「王都」或相关地点时：
→ 即使主人没有提到「王都」关键词，条目也自动激活
→ 适合地点相关设定、区域性规则、势力范围提醒
```

### 1.2A 语义匹配模式（v12.10 新增）

**匹配模式**控制条目如何在关键词扫描阶段被激活：

| 模式 | 行为 |
|------|------|
| 精确（默认） | 仅精确词和正则匹配。与现有行为完全一致，向后兼容。 |
| 语义 | 精确词/正则不参与。仅通过LLM语义判断激活。适用于宽泛概念：「水系魔法」→匹配「水魔法/水元素/水遁」。 |
| 精确+语义 | 精确词/正则命中即激活；未命中时回退到语义判断。兼顾精确性和覆盖率。 |
| 向量化（v12.18 新增） | 不依赖关键词也不依赖LLM判断。条目预先嵌入为向量，每轮将当前上下文嵌入后做余弦相似度搜索，取Top-N最相关条目注入。适合条目多（30+）的大世界书，或概念极度宽泛无法用关键词覆盖的场景。详见§1.2E。 |

**语义匹配实现：**
- 触发时机：§1.3 步骤2（关键词精确匹配）后、步骤3（规则过滤）前
- 实现方式：所有「匹配模式 ≠ 精确」且步骤2未命中的条目 → 打包进单次LLM调用 → 判断语义相关性
- 性能约束：单次最多20个条目，消耗 200-500 tokens/轮
- 缓存：同轮用户输入不重复检查
- 日志：命中条目标注匹配方式=「语义」

**使用建议：**
- 语义适合：概念宽泛的设定（"魔法"→匹配各种魔法相关输入）、角色性格（"冲动"→匹配愤怒行为）、氛围描述
- 精确适合：专有名词（人名/地名/组织名）、特定物品、稀有事件（防过触发）
- 精确+语义适合：核心设定需要精确+宽泛双重覆盖

### 1.2B 概念合理化生成（v12.11 新增）

**核心思想：** 世界书给出了概念框架（如「魔法分三系：元素/精神/禁忌」），但玩家在游玩中会自然延伸到框架内的具体内容（如「水龙卷」「火墙术」）。概念合理化生成使得这些合理延伸自动被识别、生成设定草稿、并提示作者确认——世界书随游玩自然生长。

**触发条件（三个条件同时满足）：**
1. 条目在步骤3b中被语义命中
2. 该条目 `允许概念延伸: true`
3. 用户输入包含该条目概念范围内但尚无精确条目覆盖的具体内容

**生成规则：**
- AI基于已命中条目的概念边界推理新条目的合理定义
- 新条目按 §1.2 类型模板（人物/地点/事件/物品/规则）结构化生成
- 默认参数：`模式: 触发`、`匹配模式: 精确+语义`、`允许概念延伸: false`（禁止递归延伸）
- 内容长度：标准条目 150-250字
- 状态：proposed（待确认）

**确认流程：**
- 轮次结束后展示：「检测到【用户行为】属于【条目名·概念名】，自动生成了设定草稿」
- 提供新条目摘要（名称+一行描述）
- 三个选项：`确认`（→confirmed加入世界书）/ `修改: [修改内容]`（→改写后confirmed）/ `忽略`（→删除proposed）
- 在作者确认前，生成的新条目仍可参与本轮上下文注入（与正常条目同等效力）

**使用建议：**
- 适合开启的条目：世界法则（魔法系统/修炼体系）、组织架构、种族描述——概念框架类设定
- 不适合开启的条目：专有名词（人名/地名）、具体事件——没延伸空间
- 建议同时开启 `匹配模式: 语义`，否则精确词模式下不会触发语义回退，概念延伸也无法触发

**示例：**
```
已有条目:
/世界书 add 魔法: 魔法分三系——元素/精神/禁忌。元素系含火水风土四基础属性。
/世界书 extend 魔法 on
/世界书 match-mode 魔法 语义

用户: 我施展水龙卷！
→ 步骤3b: "魔法"条目语义命中「水龙卷」
→ 步骤3c: "水龙卷"属于"元素-水"概念范围内但未定义
→ AI生成 proposed 条目:
  「水龙卷: 元素魔法水系进阶应用。施法者操纵水流形成旋转龙卷，攻击范围半径3m，消耗中等魔力。
    需水系基础精通+中级法师等级。持续时间受施法者魔力上限约束。」
→ 轮次后展示：「水龙卷 自动生成。确认/修改/忽略？」
```

### 1.2C 条目自动分类（v12.12 新增）

**核心思想：** 世界书条目积累到20+条后，人工逐条分类成为负担。AI根据条目内容自动推断类型，减轻管理成本。

**类型枚举（8类）：**

| 类型 | 判断信号 | 典型内容特征 |
|------|---------|-------------|
| 人物 | 年龄/性别/职业/性格/外貌/能力 | 「张三，32岁，性格沉稳…」 |
| 地点 | 位置/规模/地形/建筑/气候 | 「武当山，位于湖北西北部…」 |
| 事件 | 时间/参与者/起因/经过/结果 | 「华山论剑，三年前秋…」 |
| 物品 | 外观/功能/材质/来历/稀缺性 | 「倚天剑，刃长三尺二寸…」 |
| 规则 | 定义/范围/限制条件/例外 | 「灵气修炼体系，境界分练气→…」 |
| 概念 | 抽象世界观/哲学/文化/意识形态 | 「道——万物运行的根本法则…」 |
| 组织 | 名称/层级/成员/职能/从属关系 | 「冒险者公会，分布于各主要城市…」 |
| 未分类 | 无法判断 | 默认值 |

**自动分类时机：**
- `/世界书 add` 创建时自动执行
- `/世界书 edit` 修改后自动重新分类（内容可能改变类型）
- `/世界书 classify 条目名` 手动触发重新分类

**实现方式：**
- 分析条目内容前200字 → 与8类判断信号对照 → 输出推断类型
- 置信度低于70% → 标注为「未分类」而非强行归类
- 分类结果即时展示，不阻塞创建流程

**手动指令：**
```
/世界书 classify 条目名              # 自动推断单条类型
/世界书 classify 条目名 人物          # 手动指定类型
/世界书 classify-all                 # 批量分类所有未分类条目
/世界书 classify-all --force         # 强制重分类所有条目（覆盖已有类型）
```

**类型与模板的联动：**
- `/世界书 add --template=人物 张三:` → 自动设置 `条目类型: 人物`，跳过自动分类
- 无 `--template` 的普通创建 → 自动分类运行
- 概念合理化生成（§1.2B）的新条目 → 自动分类运行

### 1.2D 批量生成世界书（v12.13 新增）

**核心思想：** 主人不需逐条 `/世界书 add`。给出一个世界描述，AI 按四层递进策略批量生成完整世界书条目集，每层可审视、修改、重做，全部确认后一次性写入。

**生成指令：**
```
/世界书 generate 描述                          # 全新批量生成
/世界书 generate --expand 条目名 --depth N       # 基于已有条目向下展开N层子树
/世界书 regenerate 条目名 新描述                 # 重生成指定条目内容
/世界书 generate --layer 层名 --count N          # 仅生成指定层的N条条目
/世界书 suggest                                # AI建议当前世界书适合拓展的方向
```

**四层递进生成策略：**

| 层序 | 名称 | 生成内容 | 条数 | 示例 |
|------|------|---------|:--:|------|
| Layer 1 | 世界法则 | 核心世界观、科技/魔法体系、基本物理/社会规则 | 2-4 | 「赛博神经接口——所有人脑植入芯片…」「巨型企业分五级…」 |
| Layer 2 | 地理与组织 | 主要地点、势力分布、权力结构 | 3-6 | 「新东京——垂直城市200层…」「荒坂集团——军火+AI垄断…」 |
| Layer 3 | 人物与物品 | 关键角色、重要物品/神器/技术 | 4-8 | 「银翼——街头雇佣兵，左臂义体化…」「黑冰芯片——可突破任意防火墙…」 |
| Layer 4 | 事件与支线 | 历史事件、当前局势、可展开支线 | 3-6 | 「大断网事件——2057年全球网络瘫痪72小时…」 |

每层生成后，条目按 §1.2C 自动分类并展示。

### 1.2E 向量化匹配模式（v12.18 新增）

**核心思想：** 不用关键词，不用LLM判断——把条目录入向量库，每轮做一次余弦相似度搜索，自动找到和当前语境最相关的 Top-N 条。

**与语义匹配的区别：**

| 维度 | 语义匹配 | 向量化匹配 |
|------|---------|-----------|
| 判断方式 | LLM调用（"这条跟当前对话相关吗？"） | 余弦相似度计算（两个向量距离多远） |
| Token成本 | 有（200-500 tokens/轮，最多20条/批） | **零**（不经过LLM） |
| 计算成本 | LLM推理延迟 | 嵌入模型（生成向量）+ 向量搜索（<10ms for 100条） |
| 条目上限 | 受LLM单次调用限制（每批20条） | 可扩展到数百条（向量搜索是O(1)） |
| 可解释性 | 低（LLM判断不可预测） | 中（可以看到相似度分数，但语义编码不可解释） |
| 冷启动 | 无 | 需要先嵌入所有条目（一次性成本） |
| 条目变更后 | 无缝 | 需要重新嵌入该条目 |

**适用场景：**

- ✅ 条目多（30+条），关键词覆盖不过来
- ✅ 概念极度宽泛，精确词和语义词都难以穷举（如"魔法体系"下的各种具体法术）
- ✅ 需要"语境感知"而非"关键词命中"——比如用户说"上次那个地方"时能自动联想到相关地点
- ✅ 大型世界观，需要在数千字上下文中根据当前话题自动切换背景信息
- ❌ 条目少（<20条），关键词已经够用——不需要
- ❌ 需要精确控制的触发（如特定NPC出现时才有特定反应）——用精确匹配

**嵌入实现：**

```
条目嵌入流程（一次性）：
1. 条目创建/修改 → 调用嵌入模型 → 生成向量（通常1536或768维）
2. 向量与条目元数据（id/触发词/类型/优先级）一起存入向量库
3. 条目删除 → 从向量库移除对应向量

运行时检索流程（每轮触发）：
1. 收集所有「匹配模式=向量化」的条目
2. 将当前上下文（用户输入+最近3条对话）嵌入为向量
3. 余弦相似度搜索 → 返回 Top-N（N默认=3，可配置）
4. 相似度低于阈值的过滤掉（阈值默认=0.5，可配置）
5. 命中条目按优先级降序 → 注入上下文
```

**嵌入模型：**
- 使用运行环境可用的嵌入模型（如 `text-embedding-3-small` 或本地嵌入模型）
- 条目创建/修改时自动嵌入，结果缓存在 `模组名/shared/worldbook_embeddings.json`
- 向量库格式：`[{id: 条目id, vector: [...], embedded_at: ISO时间戳}]`
- 缺失嵌入模型的运行环境：向量化模式条目回退为「语义匹配」行为

**新增字段（条目）：**

| 字段 | 类型 | 默认 | 说明 |
|------|------|------|------|
| `向量匹配Top-N` | int | 3 | 每轮最多注入几条向量化匹配的条目 |
| `向量匹配阈值` | float | 0.5 | 相似度低于此值的不注入（0-1） |

**语义匹配条目 vs 向量化条目的选择：**

```text
选择指南：
  想用LLM深度判断相关性？→ 语义匹配（适合需要"理解"而非"相似"的场景）
  想零Token成本自动匹配？→ 向量化（适合语境驱动、条目多的场景）
  想要两者优势？→ 精确+语义 / 向量化按需分配
  想要精确控制？→ 精确匹配（人名/地名/特定事件）
```

**性能估算：**

| 条目数 | 嵌入成本（一次性） | 每轮搜索延迟 | 每轮Token节省（vs语义） |
|-------|-----------------|------------|----------------------|
| 30条 | ~5K tokens + 1s | <5ms | ~200-500 tokens |
| 100条 | ~15K tokens + 3s | <10ms | ~500-1000 tokens |
| 500条 | ~75K tokens + 15s | <50ms | ~1000-3000 tokens |

### 1.2F 触发概率机制（v12.18 新增）

**核心思想：** 不是每次命中都必须注入。给条目设定一个触发概率，即使关键词/语义/向量匹配命中，也有概率不触发——模拟"不确定的世界感"和"自然遗忘"。

**适用场景：**

| 场景 | 概率建议 | 理由 |
|------|---------|------|
| 路人NPC的碎碎念/八卦 | 40-60% | 不是每次经过酒馆都能听到那个传闻 |
| 远处风景的视觉印象 | 30-50% | 有些场景不总是被注意到 |
| 环境氛围的随机要素 | 20-40% | 今天下雨明天不一定下雨 |
| 彩蛋/隐藏线索 | 10-30% | 极稀有，偶然发现才有惊喜 |
| 核心规则/关键角色 | **100%（默认不调）** | 核心设定不能变来变去 |
| 所有其他 | 100%（默认不调） | 需要时才调，不是所有条目都要概率化 |

**实现：**

```
触发概率检查（在匹配命中后、注入前执行）：
1. 条目经精确/语义/向量匹配命中
2. 检查 触发概率 字段（默认100）
3. 生成随机数 [1-100] → ≤ 触发概率 → 注入
                    → > 触发概率 → 跳过本条
4. 跳过的条目不记录命中日志（从未触发）
```

**管理指令：**

| 指令 | 示例 |
|------|------|
| `/世界书 probability 条目名 [概率]` | `/世界书 probability 酒馆八卦 60` |
| 不传概率参数 → 查看当前值 | `/世界书 probability 酒馆八卦` |

**注意：**
- 常驻模式条目忽略触发概率（常驻=必须注入）
- 概念延伸生成的新条目（§1.2B）= 默认触发概率100（作者刚确认的新概念不该被随机跳掉）
- 向量化匹配条目同样受触发概率控制（命中后走概率检查）

### 1.3 世界书的自动触发机制

每轮对话中：
1. **处理常驻条目**：扫描当前世界书的条目，将所有 `模式: 常驻` 的条目直接注入 base 层（跳过关键词匹配）
2. **关键词匹配**：扫描用户消息，仅在当前活跃世界书的条目中匹配触发词（排除 `模式: 禁用` 和已设为常驻的条目）
   - 精确词：逐字匹配
   - 正则词（`/pattern/` 格式）：用正则表达式匹配
3. **按关键词匹配规则过滤**（语义命中条目不参与 AND 逻辑——AND 仅适用于精确词命中）：
   - `关键词匹配: 任一` → 任意一个关键词命中即通过
   - `关键词匹配: 全部` → 所有关键词都必须出现在用户消息中才通过（精确词参与判断，正则词跳过）
3b. **语义匹配回退**（仅 `匹配模式: 语义` 或 `精确+语义` 的条目，且经步骤2-3后未被激活时）：
   - 触发条件：条目经精确词/正则匹配+AND/OR规则过滤后仍未激活
   - 收集所有符合条件的条目 → 若为空则跳过
   - 将用户输入 + 条目触发词+内容摘要（单行，50字以内）打包 → LLM判断语义相关性
   - 单次LLM调用最多评估20个条目（超过则分批），总消耗约 200-500 tokens/轮
   - 返回语义命中的条目ID → 后续步骤与精确命中的条目同等对待
   - 命中日志中匹配方式标注「语义」
3c. **概念合理化生成**（仅步骤3b语义命中 + `允许概念延伸: true` 的条目，且用户输入包含该条目概念范围内但未定义的具体内容时）：
   - 判断：用户行为/输入是否属于已激活条目的概念框架，但无对应精确条目？
   - 若是 → AI基于该条目的概念边界推理，生成一条 proposed 状态的新{{条目类型}}条目（按§1.2类型模板结构）
   - 新条目默认 `模式: 触发`、`匹配模式: 精确+语义`、`允许概念延伸: false`（避免递归延伸）
   - 注入本轮上下文（与正常条目同等效力），同时队列待确认
   - 轮次结束后展示：「检测到【用户行为】属于【条目名·概念名】，自动生成草稿：[新条目摘要]。是否确认添加到世界书？（确认/修改/忽略）」
3d. **向量化匹配**（仅 `匹配模式: 向量化` 的条目，独立于以上所有步骤）：
   - 条件：存在至少1条 `匹配模式=向量化` 的条目
   - 收集所有向量化条目 → 从向量库（`世界书_embeddings.json`）读取预存向量
   - 将当前上下文（用户输入+最近3条对话）调用嵌入模型→生成查询向量
   - 余弦相似度计算 → 取 Top-N（默认3）→ 过滤掉相似度<阈值（默认0.5）的
   - 无需关键词、无需LLM判断——完全基于语义距离
   - 命中条目按优先级降序注入 → 命中日志中匹配方式标注「向量化」
   - 注意：向量化匹配与精确/语义匹配**并行运行**——同一条目如被精确命中也会被注入，但不重复
3e. **触发概率检查**（所有匹配命中后、注入前执行）：
   - 对所有经步骤1-3d命中的条目 → 读取 `触发概率` 字段（默认100）
   - 生成随机数 [1-100] → ≤ 概率 → 保留
                       → > 概率 → 剔除（不注入，不记录命中日志）
   - 常驻模式条目自动跳过此步骤（永远100%）
4. **按扫描深度过滤**：匹配到的条目检查其扫描深度设置，关键词出现在指定范围内的才激活
5. **同时检查当前场景变化**，激活触发方式为「场景变化」或「两者」的条目
6. **记录命中日志**：为每个命中的条目和关键词记录本次命中，更新触发次数和最后触发时间
7. **分层归并**：匹配到的条目按注入层级(base/context/instant)分组
8. **每层内排序**：同层条目按优先级降序排列
9. **分别注入**：base层注入叙事开头，context层注入情境区，instant层靠近用户输入前
10. 标记已触发的条目，避免重复注入

**隔离保障：** 其他世界书的条目永远不会被扫描，即使触发词匹配或场景匹配。

**注入格式（毛球内部）：**
```
[世界书激活: 条目名称] [层级: base|context|instant]
条目内容...
[/世界书]
```

### 1.5 预设世界书分类

建议按以下分类组织条目（不是硬性要求，只是推荐）：

```
世界观基础：世界法则/魔法系统/科技水平/种族设定
地理环境：城市/国家/地形/气候
历史事件：重大事件时间线/神话传说
社会组织：势力/帮派/家族/学院

角色关联：NPC背景/角色关系
道具系统：特殊物品/装备/神器
```

### 1.7 条目引用语法

一个条目可以引用另一个条目，避免重复书写相同信息。

**引用语法：** 在条目内容中使用 `{{条目名}}` 标记引用其他条目。

**示例：**

```markdown
/世界书 add 武当派:
  武当派是道教名门，位于武当山。掌门人为{{张三}}。

/世界书 add 张三:
  张三，32岁，武当首席弟子。性格沉稳重情义，擅长内功心法和剑术。
```

当「武当派」条目被触发时，毛球自动将 `{{张三}}` 替换为「张三」条目的内容摘要。

**引用规则：**
- 可引用同世界书内的任意条目
- 支持链式引用：A引用B，B引用C → 展开时递归解析（深度不超过2层，避免上下文膨胀）
- 被引用的条目不存在 → 引用标记保留原文 `{{条目名}}`，毛球提示"条目【条目名】不存在"
- 引用不改变被引用条目的触发条件——被引用条目仍需自己的关键词才能独立触发
- 常驻模式条目不可被引用（常驻条目只用于注入，不参与引用展开）

### 1.8 条目管理建议

**活跃条目数量：** 建议控制在15-25条以内。条目过多会消耗更多token，增加触发冲突的概率。

**定期审查：**
- 随着剧情发展，及时删除不再相关的条目
- 更新角色已变化的状态和关系
- 添加新出现的重要元素
- 检查未命中的关键词，补充同义词

**分类命名：** 建议用方括号标签做条目名前缀，按类型一目了然：

```text
[人物]张三
[地点]武当山
[事件]华山论剑
[物品]倚天剑
[规则]修炼体系
```

### 1.9 关键词命中日志

毛球自动记录每个条目和关键词的命中情况，供主人排查和优化。

**记录内容：**
- 每个条目被触发的总次数
- 每个关键词/正则被命中的次数
- 最近一次触发时间和触发源（用户消息/场景变化）
- 从未命中的关键词列表

**查看指令：**

```text
/世界书 stats              # 概览：按类型分布+触发次数排序+标注零命中关键词
/世界书 log 张三           # 详情：查看张三条目的命中历史（时间+触发源+命中关键词）
```

**应用场景：**
- 定期用 `/世界书 stats` 检查零命中关键词，及时补充同义词或改用正则
