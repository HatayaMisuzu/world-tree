# M19 角色卡情绪惯性系统

> world-tree v12.19 创新升级模块。定位：角色卡模式的隐性关系与情绪轨道。

## 设计原则

角色卡模式禁止展示好感度、关系等级和数值面板。情绪惯性系统只维护内部叙事信号，用来保证角色关系变化有连续性。

## 五条隐性轨道

| 轨道 | 初始 | 升温 | 紧张 | 修复 |
|------|------|------|------|------|
| 称呼 | 全名/你 | 名字/专属称呼 | 疏离称呼 | 恢复柔和称呼 |
| 距离 | 保持距离 | 不介意靠近 | 主动拉开 | 允许短暂靠近 |
| 主动性 | 等 user 开口 | 主动找话题 | 回避/沉默 | 试探性回应 |
| 秘密 | 不透露 | 透露小事 | 收回边界 | 承认一部分 |
| 身体动作 | 克制 | 小动作变多 | 动作僵硬 | 放松但不直说 |

## 状态格式

建议写入 `branches/<active_branch>/character_card/inertia.json`：

```json
{
  "character": "艾琳",
  "current_tone": "熟悉但不亲密的暧昧",
  "signals": {
    "address": "名字",
    "distance": "不介意靠近",
    "initiative": "会找话题",
    "secret": "透露小事",
    "gesture": "会看向 user"
  },
  "last_shift_reason": "user 记住了她不喝甜咖啡"
}
```

## 硬规则

- 不向 user 输出任何数值或阶段标签。
- 关系变化必须由事件触发，不得无因突变。
- 紧张后不能一轮内完全恢复，除非 user 明确处理冲突。
- 情绪惯性只影响语气、动作、称呼和透露程度，不改写 confirmed 事实。
