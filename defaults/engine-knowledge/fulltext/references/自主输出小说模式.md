# 世界树引擎 — 自主输出小说模式

> 当前状态：历史归档。v12.19 当前运行入口为 `自动推进模式.md`；本文不作为运行入口，不得覆盖 `/推进` 当前契约。

> 从互动叙事到全书自动生产：接收世界树设定/角色/预设，按章节自动输出小说 v1.0
> 调研基础：novel-writer-skills (833★) + oh-story-claudecode (1.5k★) + AI-Novel-Writing-Assistant (1.4k★) + NovelForge (868★)

---

## 设计总纲

### 两种模式

```
world-tree 引擎
├── 互动模式（现有）         ← 用户自由发挥，引擎执行规则
│   └── 每轮用户输入 → 引擎审核 → 叙事回复
│
└── 自主输出模式（新增）     ← 用户设定好世界，引擎自主写小说
    └── 接收设定 → 规划章节 → 逐章输出 → 自动质量检查
```

### 核心区别

| | 互动模式 | 自主输出模式 |
|------|---------|------------|
| 驱动方式 | 用户每轮输入 | 引擎自主推进 |
| 输出单位 | 每轮1-3段叙事 | 每轮1个完整章节 |
| 世界状态 | 实时随用户行为变化 | 按章节规划执行 |
| 子agent | W+C+S并行分析 | W+C+S + 大纲规划 agent |
| 用户角色 | 主角/操控者 | 设定者/审稿人 |
| 节奏控制 | 用户自由 | 引擎按情绪曲线控制 |

## 一、自主输出工作流（五阶段流水线）

```
阶段1: 设定      阶段2: 规划      阶段3: 写作      阶段4: 检查      阶段5: 完成
[世界书+角色]  → [章节大纲+      → [逐章输出]   → [一致性检查]  → [全书完稿]
[场景+预设]      情绪曲线+                    [去AI味]
                 线索分布]
```

### 阶段1：设定摄入 (Setup Intake)

从互动模式或直接创建接收：
- ✅ 世界书条目（M1+M2 触发式条目）
- ✅ 角色卡（M8 含声纹/外貌/性格/动机）
- ✅ 初始场景（M11）
- ✅ 风格预设（M12 normal/tense/cozy/epic/horror）
- ✅ 世界状态基线（M3）
- ✅ 组织实体（M4-M7）
- ✅ 魔法/物理规则（M15a/M15b）

用户指令：
```
/小说 开始                     ← 从当前互动状态转自主输出
/小说 从设定 [设定文件名]       ← 从文件加载设定
/小说 设定补充 [内容]           ← 补充额外设定
```

### 阶段2：规划 (Planning) — 基于 novel-writer-skills plan.md

引擎自动生成：

#### 2.1 章节大纲
```
/小说 规划 → 输出：

═══════ 章节大纲 ═══════

预计总章数: 30章
每章字数目标: 2000-3000字
总字数目标: 6-9万字

结构: 三幕结构
  第一幕（1-8章）: 建立世界 → 引入冲突 → 第一个转折
  第二幕（9-22章）: 升级冲突 → 中点反转 → 最大危机
  第三幕（23-30章）: 反击 → 高潮 → 收束

═══════════════════════
```

#### 2.2 情绪曲线（借鉴 novel-writer-skills 情绪曲线设计）

```
/小说 情绪曲线 → 输出：

情绪类型: 😤爽点 / 😭虐点 / 🤔悬念 / 💧平缓

第1-3章: 虐→爽→悬念 (强度: 中→高→中) 目标: 开局抑扬，建立追读欲
第4-8章: 平缓→虐→爽 (强度: 低→中→高) 目标: 第一波小高潮
第9-15章: 悬念→虐→爽 (强度: 中→高→高) 目标: 第二波高潮
...

情绪设计原则:
✅ 欲扬先抑: 爽点前适度铺垫虐点
✅ 张弛有度: 避免连续5章以上平缓
✅ 悬念驱动: 每章结尾留悬念
```

#### 2.3 线索分布规划

| 章节段 | 内容 | 关键事件 | 活跃线索 | 伏笔操作 |
|--------|------|---------|---------|---------|
| 1-3章 | 开局 | 主角获得能力 | PL-01⭐⭐⭐ | F-001埋设 |
| 4-8章 | 展开 | 第一次对抗 | PL-01⭐⭐ PL-02⭐⭐⭐ | F-001推进, F-002埋设 |
| 9-15章 | 反转 | 发现真相 | PL-02⭐⭐⭐ PL-03⭐⭐ | F-001回收 |

### 阶段3：写作 (Writing)

#### 3.1 每章输出格式

```
/小说 写第N章 → 输出：

═══════ 第N章: [章节标题] ═══════

[开篇钩子 - 1-2句悬念/冲突直接切入]

[正文 - 场景-续场结构]
场景1: 目标→冲突→灾难
续场1: 情绪→困境→决定  
场景2: 目标→冲突→成功

[章末钩子 - 1句悬念驱动下一章]

═══════ 第N章完 ═══════
字数: XXXX字
线索推进: PL-01→PL-02
伏笔: F-001埋设 / F-003推进
情绪峰值: 爽点(高)
下一章预告: [一句话]

用户操作:
  /继续    — 写下一章
  /改 N    — 修改本章
  /调整方向 — 调整后续规划
```

#### 3.2 章节写作引擎规则

借鉴场景-续场模型 (novel-writer-skills scene-structure SKILL.md):

**每章内部结构:**
```
[开篇钩子] (1段, ~30字)
  └─ 黄金开篇法则: 从冲突/动作/对话切入
  └─ 禁止静止场景/大段环境描写

[场景A] (~800字)
  ├─ 目标: POV角色想要什么（具体/可实现/紧迫）
  ├─ 冲突: 什么阻止他们（外部+环境+内部+时间，2-3种）
  └─ 灾难/成功: 场景如何结束

[续场A] (~300字)
  ├─ 情绪: 角色反应（身体表现，不直说）
  ├─ 困境: 两个不好选择的权衡
  └─ 决定: 选择 → 成为下个场景目标

[场景B] (~800字)
  └─ 同上结构

[章末钩子] (1段, ~30字)
  └─ 悬念/未解问题/情感牵引
```

**每章写作时自动激活的Skill:**
| 检查点 | 检查内容 |
|--------|---------|
| 对话 | 潜台词>直白 / 角色声音独特 / 无信息倾倒 |
| 场景 | 目标-冲突-灾难 / 张力上升 / 感官锚点 |
| 叙事 | 无AI味 / 视角一致 / 情绪递进 |
| 线索 | 本章线索推进状态 / 伏笔埋设/回收 |
| 一致性 | 角色特征/世界规则/时间线 |

#### 3.3 去AI味机制（借鉴 oh-story-claudecode）

```
每段输出后自动检查AI味:

❌ AI味特征 → 自动改写
  - "在这个充满...的世界里" → 删除，直接写
  - "与此同时" / "然而" / "最终" 开头 → 减少50%
  - "他意识到..." → 改为具体动作/想法
  - 大段环境描写开头 → 重写为动作/对话切入
  - 每段以"他/她"开头 → 变化句式
  - "不禁" "不由得" "突然" → 每500字限1次

✅ 人味特征 → 自动增强
  - 具体感官细节（视觉+听觉+触觉+嗅觉）
  - 不完整句子（情绪激动时）
  - 角色特有的口语/方言
  - 具体物品/动作替代抽象描述
```

### 阶段4：质量检查 (Quality Check)

#### 4.1 每章自动检查（借鉴 consistency-checker）

```
第N章写完后自动运行:

🔴 关键检查:
  - 角色特征是否与已建立事实一致？
  - 时间线是否前后矛盾？
  - 世界规则是否被违反？

⚠️ 警告检查:
  - 角色言语模式是否改变？
  - 物理距离/地理位置是否合理？
  - 线索推进是否符合规划？

📝 注意检查:
  - 对话是否每个角色声音独特？
  - 场景是否包含了感官锚点？
  - 章末是否有钩子驱动下一章？

本章质量得分: XX/100
```

#### 4.2 章节间连贯性检查

| 检查项 | 方法 |
|--------|------|
| 情节连贯 | 上一章结束状态 vs 本章开始状态 |
| 情绪连贯 | 情绪曲线规划 vs 实际实现 |
| 线索连贯 | 线索追踪表 vs 本章实际推进 |
| 伏笔连贯 | 已埋伏笔 vs 本章是否遗忘 |

### 阶段5：完成与导出

```
/小说 完成
→ 全书一致性终检
→ 生成目录
→ 导出为完整文件
→ 保存到 branches/<active_branch>/archive/
```

## 二、新增指令

| 指令 | 功能 |
|------|------|
| `/小说 开始` | 从当前互动状态转入自主输出模式 |
| `/小说 开始 --from 模组名` | 从已有模组设定启动自主输出 |
| `/小说 设定` | 查看/编辑当前自主输出的设定 |
| `/小说 规划` | AI自动生成章节大纲+情绪曲线 |
| `/小说 规划 show` | 查看当前章节规划 |
| `/小说 规划 adjust [调整内容]` | 调整规划 |

| `/小说 批量写 N-M` | 连续输出N到M章 |
| `/小说 改 [章节号]` | 修改某章（提供修改方向） |
| `/小说 检查 [章节号]` | 对某章运行质量检查 |
| `/小说 检查 all` | 对所有已完成章节运行检查 |
| `/小说 去AI味 [章节号]` | 对某章做去AI味改写 |
| `/小说 导出` | 导出全书为完整文件 |
| `/小说 退出` | 退出自主输出模式 |
| `/小说 status` | 查看自主输出状态 |

## 三、与现有系统的集成

### 与 world-tree 模块的集成

| 模块 | 自主输出模式中的角色 |
|------|-------------------|
| M1-M2 世界书 | 提供世界观基线（不变化） |
| M3 世界状态 | 按章节规划推进（不再等待用户输入） |
| M4-M7 组织 | 组织动态随章节自动推进 |
| M8-M9 角色 | 角色状态按章节变化 |
| M11 场景 | 场景卡按章节大纲切换 |
| M12 风格 | 预设控制每章写作风格 |
| M13 叙事 | 五层引擎每章自动应用 |
| M15 规则 | 每章输出后自动审核 |
| M16 时间 | 时间按章节自动推进 |
| M17 随机 | 可关闭（确定性输出）或保留（情节微调） |

### 与 Tracking 追踪系统的集成

每章输出后自动记录追踪数据：
- `change-log.jsonl` — 世界状态/角色状态变更
- `foreshadowing.json` — 本章埋设的伏笔+线索推进
- `conflicts.json` — 检测到的不一致
- `checkpoints.json` — 章节完成标记
- `parameter-snapshots.json` — 每5章保存参数快照

### 与时间树分支系统的集成

每章完成后自动创建检查点节点：
```
timeline-tree:
  root
   └─ [Chapter 1 complete]
       └─ [Chapter 2 complete]
           └─ [Chapter 3 complete]  ← 用户可在此分叉
               ├─ 继续主线
               └─ 从这里分出新分支（修改后续大纲）
```
