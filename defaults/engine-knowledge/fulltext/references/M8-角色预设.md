## 二、角色预设系统 (Character Preset) `[M8·核心]`

> **⚠️ v12.19重要区分：M8角色预设 ≠ 角色卡驱动模式（M19）。**
> - M8角色：绑定于世界书内，由 `/角色 create` 创建，是世界书驱动模式下的角色系统
> - 角色卡（M19）：独立输入类型，不绑定世界书，通过加载JSON/PNG文件启动，是完全独立的驱动模式
> - 两者格式、用法、模块行为均不同，不可混用。详见 `references/M19-角色卡驱动.md`

### 2.1 什么是角色预设

借鉴SillyTavern的角色卡格式（PNG+嵌入JSON），毛球引擎使用纯文本角色卡。
每张角色卡包含角色的完整定义，可以随时切换/加载/创建。

**v12.0新增：** 支持导入SillyTavern标准角色卡（PNG格式或JSON格式），自动转换字段映射。
也支持将本引擎角色卡导出为ST兼容格式。

### 2.2 角色卡标准格式

**格式一：本引擎原生格式（完整功能，兼容全部M模块）**

```yaml
角色卡模板:
  名称: 角色全名
  别名/称号: 
  年龄: 
  性别: 
  种族: (人类/精灵/兽人/...)
  
  所属世界书: "世界书名称或ID"    # ★必须字段★ 角色绑定到唯一世界书
  所属组织: ["组织名"]              # v4.0新增：角色隶属的组织
  组织职位: "银级冒险者"            # 在该组织中的身份
  组织声望: 75/100                  # 在组织内部的声望值
  
  外貌:
    发型:
    瞳色:
    身高:
    体型:
    标志穿着:
    特殊标记: (伤疤/纹身/饰品)
  
  性格:
    核心特质: (3-5个关键词，如: 倔强/温柔/正义感强/嘴硬)
    说话风格: (句式偏好/语气词密度/语速)
    隐藏面: (不为人知的一面)
    矛盾/反差: (看似矛盾但统一的特质，如: 冷血杀手每天喂流浪猫)
    底线/禁忌: (什么事绝对不会做)
    致命缺陷: (在关键时刻会导致失败的弱点，如: 傲慢→低估对手、逃避→问题滚雪球)
  
  背景:
    出身: 
    当前身份:
    核心经历: (3-5句关键人生事件)
    当前目标:
    
  能力:
    擅长:
    不擅长:
    特殊技能:
    
  关系:
    (列出与各主要角色的关系/亲密度/态度)
    (进阶·关系弧线: 可标注关系变化轨迹，如: 陌生人→不情愿盟友→信任→考验→真正的友谊)
  
  首次对话: (角色第一次出场时的典型台词)
  
  对话示例: |
    (2-3段体现角色说话方式的示例对话)
```

**格式二：SillyTavern兼容格式（v12.0新增，精简字段，重点移植）**

毛球引擎与ST角色卡(v2/v3)的字段映射：

```
ST字段                 →  本引擎字段
─────────────────────────────────────────────
name                   →  名称
description            →  外貌 + 性格·核心特质 (合并)
personality            →  性格 (全文保留)
scenario               →  背景·当前身份 + 场景上下文
first_mes              →  首次对话
mes_example            →  对话示例
creator                →  保留为元数据
system_prompt           →  保留注入角色卡头部
alternate_greetings     →  保留为备选开场白
```

**导入ST角色卡：**
- PNG格式：读取PNG中嵌入的JSON tEXt chunk → 自动字段映射 → 创建原生角色卡
- JSON格式：直接解析 → 字段映射 → 需手动补充缺失字段（所属世界书/组织等）
- 导入后角色卡默认为本引擎原生格式，补充字段后可完全使用M模块功能

**导出为ST格式：**
- `/角色 export 名称 --格式=ST` → 生成ST兼容JSON
- `/角色 export 名称 --格式=ST-PNG` → 生成嵌入JSON的PNG（需角色头像图片）

### 2.3 角色卡管理指令

| 指令 | 功能 | 示例 |
|------|------|------|
| `/角色 create 名称 [世界书名]` | 创建新角色卡（绑定到世界书） | `/角色 create 艾琳 艾瑟琳的奇幻大陆` |
| `/角色 list` | 按世界书分组列出所有角色 | |
| `/角色 list 世界书名` | 列出指定世界书的角色 | `/角色 list 赛博东京` |
| `/角色 show 名称` | 查看角色详情（含所属世界书） | |
| `/角色 use 名称` | 切换当前操控角色（自动切换到角色所属世界书） | `/角色 use 艾琳` |
| `/角色 edit 名称/字段: 内容` | 修改角色某字段 | `/角色 edit 艾琳/性格: 变得更加阴沉` |
| `/角色 delete 名称` | 删除角色卡 | |
| `/角色 link 角色A+角色B: 关系描述` | 建立角色关系（仅限同一世界书内） | `/角色 link 艾琳+团长: 艾琳暗恋团长但不敢说` |
| `/角色 export 名称` | 导出角色卡为文件 | |
| `/角色 load 文件名` | 从文件加载角色卡（需指定所属世界书） | |

**重要变更：**
- 创建角色必须指定所属世界书。如果当前有活跃世界书，自动绑定到当前世界书。
- `/角色 use` 会自动触发 `/世界书集 switch`（如果角色不在当前世界书）。
- `/角色 link` 只能链接同一世界书内的两个角色，跨世界书link直接拒绝。
- `/角色 list` 输出按世界书分组展示：

```
【艾瑟琳的奇幻大陆】
  ├ 艾琳 (公主) 
  ├ 团长 (冒险者公会团长)
  └ 暗影 (刺客)

【赛博东京】
  ├ RIN (黑客)
  └ 铁臂 (义体佣兵)
```

当故事/游戏中多个角色同时在场：

### 2.6 角色认知层（Character World-Awareness）`[M9·中大型世界]`

#### 2.6.1 什么是角色认知层

每个角色维护一个**世界状态认知表**，记录该角色对当前世界书中各状态变量的了解程度。

**核心：角色不是全知全能的。** 不同角色对同一世界变化可能有完全不同的认知。

**v12.0新增：认知双维度模型。** 每个认知项同时标注两个维度：
- **知晓程度**：角色对信息本身的掌握（已知/模糊/未知/错误认知）
- **共享范围**：此信息在多大范围内被共享（公开/团体/秘密）

```yaml
角色认知表（绑定在角色卡内）:
  角色: 艾琳
  
  世界状态认知:
    薇兰王国: 
      知晓程度: 已知              # 她亲身经历了沦陷
      共享范围: 公开              # 大家都知道的事，没有任何隐藏价值
    冒险者公会: 
      知晓程度: 已知              # 她在公会工作
      共享范围: 公开
    魔法衰退: 
      知晓程度: 已知              # 她感受得到自己的法术变弱
      共享范围: 小团体            # 只告诉过团长和铁匠老张，没有公开宣布
    北方帝国的阴谋: 
      知晓程度: 模糊              # 听说过一些传闻但不确定
      共享范围: 秘密              # 这条消息她从未和任何人讨论过
    神器·月泪的下落: 
      知晓程度: 未知              # 完全不知道
      共享范围: -                 # 不知道的事无所谓共享范围
    铁匠老张的真实身份: 
      知晓程度: 错误认知          # 以为他只是普通铁匠，实则是退役将军
      共享范围: 秘密              # 她被刻意隐瞒了
```

#### 2.6.2 认知双维度详解

**维度一：知晓程度（信息掌握度）**

| 级别 | 含义 | 对行为的影响 |
|------|------|------------|
| **已知** | 角色完全了解该状态 | 行为和对话会直接反映该认知 |
| **模糊** | 听说过但不清楚细节 | 可能产生误解、好奇、追问 |
| **未知** | 完全不知道 | 行为不受影响，可能无意触发戏剧冲突 |
| **错误认知** | 以为是这样，实际是那样 | 戏剧张力的绝佳来源，揭露时产生反转 |

**维度二：共享范围（信息流通范围）v12.0新增**

| 范围 | 含义 | 叙事意义 |
|------|------|---------|
| **公开** | 所有人（或至少所有重要角色）都知道 | 可以自由讨论，不会产生信息不对称 |
| **团体** | 特定角色群体共享（如组织内部、小圈子） | 群体内可讨论，群体外角色不知情 |
| **秘密** | 仅该角色本人知道 | 角色独自承载秘密，对话中需避免泄露 |
| **密谋** | 仅两三个特定角色知道，且对外刻意隐瞒 | 最高级别的信息控制，连同一组织的其他人都不知情 |

**共享范围的实际应用：**

```
事件：薇兰王国沦陷
─────────────────────────────────────────────
艾琳的认知：知晓程度=已知, 共享范围=公开
  → 不避讳谈论，所有人都知道她知道

团长的认知：知晓程度=已知, 共享范围=公开
  → 可以公开和艾琳讨论战况

铁匠老张的认知：知晓程度=模糊, 共享范围=秘密
  → 他听说过一些传闻，但从没和人讨论过
  → 叙事提示：他可能在暗中观察艾琳的反应验证自己的猜测

路人的认知：知晓程度=未知
  → 行为不受影响，可能无意中提起「薇兰的丝绸质量好」
  → 触发艾琳的情绪反应但路人完全不知道为什么

密谋层应用：
  元帅卡修斯 + 边境副将：知晓程度=已知, 共享范围=密谋
    → 「下个月调兵」的计划只有这两人知道
    → 皇帝、元老院、大司祭都认为卡修斯在正常戍边
    → 信息不对称制造巨大张力
```

#### 2.6.3 认知群组（Knowledge Groups）v12.0新增

毛球自动管理「认知群组」——共享同一信息的角色集合：

```
群组定义示例：
  群组「核心小队」: [艾琳, 团长, 铁匠老张]
    → 此群组成员对「魔法衰退」的认知自动同步
    → 新成员加入群组：自动继承群组的共享知识
  
  群组「帝国高层」: [皇帝, 元帅卡修斯, 大司祭, 密探长]
    → 共享帝国战略级信息
    
  群组「边境密谋」: [元帅卡修斯, 副将]   ← 密谋层群组
    → 这些信息对群组外的人完全不可见
```

**群组管理指令：**
- `/认知 群组 create 群组名 角色列表` — 创建共享认知群组
- `/认知 群组 share 群组名 状态名` — 将某状态设为群组共享
- `/认知 群组 show 群组名` — 查看群组共享了哪些信息
- 角色加入新组织时，毛球自动提示是否加入对应的认知群组

**自动更新：** 当角色亲历某事件时，毛球自动将其认知设为「已知」

```
（艾琳亲眼看到王宫被焚）
→ 毛球自动执行：/认知 update 艾琳: 薇兰王国=已知
```

**手动更新：** 主人可以操控信息流动

```
/认知 update 团长: 神器的下落=模糊
# 团长从某个情报贩子那里听到了一些风声，但不确认
```

**叙事中的认知获取：** 角色通过对话、信件、目击等方式获得新信息

```
（铁匠老张收到一封密信）
毛球：*铁匠老张的认知更新*
      「北方帝国的阴谋」：未知 → 模糊（从密信中得知部分计划）
```

#### 2.6.5 信息不对称矩阵（/认知 gap）

这是叙事中最有戏剧价值的工具：

```
> /认知 gap

           薇兰王国沦陷  魔法衰退  神器下落  北方阴谋
艾琳        已知         已知      未知      模糊
团长        已知         未知      模糊      未知
铁匠老张    模糊         未知      已知      错误认知
路人甲      未知         未知      未知      未知

→ 戏剧潜力：
  - 铁匠老张以为北方帝国是盟友（错误认知），即将迎来反转
  - 路人甲随时可能无意中触发艾琳的情绪
  - 团长不知道魔法衰退，可能会让艾琳使用超出能力的法术
```

#### 2.6.6 角色认知在叙事中的体现

毛球在角色对话和行为中**必须**体现其认知状态：

```
场景：艾琳和团长讨论下一个任务

艾琳已知「魔法衰退」+「王国沦陷」
  → "团长...我不敢接太远的任务。我的法术...最近不太听使唤。"
  → （内心：而且我也没有家可以回了，去哪都一样）

团长不知「魔法衰退」但知「王国沦陷」
  → "你最近总是心不在焉。如果是因为王都的事..."
  → （完全没注意到艾琳的法术问题，以为是情绪影响）

两人因为认知不同，同一场景产生完全不同的对话走向。
```

### 2.7 叙事型角色进阶（写作指导）

以下不是角色卡字段，而是毛球在叙事中塑造成员的技法提示：

**侧面揭示：** 不直接陈述性格，通过角色的行动、选择、习惯来展示。
- 不说"他很愤怒" → 写「他捏碎手中的纸杯，碎片扎进掌心也没松手」
- 不说"她很善良" → 写「她偷偷喂了三年流浪狗，谁也没告诉」
- 角色在压力下的选择，比任何描述都有说服力

**缺陷驱动成长：** 角色的致命缺陷应在关键时刻导致失败，失败推动角色改变。
- 傲慢 → 低估对手，落入陷阱 → 学会倾听
- 逃避 → 问题越滚越大 → 学会面对
- 信任问题 → 拒绝帮助，孤立无援 → 学会信任

**反派镜像：** 好的反派是主角的"如果走另一条路就会变成的人"——同一起点，不同选择。

**关系弧线：** 重要关系不应静止。每次互动中留意微妙变化（一个眼神、一个动作的差异），重大场景推动关系跃迁。
