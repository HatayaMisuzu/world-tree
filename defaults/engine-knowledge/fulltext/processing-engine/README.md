# 处理/补全引擎

> world-tree skill 前置子系统。素材库→发现→补全→推理→七步流水线→用户判断→交付。

## 数据目录

运行时数据独立存放：
```
D:/world-tree-data/_processing_engine/
├── material_pool/       ← 素材库（用户投喂+联网搜集）
├── discoveries/         ← 发现结果
├── proposals/           ← 补全提案
├── reasoning/           ← 推理验证
├── pipeline_reports/    ← 七步处理报告
├── change_log.json      ← 修改记录
├── current_draft/       ← 当前版本设定
└── packages/            ← 打包设定包
```

## 子目录说明（skill内部文档）

```
processing-engine/
├── README.md            ← 本文件（引擎总入口）
├── materials/           ← 素材处理规则
├── pending/             ← 待处理文本规则
├── processed/           ← 已处理文本规则
├── trash/               ← 垃圾桶规则
└── reports/             ← 报告模板
```

---

## 一、素材库（数据环）

投喂 + 联网搜集 → 存入 `D:/world-tree-data/_processing_engine/material_pool/`

```
/引擎 接收 [文本/文件/链接]    ← 投喂原始素材
/引擎 搜集 [关键词]            ← 联网搜索
/引擎 素材库                    ← 查看素材库
```

---

## 二、发现引擎

从素材库匹配与用户设定相似/互补的好点子。

匹配维度：同类型设定 / 冲突解决方案 / 空白补全 / 热点趋势 / 独特创新点

```
/引擎 发现                ← 运行点子发现
/引擎 发现 [维度]          ← 特定维度
```

---

## 三、补全提案

基于发现结果，对原始文本提出补充修改建议。每提案含：原文引用→问题→方案选项→素材依据→影响范围。

```
/引擎 补全                ← 生成提案
/引擎 采纳 [ID]            ← 采纳
/引擎 拒绝 [ID]            ← 拒绝
```

---

## 四、推理验证

推演补全后的设定：时间/因果/角色/世界/交互

```
/引擎 推理                ← 运行推理
```

---

## 五、七步处理流水线

| 步骤 | 检查维度 | 满分 |
|------|---------|------|
| ① 逻辑检查 | 因果完整性(10)/规则自洽(10)/推理闭合(10)/边界清晰(10) | 40 |
| ② 矛盾检测 | 设定内矛盾(10)/跨维度矛盾(10)/数值矛盾(10)/隐含矛盾(10) | 40 |
| ③ 流畅度 | 信息密度(10)/递进自然(10)/冗余度(10)/衔接平滑(10) | 40 |
| ④ 创新性 | 独创度(10)/组合创新(10)/意外感(10)/反套路(10) | 40 |
| ⑤ 读者观感 | 吸引力(10)/理解负担(10)/情感共鸣(10)/记忆点(10) | 40 |
| ⑥ 整体性 | 主题统一(10)/风格一致(10)/比例协调(10)/世界呼吸感(10) | 40 |
| ⑦ 综合 | 六步汇总+强弱项分析 | 280 |

```
/引擎 处理                  ← 运行全部
/引擎 处理 [步骤]            ← 单步
/引擎 处理 报告              ← 查看报告
```

---

## 六、用户判断

引擎只提建议，用户最终决定。

```
/引擎 采纳 P001 | /引擎 拒绝 P002 | /引擎 修改 [位置] [内容]
→ /引擎 重处理 → 满意 → /引擎 打包 → /引擎 交付 [包名]
```

---

## 七、与世界树的接口

```
/引擎 交付 [包名]
→ 生成设定包 → D:/world-tree-data/_processing_engine/packages/
→ 世界树: /引擎 new --from-package [包名]
```
