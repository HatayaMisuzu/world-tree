# v0 and image2 Generation Brief v1

用途：把下面的分段需求分别给 v0 和 image2。v0 负责界面组件/布局方向，image2 负责背景、徽记、纹样、图标和装饰资产。生成结果给 Codex 后，会筛选并接入 World Tree Desktop。

## v0 Prompt 1: 主菜单界面

为一个 Electron 桌面应用设计游戏化主菜单 UI，项目名 World Tree Terminal。风格是成熟二次元轻幻想、世界树、视觉小说/RPG 启动界面，不要程序员 dashboard，不要命令行，不要大面积深色控制台。背景使用一张奇幻世界树场景图作为 full-bleed 背景，上面有动态风感的视觉层，但 UI 设计只需要静态组件。

画面内容：

- 居中偏右的大标题 WORLD TREE。
- 标题上方是圆形世界树徽记占位。
- 四个主按钮：开始旅程、读取记忆、世界设定、观测终端。
- 按钮为纸质浅色长条，左侧有叶片、书本、罗盘、枝芽或观测图标。
- 底部只显示一个小状态条：LLM LINK ONLINE。不要显示 data path、world core、memory、debug。
- 右下角一个小圆形专家/异常入口图标，低存在感。

交互状态：

- hover 时按钮轻微亮金边和纸张浮起。
- selected/focus 状态清晰但不刺眼。
- 桌面尺寸优先 1600x900，同时给 1280x720 和 375x812 的 responsive 方案。

输出要求：

- 给 React + CSS 或 HTML + CSS 均可。
- 不要使用 Tailwind 默认紫色风格。
- 不要把功能说明文字写在界面上。
- 所有命令、路径、API、JSON 都不要出现在主菜单。

## v0 Prompt 2: 对话页

设计 World Tree Terminal 的玩家对话页。它是一个奇幻世界树 AI 叙事终端，但用户看到的是沉浸式对话，不是开发控制台。

布局：

- 顶部窄栏：返回主菜单、当前世界名、简短世界脉象，例如“世界脉象 稳定 / 神秘度升高”。
- 左侧可选轻量会话列表，但默认窄而安静。
- 中央为对话流，纸张质感背景，角落有淡植物纹样。
- 底部为输入栏，圆角胶囊形，发送按钮是叶片/羽箭图标。
- 消息气泡：玩家浅绿，世界树/引导者米白，系统提醒淡金。

隐藏规则：

- 不显示 API、路径、overlay、module、prompt、JSON。
- 待确认变更只作为小提示条：“有 2 个世界变化等待确认”，点击进入确认弹窗。
- 技术错误不要裸露堆栈，显示“连接异常 / 重新连接 / 打开设置”。

输出要求：

- 页面要简洁，不要有抢注意力的大面板。
- 对话区域和输入栏必须适配窗口大小。
- 文案中文。

## v0 Prompt 3: 世界设定页

设计 World Tree Terminal 的“世界设定”页面。它面向玩家和创作者，不是开发者。

信息架构：

- 左侧分区导航：通用设置、模型连接、叙事风格、世界书、角色与场景、关系网络、素材工坊、高级。
- 默认打开通用设置。
- 模型连接中把 `Base URL` 改叫“服务地址”，`API Key` 改叫“访问密钥”，模型叫“模型名称”。
- 世界书以条目卡片展示，可启用/停用、预览触发效果。
- 素材工坊收纳 processing/alchemy 功能，用“导入素材、识别条目、生成建议”表达。
- 高级区默认折叠，里面才放路径、命令、原始 JSON、Power User。

视觉：

- 浅纸张底，边角精致植物纹样。
- 面板分区清晰但不要厚重卡片套卡片。
- 表单要成熟、安静、紧凑。

## v0 Prompt 4: 观测终端

设计 World Tree Terminal 的“观测终端”。重点不是技术健康度，而是世界当前状态的量化快照。

第一屏：

- 世界脉象总览：综合状态、趋势箭头、最近一轮变化。
- 指标：稳定度、混乱度、神秘度、战争风险、角色压力、阵营冲突、规则完整度、记忆负载。
- 每个指标使用小型仪表/条形/刻度，不要像服务器监控。
- 右侧显示“世界变更请求”：等待确认、可采纳、可拒绝、查看影响。

第二层：

- 系统诊断作为折叠小区块：文件、模块、队列、连接。
- LLM 状态显示为在线/离线。
- Guardian 叫“叙事守护”，只展示风险摘要。

视觉：

- 比主菜单更像终端，但仍然是纸质/植物/金绿，不要黑底命令行。
- 数据密度适中，适合用户长期观察。

## v0 Prompt 5: 读取记忆/存档页

设计 World Tree Terminal 的“读取记忆”页面。

功能：

- 存档卡片列表：世界名、上次时间、轮次、简短摘要。
- 历史会话列表：当前会话、历史会话。
- 分支树/时间树：主线、分支、已合并、已搁置。
- 操作按钮：继续旅程、查看摘要、比较分支、回到此记忆点。

隐藏：

- 不显示 `/分支 switch` 命令。
- 不显示 raw canon JSON，放在“原始数据”折叠里。

视觉：

- 像图书馆/档案书页，不像文件浏览器。
- 卡片要精致但克制。

## image2 Prompt A: 世界树圆形徽记

生成透明背景 PNG。内容是精致的世界树圆形徽记，细线稿，古金色/淡金色，圆环内是一棵完整世界树，树冠、枝干、根系都清晰，左右对称但不要机械。风格参考高端奇幻视觉小说 UI 徽章，不要方形白底，不要厚重黑线，不要复杂到缩小后糊掉。

要求：

- 透明背景。
- 1024x1024。
- 线条古金色。
- 中心树形清晰，适合放在 WORLD TREE 标题上方。
- 不要文字。

## image2 Prompt B: 主菜单动态图分层资产

基于奇幻世界树主视觉，生成可用于动态主菜单的分层资产建议。需要输出多张透明 PNG：

- 前景草叶和花瓣层，透明背景。
- 飘动发丝/红色衣摆局部层，透明背景。
- 远景云雾层，透明背景。
- 光斑/树叶粒子层，透明背景。
- 水面微光层，透明背景。

风格保持二次元奇幻、阳光、世界树、少女背影、风吹草动。不要改变主图主体构图。每张资产都要适合循环动画，边缘干净，无白底。

## image2 Prompt C: 页面边角植物纹样

生成透明背景 PNG。内容是朴素但精致的植物枝叶边角纹样，用于纸张 UI 四角装饰。风格是淡金/灰绿细线，参考古典书页边饰，不要廉价简笔画，不要像随手画的叶子。

要求：

- 透明背景。
- 2048x2048。
- 四角各有不同但统一的枝叶装饰，中间留空。
- 低对比度，适合叠在米白纸张背景上。
- 不要文字，不要白底。

## image2 Prompt D: 页面背景纸张纹理

生成无缝或大尺寸纸张背景。风格为浅象牙白、轻羊皮纸、非常淡的植物暗纹和世界树水印。用于对话页、设置页、观测终端。

要求：

- 3840x2160。
- 不能太花，不能抢文字。
- 边缘可有淡淡藤蔓。
- 中央保持干净。
- 适合作为 UI 背景。

## image2 Prompt E: 四个主菜单图标

生成一套透明背景 PNG 图标，统一古金/橄榄绿细线风格，适合放在纸质按钮左侧。

图标：

- 开始旅程：叶片/道路/晨光。
- 读取记忆：打开的书/书签。
- 世界设定：罗盘/小世界树。
- 观测终端：枝芽/星盘/脉象刻度。

要求：

- 透明背景。
- 每个 512x512。
- 线条简洁，缩小到 24px 仍可识别。
- 不要文字。

## image2 Prompt F: 世界变更确认弹窗装饰

生成透明背景 UI 装饰资产，用于“世界变更请求”弹窗。轻幻想纸张边框，细金线，植物角花，中央透明或浅纸色。

要求：

- 1600x900。
- 边框精致，中心留给文字。
- 不要重阴影，不要黑色描边。
- 不要文字。

## Codex 接入备注

收到 v0/image2 结果后，优先筛选：

- 透明背景是否真实透明。
- 图标缩小是否可读。
- 纸张纹理是否不抢文字。
- 主菜单资产能否循环动效。
- v0 组件是否把命令/路径/JSON 从主界面移走。

不合格时继续让 image2 补：

- 透明背景版本。
- 更细线条版本。
- 更低对比度纹样版本。
- 图标 24px 可读版本。
