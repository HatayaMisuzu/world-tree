# v0 Revision Prompt: World Tree v0.1.8 Frontend Prototype Round 2

请基于你已经生成的 `F:\world-tree` 前端原型继续修订，不要从零重做。当前原型的整体方向是可用的：6 个一级导航、左侧桌面导航、移动端底部导航、首页“世界书 + 存档 + 加载后对话主体”的结构都应保留。

本轮目标是：补齐当前 World Tree HTML 控制台已有功能的展示覆盖，修复中文文案/编码问题，移除具体原创故事素材，并补全 mock 数据，让这个原型成为更完整的 v0.1.8 前端设计稿。

- 保留桌面端左侧导航。

- 保留观测页内部 tabs：摘要、叙事黑盒、世界脉象、世界构成、健康体检。

## 必须修复

1. 修复所有中文乱码。

当前源码和 README 中存在类似 `鍙欎簨`、`涓栫晫`、`瀛樻。` 这样的乱码。请确保所有 `.tsx`、`.ts`、`.css`、`.md` 文件都使用正常 UTF-8 中文。

示例：

- `鍙欎簨寮曟搸` 应为 `叙事引擎`
- `涓栫晫` 应为 `世界`
- `瀛樻。` 应为 `存档`
- `宸ヤ綔鍙?` 应为 `工作台`

2. 移除具体原创故事素材。

当前截图中出现了类似“灰烬群岛”“守钟人”等具体虚构世界、角色和剧情。这一版原型不能内置具体故事内容，因为真实素材会由用户后续提供。

请改为中性占位内容：

- `未命名世界`
- `示例世界 A`
- `角色卡 1`
- `世界书条目 1`
- `存档 1`
- `来源片段占位`
- `叙事摘要占位`

允许展示字段结构和状态，但不要写具体世界观、剧情、角色台词、地点设定。

3. 版本与项目名统一。

- `package.json` name 改为 `world-tree-frontend-prototype` 或 `world-tree`。
- version 改为 `0.1.8`。
- README 标题和说明中使用 `World Tree v0.1.8`。
- 不要保留 `my-project`。
