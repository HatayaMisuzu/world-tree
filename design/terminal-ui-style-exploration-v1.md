# World Tree Terminal UI Style Exploration v1

目标：把 World Tree Desktop 从“开发控制台”改成“玩家可直接使用的二次元世界树终端”。界面仍保留强大的引擎能力，但玩家第一眼看到的是旅程、世界、记忆和状态，而不是路径、JSON、命令和调试面板。

## 设计定位

关键词：成熟、安静、精致、轻幻想、纸张质感、植物纹样、世界树徽记、游戏主菜单、可长期停留。

不走方向：赛博控制台、程序员 dashboard、密集命令行、过强发光、廉价花边、过度可爱、满屏说明文字。

核心观感：主菜单像一款视觉小说/RPG 的启动界面；对话页像“世界树书页中的通讯终端”；设置和监控像被收纳在书册里的工具页。

## 信息架构

玩家第一层只保留 4 个入口：

- 开始旅程：进入对话/互动主流程。
- 读取记忆：读取存档、历史会话、世界快照。
- 世界设定：玩家能理解的设置、模型连接、显示、语言、自动保存。
- 观测终端：世界脉象、运行状态、待确认事件、LLM 在线状态。

第二层入口收纳在对应页面内：

- 世界书、角色、场景、关系、时间线：放入“世界设定”的左侧分区，不在主菜单展开。
- 分支、存档、回滚：放入“读取记忆”，用“时间树/分支树”表达。
- 世界脉象、健康检查、待确认提案、Guardian 结果：放入“观测终端”，区分“世界状态”和“系统诊断”。
- 命令、路径、JSON、Power User、overlay audit：放入“专家抽屉”，默认隐藏，需要点击“小齿轮/高级”进入。

## 页面策略

主菜单：

- 背景为动态世界树场景，按钮组居中略偏右，避免压住少女主体。
- 不显示本地路径、不显示“未命名会话”、不显示开发状态文本。
- 状态条只显示 `LLM LINK ONLINE/OFFLINE`，其余 world core/memory 等开发指标隐藏到观测终端。
- 右下角保留一个小型圆形“系统/异常”入口，低存在感。

对话页：

- 不使用主菜单背景，改用轻纸张、藤蔓边角、柔和世界树纹样。
- 输入框固定在底部，发送按钮用叶片/羽箭图标。
- 消息区域宽松、克制，用户消息偏浅绿，系统/世界树消息偏米白。
- 顶部只放返回主菜单、当前世界名、简短世界脉象，不显示 API/path/debug。

读取记忆：

- 核心是会话列表、存档卡片、世界分支树。
- “复制命令/分支 switch”这类开发操作改成按钮：“切换到此分支”“比较差异”“回到此记忆点”。
- JSON 详情默认折叠在“原始数据”中。

世界设定：

- 玩家默认看到：语言、主题、模型连接、自动保存、世界书条目开关、角色/场景管理。
- 创作者工具单独成区：创作向导、素材处理、世界书导入、关系/时间线编辑。
- 高级设置放在专家抽屉：路径、命令、overlay、Power User、默认 slash command。

观测终端：

- 第一屏显示“世界脉象”，不是技术健康度。
- 维度包括稳定度、混乱度、神秘度、战争风险、角色压力、阵营冲突、规则完整度、记忆负载。
- 技术诊断降级成小标签：“系统诊断”，需要展开才显示文件/模块/队列信息。
- 待确认提案以“世界变更请求”出现：采纳、搁置、拒绝、查看影响。

## 功能显隐原则

玩家常用，应直接显示：

- 开始/继续、读取存档、设置模型、发送消息、查看世界脉象、确认重大变更、查看世界书/角色/场景摘要。

玩家偶尔用，应放二级界面：

- 分支比较、关系网、时间线因果、记忆搜索、创作向导、素材处理。

高级用户才用，应默认隐藏：

- slash command、overlay 路径、raw JSON、audit log、engine packet、prompt、Power User、自定义命令。

不应作为 UI 文案出现：

- `data/engine/...`
- `overlay`
- `moduleKey`
- `IPC`
- `LLM Base URL` 这种技术名应在设置内改写为“服务地址”
- `/分支 switch`、`/引擎 status` 等命令文本

## 视觉规范

色彩：

- 主背景：天空蓝、树荫青绿、夕阳金、深红点缀。
- UI 底色：象牙白、浅羊皮纸、淡绿灰。
- 强调色：橄榄绿、古金、低饱和赤红。
- 避免：大面积深蓝控制台、大面积纯米色、荧光绿命令行、紫色渐变。

字体：

- 标题：优雅衬线或接近视觉小说标题感的 serif。
- 正文：清晰中文屏显字体，字号克制。
- 数值/状态：可用小型等宽，但只用于观测终端，不进入主菜单。

组件：

- 按钮：纸质矩形、轻微内阴影、左侧图标，半径 6-8px。
- 卡片：只用于存档、世界条目、提案，不做卡套卡。
- 边角纹样：植物枝叶沿边角淡淡出现，透明度低，必须精致。
- 徽记：世界树圆章，线条细、树冠完整，不要方形底。

## 新功能整合

- 世界脉象：放观测终端首页，主对话页只显示一句摘要。
- 提案确认：命名为“世界变更请求”，放观测终端和对话页轻提示。
- Pending 写入：用户文案为“等待你确认的世界变化”。
- Guardian：文案为“叙事守护”，只在有风险时提示。
- 关系网：放世界设定/世界关系。
- 时间线因果：放读取记忆/时间树，或世界设定/历史。
- 分支系统：放读取记忆/分支树。
- Director 模式：放世界设定/叙事风格，不叫 director。
- Processing/Alchemy：放世界设定/素材工坊，不叫 processing engine。
