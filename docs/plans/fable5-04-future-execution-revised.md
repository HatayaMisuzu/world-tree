# World Tree 未来发展执行文件（修订版）

> 修订说明：本版保留 fable5 的激进产品化节拍，但修正对外竞品表述与证据口径：公开文案避免"同类没有"等绝对句；当前 FTP-10 记为"按仓库可复现证据为 0/BLOCKED"；PLAYABLE 仍必须依赖真实 LLM、真人试玩、录屏三证。


性质：这是接下来两个季度的作战文件——定位、版本节奏、社区运营、开发治理、风险应对，全部落到可勾选的执行项。与《02》《03》的关系：02 管"修"，03 管"建"，本文管"往哪走、按什么节拍走、怎么让别人加入"。

## 1. 定位宣言（先定死，再谈功能）

一句话：**World Tree 是本地优先的"规则约束叙事引擎"——AI 负责讲故事，但无权篡改你的世界。**

三条支柱（全部已有代码基础，不是愿景）：

1. **正典受保护**：AI 的一切设定变更走提案审核，玩家批准才入档（proposal/review 系统）。
2. **秘密守得住**：GM 私有状态与玩家可见视图硬隔离（hidden-leak 防线 + 知识边界），支撑推理/剧本杀这类"信息不对称"玩法。
3. **玩法有结构**：骰子、线索板、资源、回合帧——比纯聊天多出的那半步游戏性。

对位表（写进 README，回答"为什么不用 X"）：

| | SillyTavern | AI Dungeon 类云服务 | World Tree |
|---|---|---|---|
| 数据归属 | 本地 | 云端订阅 | 本地 |
| 核心场景 | 角色陪聊 | 自由文冒 | 结构化叙事游戏 |
| AI 改设定 | 多依赖提示词/角色卡约束 | 多依赖服务侧生成策略 | **须经你批准** |
| 隐藏信息玩法 | 非核心能力，通常需手工约束或扩展 | 非核心卖点 | **原生支持** |

**不做清单**（同样重要，防止 agent 开发再次发散）：不做多人联机（v1.0 前）、不做云托管、不做图像生成、不做通用 agent 工作台、不新增第 9 个入口。公开传播时使用"更强调正典审核、隐藏信息边界、结构化玩法"，不要写成"竞品都没有"。

## 2. 北极星指标

**FTP-10（First Three Plays in 10 minutes）：新用户从启动到完成第 3 轮真实 AI 对话的 10 分钟达成率。**

测量方式（本地优先、无遥测上报）：内置 `npm run self-report` 脚本，读本机 usage.jsonl/chat.jsonl 生成匿名摘要，用户自愿贴到 GitHub Discussion 固定帖；发版前的 3 人真人测试必测此项。当前值：按仓库可复现证据记为 0% / BLOCKED（不是断言历史上从无人手动跑通过，而是没有可审计 PASS 记录）。v0.5 目标：≥ 80%（在已配 key 前提下）。辅助指标：单局平均轮数（留存代理）、demo 包安装率、ST 卡导入次数。

## 4. 节拍与纪律

- **双周节拍**：每两周一个 tag（哪怕只是 0.5.x 补丁），Release Note 固定三段：给玩家的（新内容/体验）、给创作者的、给开发者的。
- **内容与代码错峰**：每月中发 1 个内容包（03/D-3），与月末功能版本错开——保证社区每两周都有一次"有新东西"的感知。
- **每版本必附**：一段 ≥ 60s 实况录屏。没有录屏 = 不发版。这条纪律直接防止"没有可复现试飞证据"的问题复发。

## 6. 开发治理（针对 AI-agent 驱动开发的专项）

这个项目的开发方式（agent + truth-source）本身是资产，但需要三条新规矩防止再次"工程空转"：

1. **验收换轴**：docs/STATUS_TERMINOLOGY.md 增设 PLAYABLE 档（真 LLM 冒烟 + 真人试玩 + 录屏三证，03/E-3）。agent 可以自证"工程闭环"，**不可以单独自证 PLAYABLE**——后者必须由人类维护者基于真实 LLM 冒烟、真人试玩记录与录屏签发。这是本文最重要的一条规矩。
2. **文档瘦身**：docs/ 里 40+ 篇一次性闭环/审计报告移入 docs/archive/（DOC_REGISTRY.json 与 lifecycle 脚本已支持归档态，执行成本半天）；对外保留 ≤ 15 篇活文档：QUICKSTART、FEATURES、API_REFERENCE、ARCHITECTURE、SAVE_SYSTEM、SECURITY、TROUBLESHOOTING、CONTRIBUTING、真相源 4 件套、术语表。新规：一次性报告出生即入 archive/，不占根目录。
3. **agent 任务模板改造**：AI-GUIDE.md 中给 agent 的任务定义从"实现 X 并出闭环报告"改为"实现 X + 补测试 + **更新对应用户文档段落** + 在 PR 描述贴运行截图"。把"面向玩家的证据"塞进 agent 的完成定义里。
