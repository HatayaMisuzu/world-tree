# World Tree 产品信息架构

本文件记录 v0.5 产品体验重构后的稳定用户入口。它描述产品导航，不替代 API、数据边界或成熟度真相文档。

## 五个全局入口

| 入口 | 用户问题 | 主要内容 |
|---|---|---|
| 首页 | 我接下来做什么？ | 继续最近项目、首次示例、三个目标、最近项目、待处理状态 |
| 体验 | 我想进入怎样的世界？ | 世界探索、人物互动、桌面叙事、解谜调查、策略模拟、单人剧本杀 |
| 我的内容 | 我拥有哪些内容？ | 角色、世界书、世界数据、候选内容、审核队列和项目资产 |
| 创作 | 我如何使用自己的素材？ | 快速设定与炼金台，按“素材—草稿—预览—保存”推进 |
| 设置 | 我如何连接模型并管理应用？ | 模型连接、叙事偏好、备份导出和高级诊断 |

## 上下文页面

对话工作区、世界管理和项目观测继续保留，但它们由项目内动作打开，不占用全局导航。这样既保留全部既有能力，也避免把工程模块当成用户的第一层选择。

## 八个规范产品入口的归属

`browser/app/product-registry.js` 仍是八个入口及其成熟度、能力、限制的唯一前端注册表：

- 体验：人物互动、世界探索、桌面叙事、解谜调查、策略模拟、单人剧本杀。
- 创作：快速设定、炼金台。

入口矩阵的浏览器验证必须先进入所属全局页面，再检查标题、输入、主动作、必要的次级动作和 API 产品闭环。首页不再同时铺开八个入口。

## 首页决策优先级

1. 有最近项目时优先继续；没有项目时选择体验。
2. 首次使用可直接安装内置示例。
3. 三个目标分别覆盖进入世界、人物互动和使用自己的内容。
4. 最近项目与待确认内容位于第二屏，不与主行动争夺注意力。
5. 模型未配置时明确提示，但不会伪装成已连接。
