# 体验工作区

v0.5 的体验工作区把所有可持续互动模式放进同一套可理解的循环：行动、世界回应、状态变化、显式确认与本地保存。

## 用户循环

1. 从首页继续项目，或在“体验”中创建项目。
2. 输入行动，观察分阶段生成和流式世界回应。
3. 查看当前世界状态；需要时切换到详细变化。
4. 对重大世界变化作出接受、稍后决定或拒绝的明确选择。
5. 每个完成回合自动保存；返回首页后可以继续，并可准备世界包导出。

## 消息语义

- 你的行动：用户输入。
- 世界回应：已完成且由服务端保存的叙事回合。
- 系统提示：错误、流程边界或确认结果。
- 等待确认：候选变化，尚未进入正式状态。
- 已确认：经过用户确认的变化。
- 未完成片段：用户中止流式生成前已看到的文本，只在当前页面保留，不算完成回合。

## 流式中止与重试

停止生成会立即中断浏览器请求。可见片段保留在当前页面并标记“已中止 · 未保存”；服务端仅在 `done` 后写入完整回合。用户可以把原问题恢复到输入框，检查后重新发送。失败回合仍使用带失败回合标识的服务端重试路径。

## 自动化证据

`npm run smoke:experience-golden-path` 使用隔离数据目录与本地假模型，经真实浏览器验证：示例安装、三轮流式互动、自动保存、提案确认、返回首页、继续历史、世界包准备、中止片段边界、恢复输入以及移动端工作区。该脚本不记录密钥，也不替代真人体验签收。
